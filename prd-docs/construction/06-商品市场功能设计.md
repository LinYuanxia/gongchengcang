## 一、功能概述

### 1.1 功能定位

商品市场是施工方端**商品选购的核心入口**，施工方通过浏览、搜索、筛选找到所需建材并加入购物车，为后续下单采购做准备。

### 1.2 核心概念

| 概念 | 说明 | 示例 |
|:----|------|------|
| 工程仓商品 | 工程仓上架的商品，可见范围限于当前项目关联工程仓 | "南山工程仓-水泥32.5R" |
| 分类筛选 | 按商品分类Tab切换筛选，帮助快速定位品类 | "水泥"、"钢材"、"管材" |
| 商品详情 | 查看商品的完整信息与规格选择 | 价格/库存/规格参数 |

### 1.3 目标用户

- **采购员**（核心用户）：日常选购商品，浏览、搜索、加入购物车
- **管理员**：查看商品市场数据，了解可选商品范围
- **仓管员**：查看商品信息，辅助采购决策

### 1.4 模块范围

| 功能分类 | 主要功能 | 涉及角色 |
|:--------|---------|---------|
| 商品浏览 | 商品浏览列表、分类筛选 | 所有角色 |
| 商品搜索 | 关键词搜索、搜索历史 | 所有角色 |
| 商品操作 | 加入购物车、商品详情查看 | 管理员、采购员 |

## 二、核心设计原则

> **商品市场只展示当前项目关联工程仓的在售商品，遵循"所见即所购"原则。**

### 2.1 商品可见性约束

- 仅展示当前项目所选工程仓的在售商品
- 已下架/已删除/库存为0的商品默认不可见
- 商品信息数据来自上游工程仓端，施工方端只读消费

### 2.2 搜索优先原则

- 搜索框常驻顶部，支持全文搜索（商品名称+规格）
- 搜索结果与当前分类筛选叠加查询
- 搜索历史本地存储，支持一键清空

### 2.3 轻量列表原则

- 商品列表采用卡片式懒加载，首次加载20条，滚动触底追加
- 每个商品卡片展示关键信息（图片/名称/价格/库存），点击进入详情获取完整信息

## 三、业务规则

### 3.1 商品展示规则

- **默认排序**：综合排序=销量×0.5+评分×0.3+上架时间倒序×0.2
- **支持切换排序**：销量优先、价格升序/降序、上架时间
- **分页规则**：滚动触底加载，每页20条
- **筛选规则**：切换分类时重置搜索关键词；分类Tab展示商品数量角标
- **空状态**：无数据/搜索无结果时展示空状态插画

### 3.2 搜索规则

- 搜索范围：商品名称、规格属性（模糊匹配）
- 搜索历史：记录最近10条搜索历史，支持一键清空
- 搜索触发：输入关键词+回车/点击搜索按钮，防抖500ms

### 3.3 加入购物车规则

- 同一商品重复加入购物车 → 累加数量
- 库存不足时阻止加入并提示
- 加入成功后购物车Tab显示数量角标

### 3.4 实时性策略

| 场景 | 实时性策略 | 用户提示 |
|:----|:----------|---------|
| 商品列表 | 每次进入/刷新时获取最新数据 | 下拉刷新 |
| 商品详情 | 进入时获取最新 | 价格变化≥10%时提示 |
| 加入购物车 | 提交时后端校验库存 | 库存不足时提示 |

## 五、非功能性需求

### 5.1 性能要求

| 接口/场景 | 指标 | P95要求 | 说明 |
|:---------|:----|:-------:|------|
| 商品列表查询 | 响应时间 | ≤ 500ms | 分页20条/页 |
| 商品搜索 | 响应时间 | ≤ 500ms | 含分词匹配 |
| 商品详情查询 | 响应时间 | ≤ 300ms | 缓存30s |
| 加入购物车 | 响应时间 | ≤ 300ms | 含库存校验 |

### 5.2 埋点需求

| 页面 | 事件名 | 触发时机 | 上报字段 |
|:----|:------|---------|---------|
| 商品市场 | market_view | 进入商品市场页 | `warehouseId` |
| 商品市场 | market_search | 执行搜索 | `keyword`, `categoryId` |
| 商品市场 | market_category_click | 点击分类Tab | `categoryId` |
| 商品市场 | market_cart_add | 加入购物车 | `skuId`, `quantity`, `price` |
| 商品详情 | product_detail_view | 查看商品详情 | `skuId` |

### 5.3 安全要求

| 风险点 | 防护措施 | 实现方式 |
|:------|---------|---------|
| 越权加购物车 | 接口角色校验 | 仓管员角色调用购物车接口返回403 |
| 库存超卖 | 库存实时校验 | 加入购物车时校验最新库存 |
| 重复提交 | 前端防抖 | 按钮500ms防抖 |

## 六、功能点详细设计

### 6.1 商品浏览（P0）

#### 交互逻辑

1. 页面加载：请求商品列表接口（默认展示全部商品，按综合排序）
2. 下拉加载更多：触底自动加载下一页（每页20条）
3. 点击商品卡片 → 进入商品详情
4. 右下角"加入购物车"按钮快捷操作
5. 支持下拉刷新

#### 原子字段定义

| 字段 | 类型 | 必填 | 来源 | 校验规则 | 展示规则 | 默认值 |
|:----|:----|:----:|:----|:--------|:--------|:-----:|
| 商品图片 | URL | 否 | 商品接口 | URL格式 | 正方形缩略图，加载失败显示占位图 | - |
| 商品名称 | String(100) | 是 | 商品接口 | 非空 | 左对齐，单行截断 | - |
| 规格 | String(50) | 否 | 商品接口 | - | 次要文本 | - |
| 销售价 | Decimal(10,2) | 是 | 商品接口 | >0 | 红色加粗，右对齐，单位"元" | - |
| 库存 | Integer | 否 | 商品接口 | ≥0 | 展示"库存{N}" | 0 |

#### 边界情况覆盖

| 场景 | 处理逻辑 | 提示文案 |
|:----|:--------|---------|
| 列表为空 | 显示空状态插画 | "暂无商品" |
| 图片加载失败 | 显示默认商品占位图 | - |
| 库存为0 | 卡片置灰，隐藏加入购物车按钮 | "暂时无货" |
| 触底加载无更多数据 | 显示"已加载全部" | "已经到底了" |

---

### 6.2 分类筛选（P0）

#### 交互逻辑

1. 展示一级分类Tab（横向可滚动）
2. 点击分类Tab → 商品列表刷新为该分类下商品
3. 选中态高亮：蓝色底部线+加粗文字
4. 分类Tab显示该分类商品数量角标
5. 切换分类时重置搜索关键词

#### 原子字段定义

| 字段 | 类型 | 必填 | 来源 | 校验规则 | 展示规则 | 默认值 |
|:----|:----|:----:|:----|:--------|:--------|:-----:|
| 分类ID | Integer | 是 | 分类接口 | 非空 | - | - |
| 分类名称 | String(20) | 是 | 分类接口 | 非空 | Tab文字 | - |
| 商品数量 | Integer | 否 | 分类接口 | ≥0 | 数字角标 | 0 |
| 选中态 | Boolean | 否 | 前端状态 | - | 蓝色底部线+加粗 | false |

#### 边界情况覆盖

| 场景 | 处理逻辑 | 提示文案 |
|:----|:--------|---------|
| 分类无商品 | 分类Tab正常显示数量=0，选中后列表显示空状态 | "该分类暂无商品" |
| 分类数据加载失败 | 默认不显示分类Tab，商品列表展示全部 | - |

---

### 6.3 搜索商品（P0）

#### 交互逻辑

1. 点击搜索框 → 弹出键盘+展示搜索历史（最近10条）
2. 输入关键词 → 防抖500ms后自动搜索
3. 搜索结果显示在商品列表区域（卡片展示）
4. 支持回车确认搜索
5. 无结果时展示空状态插画
6. 搜索历史支持单条删除和全部清空

#### 原子字段定义

| 字段 | 类型 | 必填 | 来源 | 校验规则 | 展示规则 | 默认值 |
|:----|:----|:----:|:----|:--------|:--------|:-----:|
| 关键词 | String(50) | 否 | 用户输入 | 长度≤50 | 搜索框+清空按钮 | 空 |
| 搜索历史 | Array[String] | 否 | 本地存储 | 最多10条 | 列表+一键清空 | [] |

#### 边界情况覆盖

| 场景 | 处理逻辑 | 提示文案 |
|:----|:--------|---------|
| 搜索无结果 | 显示空状态插画 | "未找到相关商品" |
| 输入为空搜索 | 不执行搜索，维持当前列表 | - |
| 搜索词过长 | 截断前50字符 | - |
| 搜索词含特殊字符 | 转义后搜索，防止XSS | - |

---

### 6.4 加入购物车（P0）

#### 交互逻辑

1. 点击"加入购物车"按钮 → 弹出数量选择器
2. 默认为1件，可使用步进器调整数量
3. 确认数量 → 调用加入购物车接口 → 校验库存
4. 加入成功：按钮变为"已加入"+红色角标数量增加
5. 购物车Tab显示当前项目购物车商品总数角标

#### 原子字段定义

| 字段 | 类型 | 必填 | 来源 | 校验规则 | 展示规则 | 默认值 |
|:----|:----|:----:|:----|:--------|:--------|:-----:|
| SKU ID | Integer | 是 | 商品数据 | - | - | - |
| 数量 | Integer | 是 | 用户选择 | ≥1，≤库存 | 步进器 | 1 |
| 库存 | Integer | 否 | 实时库存 | ≥0 | 展示最大可购买数 | 0 |
| 当前工程仓ID | String(32) | 是 | 前端全局状态 | 非空 | - | - |

#### 边界情况覆盖

| 场景 | 处理逻辑 | 提示文案 |
|:----|:--------|---------|
| 加入失败 | Toast提示 | "加入购物车失败，请稍后重试" |
| 库存不足 | 阻止加入 | "库存不足，最多可购买{库存}件" |
| 商品已下架 | 按钮置灰不可点击 | "该商品已下架" |
| 未选择工程仓 | 提示先选择工程仓 | "请先选择工程仓" |

---

### 6.5 商品详情（P0）

#### 交互逻辑

1. 加载商品详情 → 展示图片轮播+基本信息+规格参数+详情描述
2. 底部固定操作栏：加入购物车按钮
3. 图片轮播支持左右滑动查看
4. 规格参数可展开/收起

#### 原子字段定义

| 字段 | 类型 | 必填 | 来源 | 校验规则 | 展示规则 | 默认值 |
|:----|:----|:----:|:----|:--------|:--------|:-----:|
| 商品图片集 | Array[URL] | 否 | 详情接口 | - | 轮播图 | [] |
| 商品名称 | String(100) | 是 | 详情接口 | 非空 | 主标题 | - |
| 规格描述 | String(100) | 否 | 详情接口 | - | 次要文本 | - |
| 销售价 | Decimal(10,2) | 是 | 详情接口 | >0 | 红色加粗大号字体 | - |
| 市场价/原价 | Decimal(10,2) | 否 | 详情接口 | - | 灰色删除线 | - |
| 库存 | Integer | 否 | 详情接口 | ≥0 | "库存{N}件" | 0 |
| 商品详情 | RichText | 否 | 详情接口 | - | 富文本渲染 | - |

#### 边界情况覆盖

| 场景 | 处理逻辑 | 提示文案 |
|:----|:--------|---------|
| 商品已下架 | 底部操作栏隐藏 | "该商品已下架" |
| 详情加载失败 | 显示重试按钮 | "商品信息加载失败" |
| 价格变化超过10% | 进入详情时Toast提示 | "价格已更新，最新价¥{price}" |
| 图片加载失败 | 显示占位图 | - |

## 七、异常处理汇总表

| 异常场景 | 触发条件 | 前端处理 | 后端处理 | 提示文案 |
|:--------|:--------|:--------|:--------|---------|
| 商品列表加载失败 | 接口返回错误/超时 | 显示重试按钮 | 无特殊处理 | "商品加载失败，请重试" |
| 搜索接口异常 | 搜索请求失败 | Toast提示 | 无特殊处理 | "搜索失败，请重试" |
| 加入购物车失败 | 网络异常/后端错误 | Toast提示，保持购物车前状态 | 不写入购物车 | "加入购物车失败，请稍后重试" |
| 库存不足 | 实时库存为0或不足 | 阻止加入，提示 | 返回库存不足错误码 | "库存不足，最多可购买{库存}件" |
| 分类筛选接口超时 | 响应>3s | 默认展示全部商品 | - | - |
| 商品详情加载失败 | 接口异常 | 显示重试按钮 | 无特殊处理 | "商品信息加载失败" |
