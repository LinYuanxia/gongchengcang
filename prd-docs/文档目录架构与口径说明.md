# 产品文档目录架构与口径说明

> 本文档展示全套产品文档的目录结构和各模块口径定义。
> 帮助读者快速了解"哪里有啥、该看哪个"。

---

## 一、文档体系总览

### 核心原则：按期数组织，边界清晰

```
PRD文档/                      ← ✨ 核心：按期数组织！
└── 26.4.1期 - 0-1搭建       ← 每一期就是完整独立的一包！
    ├── 版本概览              ← 本期全景总览
    ├── 跨部门协同            ← 本期需要关注的协同点
    ├── 更新日志              ← 本期的更新记录
    ├── 工程仓端              ← 本期交付的工程仓端功能
    ├── 资源平台端            ← 本期交付的平台端功能
    ├── 供应商端              ← 本期交付的供应商端功能
    └── 施工方端              ← 本期交付的施工方端功能
```

### 顶层目录结构

```
项目总览/                     ← 全局，不随期数变
├── 项目总览
├── 系统功能说明书
├── 版本迭代记录
├── 跨部门协同PRD             ← 全局架构级的协同总纲
└── 文档目录架构与口径说明     ← 当前文档

PRD文档/                      ← 按"期数"组织，每一期独立完整
└── 26.4.1期 - 0-1搭建
    ├── 版本概览
    ├── 跨部门协同
    ├── 更新日志
    ├── 工程仓端
    ├── 资源平台端
    ├── 供应商端
    └── 施工方端

系统操作手册/                  ← 面向最终用户的操作指南
├── 平台端操作手册
├── 供应商端操作手册
├── 施工方端操作手册
└── 供应商端操作手册（示例）

历史归档/                     ← 旧版期数归档（空，待后续版本迭代后移入）
```

### 层次说明

| 层级 | 定位 | 维护频率 | 核心受众 |
|:----|:-----|:--------|:--------|
| **项目总览** | 全局定稿文档 | 低频（架构变更时更新） | 全员 |
| **PRD文档** | 按版本期数组织 | 每一期迭代新增一期 | 产品/开发/测试 |
| **系统操作手册** | 最终用户操作指南 | 功能上线后补充 | 最终用户 |
| **历史归档** | 过期的期数移入 | 只入不出 | 回溯查阅 |

## 二、目录架构详情

### 2.1 项目总览（全局，不随期数变）

```
📌 项目总览
├── 项目总览.md               ← 系统概述 + SaaS架构 + 交易链路 + 产品架构
├── 系统功能说明书.md         ← 四端功能清单对照总表
├── 版本迭代记录.md           ← 全版本发布历史总览
├── 跨部门协同PRD.md          ← 【全局】项目级跨系统协同总纲、接口契约、泳道图
└── 文档目录架构与口径说明.md  ← 当前文档，指路用
```

**口径说明：** 
- 这里的内容是**整个项目级**的，不是某一期的
- 内容相对稳定，仅在项目架构发生重大变更时更新
- 提供"先看概览、再读细节"的导航能力

---

### 2.2 PRD文档（核心：按期数组织！）

```
PRD文档
└── 26.4.1期 - 0-1搭建       ← 命名规范：YY.M.N-关键词
     ├── 版本概览              ← ✨ 本期全景：范围、目标、功能清单
     ├── 跨部门协同            ← 【本期视角】这个版本开发需要关注的协同点
     ├── 更新日志              ← 【本期的】PRD变更记录
     ├── 工程仓端
     │   ├── 文档总纲
     │   ├── 系统概览与架构
     │   ├── 业务流程设计
     │   └── 各模块功能设计
     ├── 资源平台端
     ├── 供应商端
     └── 施工方端
```

#### 📋 版本期数命名规范

```
YY.M.N-关键词
```

| 部分 | 含义 | 示例 |
|:----|:-----|:-----|
| YY | 年份后两位 | 26 = 2026年 |
| M | 月份 | 4 = 4月 |
| N | 本月第N个需求 | 1 = 本月第1期 |
| 关键词 | 本期核心主题 | 0-1搭建 / 购物车优化 |

**示例：**
- `26.4.1-0-1搭建` = 2026年4月第1个需求，系统从0到1搭建
- `26.5.1-购物车优化` = 2026年5月第1个需求，购物车体验优化

#### 🔄 后续版本迭代添加方式

```
PRD文档
   ├── 26.4.1期 - 0-1搭建
   ├── 26.5.1期 - 购物车优化    ← 下一期直接并列加在这里
   └── 26.5.2期 - 支付接入
```

#### 两个"跨部门协同"的区别

| 位置 | 文档 | 定位 | 读者视角 |
|------|:-----|:-----|:---------|
| **项目总览** | 跨部门协同**PRD** | 全局架构级 | 刚加入项目的架构师/负责人 |
| **某一期里** | 跨部门协同 | 本期开发级 | 当前迭代的开发/测试 |

### 2.3 系统操作手册

```
📖 系统操作手册
├── 平台端操作手册            ← PC管理后台图文操作指南
├── 供应商端操作手册          ← 供应商门户图文操作指南
├── 施工方端操作手册          ← 施工方小程序图文操作指南
└── 供应商端操作手册（示例）  ← 模板示例
```

**口径说明：** 
- 面向**最终用户**
- 以"系统真实截图 + 操作步骤"为核心
- 与PRD功能设计互补：PRD讲"要做什么"，操作手册讲"怎么操作"

---

### 2.4 历史归档

```
📦 历史归档
└── （当前为空，后续旧版本自动移入）
```

**归档规则：** 版本上线半年后，可将对应期数从「PRD文档」移入历史归档，按年份/季度组织。

| 查找历史版本资料 | 📦 历史归档 |
