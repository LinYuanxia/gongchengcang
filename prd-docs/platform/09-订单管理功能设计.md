## 一、功能概述

### 1.1 功能定位

订单管理是平台端**跨端订单监控**的入口，平台管理员可查看全平台（供应商+工程仓）所有订单的完整信息，追踪异常订单，协调售后处理。平台端订单管理的核心是"监控和协调"，不直接参与交易操作。

### 1.2 核心概念

| 概念 | 说明 |
|-----|------|
| 全平台订单 | 包括供应商→工程仓采购订单 + 工程仓→施工方销售订单 |
| 订单溯源 | 跨端查看订单从创建到完成的完整流转 |
| 售后协调 | 平台介入处理供应商/工程仓/施工方之间的售后纠纷 |

### 1.3 模块范围

| 功能分类 | 主要功能 | 优先级 |
|---------|---------|:------:|
| 订单查询 | 全量订单查询 | P0 |
| 订单查看 | 订单详情 | P0 |
| 订单追踪 | 订单状态追踪（时间线） | P1 |
| 数据导出 | 订单导出 | P2 |

---

## 二、业务规则

- 平台端可查看所有端的订单，不可编辑和操作
- 异常订单包含：售后中、超48小时未处理
- 订单详情展示三状态（订单状态/支付状态/发货状态）

## 三、功能点详细设计

### 3.1 全量订单查询（P0）

#### 功能说明
平台管理员可查询全平台所有端（供应商/工程仓/施工方）的订单，支持多维度筛选。

#### 入口路径
订单管理 → 订单查询

#### 交互逻辑
1. 页面加载：获取全量订单列表 → 支持分页展示（每页20条）
2. 多条件筛选：
   - 订单端: 全部/供应商端/工程仓端/施工方端
   - 订单编号：精确搜索
   - 订单状态：待付款/待发货/待收货/已完成/已取消/售后中
   - 时间范围：下单时间起止
3. 列表字段：订单编号/下单端/买家/金额/状态/下单时间/操作

#### 字段说明
| 字段 | 说明 |
|-----|------|
| 订单编号 | 全局唯一订单号 |
| 下单端 | 标识订单来源端（标签展示） |
| 买家 | 下单用户/单位名称 |
| 商品金额 | 订单商品总金额 |
| 状态 | 订单状态+售后状态组合标签 |
| 下单时间 | 下单时间 |

#### 业务规则
- 平台端只读查看，不可编辑
- 异常订单（售后中、超48小时未处理）红色标记

---

### 3.2 订单详情（P0）

#### 功能说明
查看单笔订单的完整信息，展示三状态（订单状态/支付状态/发货状态）。

#### 入口路径
订单列表 → 点击订单编号

#### 交互逻辑
1. 订单基本信息区：订单编号/下单时间/下单端/买家（只读）
2. 三状态卡片：
   - 订单状态卡片：待付款→待发货→待收货→已完成/已取消
   - 支付状态卡片：未支付→已支付→已退款
   - 发货状态卡片：未发货→已发货→已签收
3. 商品列表区：该订单包含的商品明细（SKU/数量/单价/小计）
4. 收货信息区：收货人/电话/地址

#### 字段说明
| 字段 | 说明 | 组件类型 |
|-----|------|:--------:|
| 订单编号 | 全局唯一 | Text |
| 下单端 | 来源端 | Tag |
| 订单状态 | 状态流转 | Steps |
| 支付状态 | 支付结果 | Steps |
| 发货状态 | 物流进度 | Steps |
| 商品列表 | 明细 | Table |
| 收货信息 | 地址 | Card |

---

### 3.3 订单状态追踪（P1）

#### 功能说明
以时间线方式展示订单从创建到当前的全流程状态变更记录。

#### 入口路径
订单详情 → 状态时间线Tab

#### 交互逻辑
1. 时间线展示：按时间倒序排列的状态变更记录
2. 每条记录包含：操作时间/操作人/状态变更（前→后）/备注
3. 支持折叠展开

---

### 3.4 订单数据导出（P2）

#### 功能说明
导出订单数据到Excel文件，用于对账和分析。

#### 交互逻辑
1. 点击"导出"按钮 → 弹窗选择导出条件
   - 导出范围：当前筛选结果全部/当前页/自定义
   - 导出字段：勾选需要导出的字段
2. 确认导出 → 异步生成文件 → 完成后提供下载链接
3. 导出字段：订单编号/下单端/买家/商品/数量/金额/状态/下单时间
