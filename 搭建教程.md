## 第一章：项目背景和技术栈

### 1.1 项目定位

我们要搭建的是**采供一体化平台 - 工程仓SaaS系统**，主要面向建筑工程行业的仓库管理场景。

**核心目标用户**：
- 工程项目负责人
- 仓库管理人员
- 采购人员

**主要业务场景**：
- 多仓库统一管理
- 库存实时监控
- 出入库流程管理
- 库存盘点和调拨

### 1.2 技术栈选型

| 技术 | 选型 | 说明 |
|------|------|------|
| 前端框架 | Vue 3 | Composition API + Script Setup |
| 编程语言 | TypeScript | 类型安全 |

## 第三章：路由配置和菜单系统

### 3.1 路由设计思路

**重要经验**：路由配置是SaaS系统的骨架。我们采用了「菜单即路由」的设计理念：

- 一级路由对应一级菜单
- 二级路由对应二级菜单  
- `hideInMenu` 控制是否在侧边栏显示
- `meta` 存储菜单标题、图标等信息

### 3.2 完整路由配置

创建 `src/router/index.ts`：

```typescript
import { createRouter, createWebHistory } from 'vue-router'
import type { RouteRecordRaw } from 'vue-router'

const routes: RouteRecordRaw[] = [
  {
    path: '/',
    redirect: '/dashboard',
    component: () => import('@/layouts/WarehouseLayout.vue'),
    children: [
      // 工作台
      {
        path: 'dashboard',
        name: 'Dashboard',
        component: () => import('@/views/dashboard/index.vue'),
        meta: { title: '工作台', icon: 'icon-home' },
      },
      
      // 仓库管理 - 核心模块
      {
        path: 'warehouse',
        name: 'WarehouseManage',
        redirect: '/warehouse/warehouse/list',
        meta: { title: '仓库管理', icon: 'icon-storage' },
        children: [
          {
            path: 'list',
            name: 'WarehouseList',
            component: () => import('@/views/warehouse/warehouse/list/index.vue'),
            meta: { title: '仓库列表' },
          },
          {
            path: 'detail',
            name: 'WarehouseDetail',
            component: () => import('@/views/warehouse/warehouse/detail/index.vue'),
            meta: { title: '仓库详情', hideInMenu: true },
          },
        ],
      },
      
      // 更多路由...
    ]
  }
]

const router = createRouter({
  history: createWebHistory('/gongchengcang/'),
  routes,
})

export default router
```

## 第六章：关键开发决策和经验

### 6.1 遇到的问题和解决方案

**问题1：Edit工具字符串匹配失败**
- 现象：修改大文件时经常出现 "String to replace not found"
- 原因：HTML结构复杂，缩进、空格等微小差异导致匹配失败
- ✅ 解决方案：改用 `Write` 操作完全重写文件，效率更高

**问题2：路由菜单和页面不同步**
- 现象：新增页面后侧边栏不显示
- 原因：忘记配置路由meta信息
- ✅ 解决方案：建立「新增页面 Checklist」：
  1. 创建vue文件
  2. 路由配置
  3. 检查meta.title和icon
  4. 检查hideInMenu是否正确

**问题3：开发启动命令混淆**
- 现象：`npm run dev` 报错
- 原因：MonoRepo根目录和子目录命令不同
- ✅ 解决方案：
  - 根目录：`npm run dev:pc`
  - pc目录：`npm run dev`

### 6.2 开发最佳实践

1. **先设计数据结构再写UI**
   - 先定义TypeScript接口
   - 准备好mock数据
   - 最后写模板和样式

2. **表单验证要尽早做**
   - 每个输入框都要考虑空值情况
   - 给用户明确的错误提示
   - 提交前集中校验

3. **用户体验细节**
   - 点击仓库名称跳转详情
   - 状态用颜色区分（绿/黄/红）
   - 库存进度条随使用率变色
   - 批量操作先勾选再执行
