## 一、功能定位

### 1.1 工程仓端定位

工程仓端是工程材料管理采供一体化平台的**核心业务节点**，承担**双重角色**：

```
┌─────────────────────────────────────────────────────────────┐
│                    工程仓端（双重角色）                        │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  角色一：作为采购方（链路一）                                  │
│  ┌──────────────────┐    ┌──────────────────┐              │
│  │ 浏览供应商商品市场 │ →  │ 创建采购订单      │              │
│  │ 加购+下单         │    │ 收货入库          │              │
│  │                  │    │ 付款给供应商      │              │
│  └──────────────────┘    └──────────────────┘              │
│                                                              │
│  角色二：作为销售方（链路二）                                  │
│  ┌──────────────────┐    ┌──────────────────┐              │
│  │ 接收施工方订单    │ →  │ 确认销售订单      │              │
│  │ 出库发货          │    │ 收款开票          │              │
│  └──────────────────┘    └──────────────────┘              │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 系统设计哲学

| 原则 | 说明 |
|------|------|
| **三状态分离** | 订单状态、支付状态、发货状态三个轨道独立运行，互不影响 |
| **不做在线支付** | 只做线下转账凭证记录，不接三方支付网关 |
| **熟人生意** | 不做强校验，先发货后付款，信任机制优先 |
| **库存实时** | 库存随入库/出库实时更新，乐观锁防超卖 |
| **RBAC权限** | 通过角色控制功能访问权限 |

### 1.3 核心价值

- **支撑双交易链路**：连接供应商（采购端）和施工方（销售端），承上启下
- **库存管理中枢**：管理多子仓库的库存出入库和盘点
- **业务闭环**：从采购计划→商品市场→采购订单→入库→库存管理→出库→销售订单→对账

## 二、目标用户

| 角色 | 系统标识 | 核心职责 | 核心模块 | 使用端 |
|:----:|:-------:|---------|---------|:------:|
| **采购员** | purchaser | 浏览商品市场、加购下单、采购计划 | 商品市场、采购计划、采购订单 | PC |
| **仓管员** | storekeeper | 入库收货（含货损）、出库发货、盘点 | 仓库管理、商品中心 | PC |
| **主管** | admin | 数据看板、订单监控、员工/角色/账号管理 | 全部模块 | PC |
| **财务** | finance | 凭证上传、发票管理、对账 | 财务中心 | PC |
| **销售** | seller | 销售订单确认、发货 | 销售订单 | PC |

### 3.2 技术栈

| 层次 | 技术选型 |
|------|---------|
| 前端框架 | Vue 3 + TypeScript |
| UI组件库 | Arco Design（PC） |
| 状态管理 | Pinia |
| 路由 | Vue Router |

## 七、非功能性需求

| 指标 | 要求 | 说明 |
|:----:|:----:|------|
| 页面加载 | < 2秒 | PC端首屏加载时间 |
| API响应 | < 500ms | 95%请求的响应时间 |
| 并发用户 | 500+ | 工程仓用户规模较小 |
| 系统可用性 | 99.9% | 年度可用性 |
| 数据同步 | 实时 | 库存变动即时生效 |

**安全要求：**
- JWT Token + Refresh Token 认证
- RBAC角色权限模型
- 敏感数据AES加密存储
- HTTPS传输
- 关键操作审计日志
