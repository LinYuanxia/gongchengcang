## 一、功能概述

### 1.1 功能定位

商品管理是平台端**功能最密集的模块**（14个功能点），负责商品基础数据的全生命周期管理——从分类定义→属性规格→SPU/SKU创建→供应商供货关系→商品审核上架。平台端商品中心是整个系统的商品数据源头。

### 1.2 核心概念

| 概念 | 说明 |
|-----|------|
| 商品分类 | 三级树形分类体系，平台统一维护 |
| 属性组+属性值 | 规格属性定义（如"颜色:红/蓝/绿"） |
| SPU | 标准产品单元，商品定义最上层 |
| SKU | 库存量单位，具体规格商品 |
| 供货关系 | 供应商与SKU的关联+价格 |

### 1.3 目标用户

- **平台运营**：核心用户，管理商品基础数据
- **平台管理员**：审核商品上架

### 1.4 模块范围

| 功能分类 | 主要功能 | 优先级 | 功能数 |
|---------|---------|:------:|:------:|
| 分类管理 | 分类树管理 | P0 | 1 |
| 属性管理 | 属性组管理、属性值管理 | P0 | 2 |
| SPU管理 | SPU新增、SPU详情、SPU列表查询、SPU编辑 | P0+P1 | 4 |
| SKU管理 | SKU列表、SKU详情、SKU编辑 | P0+P1 | 3 |
| 供货管理 | 供应商供货列表、设置供货价、供货状态切换 | P0 | 3 |
| 批量操作 | 批量上下架 | P1 | 1 |

## 二、业务规则

### 2.1 分类规则
- 三级树形结构（level=1/2/3）
- 删除分类时校验是否有下级分类和关联商品
- 排序号越大越靠前

### 2.2 SPU→SKU规则
- 创建SPU时选择规格属性→系统自动笛卡尔积生成SKU
- SKU编码格式：SPU_ID + 规格值缩写
- 一个分类下SPU名称不可重复

### 2.3 供货规则
- 一个SKU可被多个供应商供应
- 供货价必须大于0
- 供货状态=停用时，工程仓端不可见
- 供货价变更→记录价格变更历史

## 三、功能点详细设计

### 3.1 分类树管理（P0）

#### 功能说明
维护商品分类的三级树形结构，支持增删改查、拖拽排序。分类是商品组织的基础，所有商品必须关联到具体分类。

#### 入口路径
商品中心 → 分类管理

#### 交互逻辑
1. 页面加载：获取全部分类 → 渲染树形结构
2. 拖拽排序：拖拽节点调整层级/排序 → 自动保存
3. 右键菜单：新增同级/新增子级/编辑/删除
4. 编辑即时刷新：修改后树结构即时更新

#### 字段说明
| 字段名称 | 字段说明 | 组件类型 | 默认值 | 数据来源 |
|---------|---------|---------|:------:|---------|
| 分类名称 | 分类文字名称 | Input | — | 表单输入 |
| 父级分类 | 上级分类（自动生成） | 树选择 | 当前选中 | 树结构 |
| 层级 | 1/2/3级（自动生成） | 只读文本 | 自动 | 树结构 |
| 排序号 | 同级排序（越小越靠前） | InputNumber | 0 | 表单输入 |

#### 业务规则
- 最多三级分类
- 删除分类时校验：有下级→不允许删除；有商品→二次确认后删除
- 同级分类名称不可重复

#### 异常处理
| 场景 | 处理方式 | 提示信息 |
|:----:|---------|---------|
| 有下级不可删除 | 按钮置灰+Tooltip | "该分类下有子分类，无法删除" |
| 有商品删除→确认 | Modal警告 | "该分类下有N个商品，删除后将影响这些商品" |
| 名称重复 | Toast | "同级已存在同名分类" |

#### 权限控制
平台运营/管理员可管理分类。

---

### 3.2 属性组管理（P0）

#### 功能说明
管理商品的规格属性分组，如"颜色组""尺寸组"。属性组是属性值的上层容器。

#### 入口路径
商品中心 → 属性管理 → 属性组管理

#### 交互逻辑
1. 属性组列表展示：名称+排序号+关联属性数
2. 新增弹窗：输入属性组名称+排序号
3. 编辑弹窗：修改属性组信息
4. 删除确认：校验是否有关联属性值

#### 业务规则
- 属性组删除时校验是否有关联属性值
- 属性组名称全局唯一

#### 异常处理
| 场景 | 处理方式 | 提示信息 |
|:----:|---------|---------|
| 名称重复 | Toast | "该属性组名称已存在" |
| 有属性值→删除 | Modal警告 | "该属性组下有N个属性值，删除后属性值将一并删除" |

---

### 3.3 属性值管理（P0）

#### 功能说明
管理每个属性组下的属性值，如"颜色组"下管理"红/蓝/绿"等颜色值。

#### 入口路径
商品中心 → 属性管理 → 属性值管理

#### 交互逻辑
1. 按属性组左侧筛选 → 右侧展示当前组所有属性值
2. 新增：弹窗输入属性值名称+规格值
3. 编辑/删除：行级操作

#### 业务规则
- 一个属性组内属性值名称不可重复
- 删除属性值时校验是否有SKU使用该属性值

#### 异常处理
| 场景 | 处理方式 | 提示信息 |
|:----:|---------|---------|
| 名称重复 | Toast | "该属性组下已存在同名属性值" |
| 被SKU引用→删除 | Modal警告 | "有N个SKU使用了该属性值，删除后将影响这些SKU" |

---

### 3.4 SPU新增（P0）

#### 功能说明
创建SPU商品定义，3步骤表单：选择分类→填写信息→预览SKU。规格属性自动笛卡尔积生成SKU组合。

#### 入口路径
商品中心 → SPU管理 → 新建SPU

#### 交互逻辑
1. 步骤一：选择商品分类（树形选择器，仅可选三级分类）
2. 步骤二：填写SPU名称/品牌/描述/主图 + 选择规格属性组
3. 步骤三：预览自动生成的SKU组合表格（规格值笛卡尔积）
4. 提交：校验必填项 → 保存SPU → 跳转SPU详情

#### 字段说明
| 字段 | 说明 | 组件 | 必填 |
|-----|------|------|:---:|
| 所属分类 | 三级分类节点 | 树选择 | Y |
| SPU名称 | 产品标准名称 | Input | Y |
| 品牌 | 商品品牌 | Input | N |
| 商品描述 | 富文本描述 | RichEditor | N |
| 主图 | 商品主图 | ImageUpload | N |
| 规格属性 | 可选规格属性组 | CheckboxGroup | N |

#### 业务规则
- 规格属性选择后提交时会自动笛卡尔积生成SKU
- 生成的SKU数>100个→提示"规格组合超过100个，请精简规格属性"
- 一个分类下SPU名称不可重复

---

### 3.5 SPU详情（P0）

#### 功能说明
查看SPU的完整信息，分卡片展示：基本信息+关联SKU列表+规格属性。

#### 入口路径
SPU列表 → 点击SPU项

#### 交互逻辑
1. 基本信息区：SPU名称/分类/品牌/描述/主图（只读）
2. SKU列表区：当前SPU下所有SKU的表格（编码/规格/价格/状态）
3. 规格属性区：展示选中的规格属性组和属性值
4. 操作按钮区：编辑/删除（根据状态动态展示）

---

### 3.6 SKU列表（P0）

#### 功能说明
查询和浏览所有SKU商品，支持多条件筛选。

#### 入口路径
商品中心 → SKU管理

#### 交互逻辑
1. 页面加载：获取SKU列表 → 渲染表格
2. 多条件筛选：分类/SPU名称/品牌/SKU编码/状态
3. 分页展示：每页20条
4. 点击SKU编码→跳转SKU详情

#### 字段说明
| 字段 | 说明 |
|-----|------|
| SKU编码 | 全局唯一编码（可点击跳转） |
| SPU名称 | 所属SPU |
| 规格描述 | 规格值组合 |
| 参考单价 | 平台设定价格 |
| 状态 | enabled/disabled |
| 关联供应商数 | 当前SKU的供应商数量 |

---

### 3.7 SKU详情（P0）

#### 功能说明
查看SKU的完整信息：基本信息卡片+规格属性标签+供应商供货列表。

#### 入口路径
SKU列表 → 点击SKU编码

#### 交互逻辑
1. 基本信息卡片：SKU编码/SPU名称/分类/参考单价
2. 规格属性标签：以标签形式展示规格值
3. 供应商列表Tab：展示当前SKU的所有关联供应商+供货价+状态
4. 操作按钮：编辑/管理供货关系

---

### 3.8 供应商供货列表（P0）

#### 功能说明
查看当前SKU下所有供应商的供货关系列表，支持筛选。

#### 入口路径
SKU详情 → 供应商列表Tab

#### 交互逻辑
1. 列表展示：供应商名称/供货价/审核状态/启用状态/操作按钮
2. 筛选项：按供应商名称/状态筛选
3. 操作：设置供货价/切换启用状态

---

### 3.9 设置供货价（P0）

#### 功能说明
设置/修改供应商的供货价格。

#### 交互逻辑
1. 弹窗编辑：输入供货价+备注
2. 价格校验：价格>0
3. 提交：记录价格变更历史 → 刷新列表

#### 异常处理
| 场景 | 提示信息 |
|:----:|---------|
| 价格≤0 | "供货价格必须大于0" |
| 重复提交 | "价格未变更" |

---

### 3.10 供货状态切换（P0）

#### 功能说明
启用/停用供应商的供货关系。

#### 交互逻辑
1. 点击开关 → Modal二次确认
2. 确认后更新状态 → 刷新列表
3. 停用后工程仓端不可见该商品

#### 业务规则
- 停用后工程仓端无法看到该商品
- 启用后实时可见
- 已有订单不受影响

---

### 3.11 SPU列表查询（P1）

#### 功能说明
按条件筛选查询SPU列表。

#### 交互逻辑
1. 树形分类筛选：左侧分类树选择 → 右侧展示该分类下SPU
2. 多条件搜索：品牌/状态/创建时间
3. 列表展示：SPU名称/分类/品牌/SKU数量/状态/创建时间

---

### 3.12 SPU编辑（P1）

#### 功能说明
修改SPU的基本信息（名称/品牌/描述/主图）。

#### 权限控制
仅平台运营/管理员可编辑。

---

### 3.13 SKU编辑（P1）

#### 功能说明
修改SKU的价格信息和基础属性。

#### 业务规则
- 价格修改记录变更日志（操作人/旧价格/新价格/时间）
- SKU编码不可修改

---

### 3.14 批量上下架（P1）

#### 功能说明
批量操作商品上下架。

#### 交互逻辑
1. 列表勾选多个SKU → 点击"批量上架"/"批量下架"按钮
2. Modal二次确认："确认对选中的N个商品执行上架/下架操作？"
3. 确认后显示进度条+结果汇总："成功N条，失败M条，失败原因：XXX"
