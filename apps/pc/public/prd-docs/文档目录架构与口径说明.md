# 产品文档目录架构与口径说明

> 本文档展示全套产品文档的目录结构和各模块口径定义。
> 帮助读者快速了解"哪里有啥、该看哪个"。

---

## 一、文档体系总览

### 核心分层

```
docs/                          ← 源文档仓库（产品经理维护）
├── 00_项目总览/               ← 项目级定稿文档
├── 01_PRD/                   ← 各端PRD（按端组织）
├── 02_系统操作手册/          ← 图文操作指南
├── 03_通用资源/              ← 模板/规范复用库
└── 04_历史归档/              ← 历史版本/废弃资料

apps/pc/public/prd-docs/       ← PRD查看器展示层（git同步）
├── prd.md                     ← 工程仓端PRD总纲
├── platform/                  ← 平台端PRD
├── supplier/                  ← 供应商端PRD
├── construction/              ← 施工方端PRD
├── 项目总览.md                ← 项目级文档
├── 跨部门协同*.md             ← 协同专题
└── archives/                  ← 归档同步
```

### 层次说明

| 层级 | 定位 | 维护频率 | 核心受众 |
|:----|:-----|:--------|:--------|
| 项目总览 | 项目级定稿文档 | 低频（架构变更时更新） | 全员 |
| PRD文档 | 功能需求详细设计 | 高频（每期迭代同步） | 产品/开发/测试 |
| 系统操作手册 | 面向终端的操作指南 | 中频（功能上线后补充） | 最终用户 |
| 历史归档 | 历史版本保留 | 不更新（只入不出） | 回溯查阅 |

## 二、目录架构

### 2.1 项目总览

```
项目总览/
├── 项目总览.md               ← 系统概述 + SaaS架构 + 交易链路 + 产品架构
├── 系统功能说明书.md         ← 各端功能清单对照
├── 版本迭代记录.md           ← 全版本发布总览
├── 跨部门协同PRD.md          ← 项目级跨系统协同总纲
└── 文档目录架构与口径说明.md  ← 当前文档
```

**口径说明：** 项目总览类是面向全员的顶层入口文档，内容相对稳定，仅在项目架构发生重大变更时更新。提供"先看概览、再读细节"的导航能力。

### 2.2 PRD文档

```
PRD文档/
├── 总更新日志                ← 全端PRD变更记录
├── 跨部门协同                ← 当前版本跨部门协同设计
├── 工程仓端/
│   ├── prd.md（文档总纲）    ← 51功能全景、设计原则、术语、角色权限
│   ├── 01-系统概览与架构     ← 工程仓端全景定位
│   ├── 02-业务流程设计       ← 双交易链路+核心流程
│   ├── 04-领域模型设计       ← DDD领域实体与事件
│   ├── 05~14-功能设计（共9个） ← 各模块功能详细设计
│   └── 14-页面导航设计       ← 页面索引与导航图
├── 平台端/
│   ├── prd.md（文档总纲）    ← 54功能全景、设计原则、角色权限
│   ├── 01-系统概览与架构     ← 平台端全景定位
│   ├── 02-业务流程设计       ← 审核/商品/订单监控流程
│   ├── 04-领域模型设计       ← DDD领域实体与事件
│   └── 05~13-功能设计（共9个） ← 各模块功能详细设计
├── 供应商端/
│   ├── prd.md（文档总纲）    ← 31功能全景、设计原则、角色权限
│   ├── 01-系统概览与架构     ← 供应商端全景定位
│   ├── 02-业务流程设计       ← 订单履约/商品审核流程
│   ├── 04-领域模型设计       ← DDD领域实体与事件
│   ├── 05~10-功能设计（共6个） ← 各模块功能详细设计
│   └── 11-操作手册           ← 供应商端操作指南
└── 施工方端/
    ├── prd.md（文档总纲）    ← 30功能全景、设计原则、角色权限
    ├── 01-系统概览与架构     ← 施工方端全景定位
    ├── 02-业务流程设计       ← 采购下单/售后流程
    ├── 04-领域模型设计       ← DDD领域实体与事件
    └── 05~10-功能设计（共6个） ← 各模块功能详细设计
```

**口径说明：** PRD文档是项目核心交付物，每个端独立成目录，每端内部按"总纲→概览→流程→模型→功能设计"顺序组织。功能设计文档采用统一的"功能设计"颗粒度描述，覆盖交互逻辑、业务规则、字段说明、状态治理。

#### 各端功能模块一览

| 端 | 功能模块 | 模块数 | 功能点数 |

| 平台端 | 工作台、商户管理、商品管理、商品市场管理、订单管理、库存管理、财务中心、系统设置 | 8个 | 54个 |
| 供应商端 | 工作台与商户中心、商品中心、订单管理、财务中心、系统设置 | 5个 | 31个 |
| 施工方端 | 首页与项目、商品市场、购物车、订单管理、库存与个人中心 | 5个 | 30个 |

### 2.3 系统操作手册

```
系统操作手册/
├── 平台端操作手册            ← PC管理后台图文操作指南
├── 供应商端操作手册          ← 供应商门户图文操作指南
└── 工程仓端操作手册          ← 工程仓小程序图文操作指南
```

**口径说明：** 操作手册面向最终用户，以系统真实截图+操作步骤为核心，与PRD功能设计互补。PRD讲"要做什么"，操作手册讲"怎么操作"。

### 2.4 历史归档

```
历史归档/
├── 2025 Q4 旧版详细设计      ← 2025年Q4及之前的详细设计文档
├── 2026 Q1 PM纯净版备份      ← 2026年Q1 PM纯净版备份（四端）
└── 旧资料归档                ← CSV功能清单/旧PRD/项目排期
```

**口径说明：** 历史归档为只读区域，所有资料来自旧版本文档体系，保留历史记录供回溯查阅。不在当前迭代范围内更新。

## 三、口径说明（怎么读这些文档）

### 3.1 阅读路径

| 阅读目标 | 推荐路径 |
|:--------|:--------|
| 快速了解项目全貌 | 项目总览 → 项目总览.md |
| 查看某端功能清单 | 项目总览 → 系统功能说明书.md |
| 了解某端设计细节 | 项目总览 → 对应端PRD → 文档总纲 → 具体功能设计 |
| 了解跨系统协同 | 项目总览 → 跨部门协同PRD（全局）/ PRD文档 → 跨部门协同（当前版本） |
| 学习怎么操作系统 | 系统操作手册 → 对应端手册 |
| 查找历史版本资料 | 历史归档 → 按时间查找 |

### 3.2 文件命名规范

- **文档总纲**：`prd.md` — 各端PRD的入口文件，含全景、设计原则、术语表
- **概览/流程/模型**：`01/02/04-*.md` — 项目级交叉文档，每端一份
- **功能设计**：`05~14-模块名功能设计.md` — 按模块组织，一个模块一个文件
- **页面导航**：`13/14-页面导航设计.md` — 各端页面索引与导航图

### 3.3 文档状态标识

| 标识 | 含义 |
|:----|:-----|
| ✅ 已有文档 | 已完成初稿，内容稳定 |
| ⚠️ 待完善 | 存在已知内容缺口 |
| 📦 历史归档 | 旧版本参考，不在当前迭代范围 |
