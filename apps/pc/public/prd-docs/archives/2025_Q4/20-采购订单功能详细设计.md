# 工程材料管理采供一体化平台 - 采购订单功能详细设计

> 版本：v1.0  
> 文档状态：评审中  
> 所属模块：采购管理 - 采购订单

---

## 一、功能概述

### 1.1 功能定位

采购订单管理是工程材料管理采供一体化平台的核心功能之一，为工程仓提供系统化、规范化的采购流程管理。采购订单连接平台商品市场与工程仓库存，实现从商品选择、订单创建、供应商确认、发货、收货到入库的全流程闭环管理。

### 1.2 核心概念

| 概念 | 说明 | 示例 |
|-----|------|------|
| 采购订单 | 工程仓向供应商发起的采购请求 | PO202401150001 |
| 平台订单号 | 平台生成的订单编号，用于平台内跟踪 | PL202401150001 |
| 发货单 | 供应商发货时生成的单据 | SH202401160001 |
| 入库单 | 工程仓收货时生成的单据 | IB202401160001 |
| 订单状态 | 采购订单的生命周期状态 | 待确认、已确认、发货中、已完成 |

### 1.3 目标用户

- **工程仓管理员**：创建采购订单、确认收货、入库管理
- **平台管理员**：查看所有采购订单、监督采购流程
- **供应商**：接收采购订单、确认发货

## 二、业务规则

### 2.1 采购列表展示规则

- **默认排序**：采购列表默认按创建时间倒序展示
  - 最新创建的订单显示在最前面
  - 支持按订单编号、订单金额、创建时间进行排序切换
- **分页展示**：当订单数量超过10条时采用分页展示
  - 每页显示10条订单
  - 支持跳转到指定页码
  - 显示总记录数和当前页码信息
- **状态标签**：不同状态的订单使用不同颜色的标签展示
  - 待确认：橙色标签
  - 已确认：蓝色标签
  - 发货中：绿色标签
  - 已完成：灰色标签
- **金额显示**：订单金额统一使用红色字体突出显示
  - 金额保留2位小数
  - 金额超过1000时使用千分位分隔符（如1,234.56）
- **搜索规则**：搜索功能支持模糊匹配
  - 按订单编号搜索：支持前缀匹配（如输入"PO2024"可匹配PO202401001）
  - 按供应商名称搜索：支持包含匹配（如输入"建材"可匹配"广东建材供应商"）
  - 搜索结果自动高亮匹配的关键词
- **标签页切换**：采购列表支持按订单状态进行标签页切换
  - 全部订单：显示所有状态的订单
  - 待确认：只显示待确认状态的订单
  - 已确认：只显示已确认状态的订单
  - 发货中：只显示发货中状态的订单
  - 已完成：只显示已完成状态的订单
  - 标签页右上角显示该状态的订单数量

### 2.2 列表操作权限规则

- **查看权限**：所有用户都可以查看采购列表
  - 平台管理员可以查看所有采购订单
  - 工程仓管理员只能查看本工程仓的采购订单
  - 供应商只能查看自己的采购订单
- **操作权限**：不同角色有不同的操作权限
  - 平台管理员：可以查看订单详情，不能进行确认、发货、收货、付款操作
  - 工程仓管理员：可以查看订单详情、确认收货、付款，不能进行确认、发货操作
  - 供应商：可以查看订单详情、确认订单、发货，不能进行收货、付款操作
- **按钮显示规则**：根据订单状态和用户角色动态显示操作按钮
  - 待确认状态：供应商显示"确认"按钮，工程仓和平台不显示
  - 已确认状态：供应商显示"发货"按钮，工程仓和平台不显示
  - 发货中状态：工程仓显示"入库"按钮，供应商和平台不显示
  - 已完成状态：所有角色都只显示"详情"按钮

### 2.3 列表数据加载规则

- **初始加载**：页面加载时默认加载第一页数据
  - 加载状态显示loading动画
  - 加载失败时显示"加载失败，请重试"提示
- **分页加载**：切换页码时异步加载对应页码的数据
  - 切换页码时保持当前的搜索条件和筛选条件
  - 页码切换时平滑过渡，避免页面闪烁
- **数据更新**：订单状态变更后自动刷新列表数据
  - 刷新后保持当前页码和筛选条件
  - 刷新后高亮显示刚更新的订单

### 2.4 订单基本信息规则

- **订单编号**：订单编号在系统中唯一
  - 格式：PO + 年月日 + 序号
  - 如"PO202401150001"
  - 系统自动生成
- **平台订单号**：平台订单号在系统中唯一
  - 格式：PL + 年月日 + 序号
  - 如"PL202401150001"
  - 系统自动生成
- **收货仓库**：必须选择收货仓库
  - 从工程仓的仓库列表中选择
  - 如"深圳湾科技园主仓"
- **期望交货日期**：必须设置期望交货日期
  - 不能早于当前日期
  - 如"2024-01-20"
- **收货地址**：必须填写收货地址
  - 最多支持500个字符
  - 如"广东省深圳市南山区科技园南区"
- **联系人**：必须填写联系人
  - 最多支持50个字符
  - 如"张三"
- **联系电话**：必须填写联系电话
  - 必须是有效的手机号码格式
  - 如"13800138000"
- **订单备注**：订单备注信息
  - 最多支持500个字符
  - 支持换行和空格

### 2.2 商品选择规则

- **商品数量**：每个商品的采购数量必须大于0
  - 不能超过供应商的库存数量
  - 如库存1000，采购数量不能超过1000
- **商品单价**：商品单价必须大于0
  - 从平台商品市场获取
  - 如"420元/袋"
- **商品金额**：商品金额自动计算
  - 金额 = 单价 × 数量
  - 保留2位小数
- **订单总金额**：订单总金额自动计算
  - 总金额 = 所有商品金额之和
  - 保留2位小数
- **运费**：运费由供应商承担
  - 默认运费为0
  - 如有特殊情况可手动调整

### 2.3 订单状态规则

- **待确认（pending）**：订单已创建，等待供应商确认
  - 工程仓创建订单后，订单状态为"待确认"
  - 供应商可以确认或拒绝订单
  - 工程仓可以取消订单
- **已确认（confirmed）**：供应商已确认订单，准备发货
  - 供应商确认订单后，订单状态为"已确认"
  - 供应商可以开始发货
  - 工程仓可以查看订单详情
- **发货中（shipping）**：供应商已发货，商品在运输途中
  - 供应商发货后，订单状态为"发货中"
  - 工程仓可以查看物流信息
  - 工程仓可以确认收货
- **已完成（completed）**：商品已收货入库，订单完成
  - 工程仓确认收货后，订单状态为"已完成"
  - 订单生命周期结束
  - 可以查看订单历史记录

### 2.4 入库状态规则

- **未入库（none）**：订单商品尚未入库
  - 订单创建后，入库状态为"未入库"
  - 供应商尚未发货
- **部分入库（partial）**：部分商品已入库，部分未入库
  - 部分商品已收货入库
  - 部分商品尚未发货或尚未收货
- **已入库（completed）**：所有商品已入库
  - 所有商品都已收货入库
  - 订单入库流程完成

### 2.5 付款状态规则

- **未付款（unpaid）**：订单尚未付款
  - 订单创建后，付款状态为"未付款"
  - 工程仓可以进行付款操作
- **部分付款（partial）**：部分金额已付款
  - 部分金额已付款
  - 部分金额尚未付款
- **已付款（paid）**：订单全额已付款
  - 订单全额已付款
  - 付款流程完成

### 2.6 删除约束规则

- **订单删除**：删除订单前必须检查是否有关联数据
  - 检查是否有发货单：有发货单的订单不能直接删除，需先删除发货单
  - 检查是否有入库单：有入库单的订单不能直接删除，需先删除入库单
  - 检查是否有付款记录：有付款记录的订单不能直接删除
- **删除提示**：删除时根据校验结果给出明确的提示信息
  - 有发货单时提示："该订单下有发货单，请先删除发货单"
  - 有入库单时提示："该订单下有入库单，请先删除入库单"

## 三、功能点详细设计

### 3.1 采购订单列表展示

#### 功能说明
以列表形式展示采购订单，支持标签页切换和搜索

#### 交互逻辑
1. 页面加载时显示所有采购订单
2. 订单列表展示字段：
   - 订单编号：可点击查看详情
   - 平台订单号：平台生成的订单编号
   - 供应商：供应商名称
   - 商品数量：订单包含的商品种类数量
   - 订单金额：订单总金额，红色字体显示
   - 入库状态：未入库、部分入库、已入库
   - 付款状态：未付款、部分付款、已付款
   - 订单状态：待确认、已确认、发货中、已完成
   - 创建时间：订单创建时间
   - 操作按钮：详情、确认、入库、付款
3. 支持标签页切换：全部订单、待确认、已确认、发货中、已完成
4. 支持分页加载，当订单数量较多时采用分页方式

#### 字段说明
| 字段名称 | 字段说明 | 数据类型 | 展示方式 |
|---------|---------|---------|---------|
| 订单编号 | 订单唯一编号 | VARCHAR | 链接展示，可点击查看详情 |
| 平台订单号 | 平台生成的订单编号 | VARCHAR | 链接展示 |
| 供应商 | 供应商名称 | VARCHAR | 文本展示 |
| 商品数量 | 订单包含的商品种类数量 | INT | 文本展示，居中对齐 |
| 订单金额 | 订单总金额 | DECIMAL | 文本展示，右对齐，红色字体 |
| 入库状态 | 入库状态 | VARCHAR | 标签展示 |
| 付款状态 | 付款状态 | VARCHAR | 标签展示 |
| 订单状态 | 订单状态 | VARCHAR | 标签展示 |
| 创建时间 | 订单创建时间 | DATETIME | 文本展示，格式YYYY-MM-DD HH:mm:ss |

#### 界面元素
- 订单标签页：全部订单、待确认、已确认、发货中、已完成
- 订单列表表格
- 分页控件
- 操作按钮（详情、确认、入库、付款）

### 3.2 搜索采购订单

#### 功能说明
在搜索框输入关键词，支持模糊搜索采购订单

#### 交互逻辑
1. 搜索框位于页面顶部，默认显示占位文字"搜索订单编号/供应商名称"
2. 输入关键词后点击搜索按钮触发搜索
3. 支持按回车键触发搜索
4. 搜索结果处理：
   - 匹配的订单在列表中展示
   - 匹配的关键词高亮显示
   - 无匹配结果时显示"暂无搜索结果"提示
5. 清空搜索框时恢复订单列表的原始展示状态

#### 字段说明
| 字段名称 | 字段说明 | 匹配规则 |
|---------|---------|---------|
| 订单编号 | 按订单编号精确匹配或前缀匹配 | 完全匹配或前缀匹配 |
| 供应商名称 | 按供应商名称模糊匹配 | 包含关键词即匹配 |

#### 界面元素
- 搜索输入框
- 搜索图标
- 搜索结果提示（无结果时显示）

### 3.4 新建采购订单

#### 功能说明
通过三步流程创建新的采购订单，包括选择商品、确认订单、提交成功

#### 交互逻辑
1. 点击"新建采购订单"按钮跳转到新建采购订单页面
2. **第一步：选择商品**：
   - 页面显示平台商品市场
   - 支持搜索商品名称/编码
   - 支持按分类筛选商品（全部、水泥、钢材、砂石、混凝土）
   - 商品卡片展示：商品图片、名称、规格、供应商、价格、库存
   - 选择商品：设置采购数量，点击"添加"按钮
   - 已选商品列表：显示已选商品的名称、规格、供应商、单价、数量、小计
   - 支持移除已选商品
   - 支持修改已选商品的数量
   - 显示购物车汇总：商品种类数量、合计金额
   - 点击"下一步"按钮，校验是否有已选商品
3. **第二步：确认订单**：
   - 填写订单信息：
     - 收货仓库：选择收货仓库
     - 期望交货日期：选择期望交货日期
     - 收货地址：填写收货地址
     - 联系人：填写联系人
     - 联系电话：填写联系电话
     - 订单备注：填写订单备注
   - 确认商品明细：显示商品明细列表
   - 确认费用明细：商品金额、运费、订单总额
   - 点击"上一步"按钮，返回第一步
   - 点击"提交订单"按钮，校验表单完整性
4. **第三步：提交成功**：
   - 显示提交成功页面
   - 显示订单编号
   - 提供"查看订单列表"和"继续采购"按钮
5. 提交成功后跳转到订单列表页面，弹出成功提示："采购订单提交成功"
6. 点击"取消"按钮或返回按钮，取消创建，不保存数据

#### 字段说明
| 字段名称 | 字段说明 | 数据类型 | 约束 | 默认值 |
|---------|---------|---------|------|--------|
| 收货仓库 | 收货仓库ID | VARCHAR | 必填 | - |
| 期望交货日期 | 期望交货日期 | DATE | 必填 | - |
| 收货地址 | 收货地址 | VARCHAR(500) | 必填 | - |
| 联系人 | 联系人 | VARCHAR(50) | 必填 | - |
| 联系电话 | 联系电话 | VARCHAR(20) | 必填 | - |
| 订单备注 | 订单备注 | VARCHAR(500) | 可选 | - |

#### 界面元素
- 页面标题：新建采购订单
- 步骤导航：选择商品 / 确认订单 / 提交成功
- 商品搜索框
- 商品分类标签页
- 商品卡片列表
- 已选商品列表
- 购物车汇总
- 订单信息表单
- 商品明细列表
- 费用明细列表
- 步骤操作按钮：上一步、下一步、提交订单、取消
- 成功页面

### 3.5 查看采购订单详情

#### 功能说明
查看采购订单的详细信息，包括订单基本信息、商品信息、发货记录、入库记录、操作日志

#### 交互逻辑
1. 在订单列表的操作按钮中点击"详情"按钮
2. 跳转到订单详情页面
3. **订单基本信息**：
   - 订单编号
   - 供应商
   - 订单金额
   - 订单状态
   - 支付状态
   - 收货状态
   - 创建时间
   - 支付时间
4. **商品信息**：
   - 显示所有商品的列表
   - 商品名称、规格、单位、采购数量、已发货数量、已收货数量、待收货数量、单价、金额
5. **发货记录**：
   - 显示所有发货单的列表
   - 发货单号、发货时间、物流公司、物流单号、发货数量、收货状态、入库单号
   - 操作按钮：收货、详情
6. **入库记录**：
   - 显示所有入库单的列表
   - 入库单号、入库时间、关联发货单、入库类型、入库数量、入库状态、操作人
   - 操作按钮：详情
7. **操作日志**：
   - 显示订单的所有操作记录
   - 按时间倒序展示
8. 点击"确认收货"按钮：跳转到确认收货页面
9. 点击"物流跟踪"按钮：查看物流跟踪信息
10. 点击"返回"按钮：返回到订单列表页面

#### 字段说明
| 字段名称 | 字段说明 | 数据类型 | 展示方式 |
