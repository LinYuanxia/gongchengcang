# 工程材料管理采供一体化平台 - 商品属性功能详细设计

> 版本：v1.0  
> 文档状态：评审中  
> 所属模块：商品中心 - 商品属性

---

## 一、功能概述

### 1.1 功能定位

商品属性管理是平台商品体系的重要组成部分，负责定义商品的规格参数和特征属性，为商品的标准化管理和搜索筛选提供基础支撑。属性与分类关联，支持自定义属性和预设属性值，为商品的多样性提供灵活的配置能力。

### 1.2 目标用户

- 平台管理员
- 平台运营人员

---

## 二、业务规则

### 2.1 属性分类关联规则

- **分类关联**：属性必须关联到具体的商品分类
  - 一个属性只能关联到一个分类
  - 一个分类下可以有多个属性
- **级联继承**：子分类继承父分类的属性
  - 父分类的属性在子分类中自动显示
  - 子分类可以添加自己的专属属性
- **分类变更**：属性的分类可以修改，但修改后：
  - 原分类下的商品不再使用该属性（已有商品不受影响）
  - 新分类下的商品可以使用该属性

### 2.2 属性编码规则

- **编码规则**：使用分类编码+属性序号的方式
  - 例如：分类编码为001，属性编码为00101、00102、00103
- **编码唯一性**：属性编码在系统中全局唯一
- **编码自动生成**：新增属性时，系统自动根据分类和序号生成属性编码
- **编码不可修改**：属性编码一旦创建不可修改

### 2.3 属性输入类型规则

- **输入类型**：支持5种输入类型
  1. **文本**：单行文本输入，适用于短文本内容
  2. **多行文本**：多行文本输入，适用于长文本内容
  3. **单选**：单选按钮组，从预设值中选择一个
  4. **多选**：多选复选框，从预设值中选择多个
  5. **下拉选择**：下拉选择框，从预设值中选择一个
- **类型约束**：
  - 文本和多行文本类型不需要预设属性值
  - 单选、多选、下拉选择类型必须预设属性值
  - 输入类型创建后不可修改

### 2.4 属性值规则

- **属性值预设**：单选、多选、下拉选择类型必须预设属性值
  - 一个属性可以有多个属性值
  - 属性值按排序值排序显示
- **属性值唯一性**：同一属性下的属性值名称不能重复
- **属性值编码**：属性值编码建议使用 `属性编码_序号` 的形式
  - 例如：属性编码为00101，属性值编码为0010101、0010102
- **属性值启用/禁用**：属性值可以单独启用或禁用
  - 禁用的属性值在创建商品时不可选
  - 已使用该属性值的商品不受影响

### 2.5 必填规则

- **必填属性**：属性可以设置为必填或选填
  - 必填属性：创建商品时必须填写该属性
  - 选填属性：创建商品时可以不填写该属性
- **必填校验**：创建商品时校验必填属性是否填写
- **必填变更**：属性的必填状态可以修改
  - 修改为必填后，已有商品不受影响
  - 新创建商品时需要校验

### 2.6 状态约束规则

- **属性状态**：属性有两种状态
  - 启用：属性正常使用，创建商品时可以选择
  - 禁用：属性暂停使用，创建商品时不可选
- **状态级联**：
  - 父分类禁用时，该分类下的属性自动禁用
  - 父分类启用时，属性状态保持不变（需手动启用）
- **状态校验**：禁用的属性在创建商品时不可选，已使用该属性的商品不受影响

### 2.7 删除约束规则

- **删除校验**：删除属性前必须检查是否有关联数据
  - 检查是否有属性值：有属性值的属性不能直接删除，需先删除属性值
  - 检查是否被商品使用：被商品使用的属性不能直接删除，需先取消商品关联
- **删除提示**：删除时根据校验结果给出明确的提示信息
  - 有属性值时提示："该属性下有属性值，请先删除属性值"
  - 被商品使用时提示："该属性被商品使用，请先取消商品关联"
- **级联删除**：删除属性时自动删除其下的所有属性值

### 2.8 排序规则

- **属性排序**：同一分类下的属性支持排序
- **属性值排序**：同一属性下的属性值支持排序
- **排序方式**：
  - 手动拖拽排序：在列表中拖拽调整顺序
  - 手动输入排序值：在编辑页面输入排序值
- **排序值**：使用整数排序值，数值越小排序越靠前
- **排序重排**：拖拽排序后自动重新计算排序值

## 三、功能点详细设计

### 3.1 属性列表展示

#### 功能说明
以列表形式展示商品属性，支持按分类筛选和搜索

#### 交互逻辑
1. 页面顶部显示分类筛选器：
   - 下拉选择框，以树形结构展示分类
   - 默认选中第一个分类
   - 选择分类后刷新属性列表
2. 属性列表展示字段：
   - 属性名称（最多显示10字，超出显示省略号）
   - 属性编码（灰色小字）
   - 所属分类（显示分类名称）
   - 输入类型（文本/多行文本/单选/多选/下拉选择）
   - 是否必填（是/否标签）
   - 状态（启用/禁用标签）
   - 属性值数量（显示该属性下的属性值数量，可点击展开查看）
   - 操作按钮（编辑/删除/启用/禁用）
3. 点击属性值数量标签可展开/折叠该属性的属性值列表
4. 支持分页加载，当属性数量较多时采用分页方式

#### 字段说明
| 字段名称 | 字段说明 | 数据类型 | 展示方式 |
|---------|---------|---------|---------|
| 属性名称 | 属性的名称，最多20字 | VARCHAR | 主标题展示 |
| 属性编码 | 属性的唯一编码，按规则生成 | VARCHAR | 灰色小字展示 |
| 所属分类 | 属性关联的分类名称 | VARCHAR | 文本展示 |
| 输入类型 | 属性的输入类型 | TINYINT | 标签展示 |
| 是否必填 | 属性是否必填 | TINYINT | 标签展示 |
| 状态 | 属性状态（启用/禁用） | TINYINT | 标签展示 |
| 属性值数量 | 该属性下的属性值数量 | INT | 数字展示，可点击 |
| 排序 | 属性的排序值 | INT | 不展示，用于排序 |
| 创建时间 | 属性的创建时间 | DATETIME | 不展示 |
| 更新时间 | 属性的更新时间 | DATETIME | 不展示 |

#### 界面元素
- 分类筛选下拉框
- 属性列表容器
- 属性值展开/折叠按钮
- 操作按钮组
- 分页控件

### 3.2 搜索属性

#### 功能说明
在搜索框输入关键词，支持模糊搜索属性信息，搜索结果在列表中展示

#### 交互逻辑
1. 搜索框位于页面顶部，分类筛选器右侧
2. 搜索框默认显示占位文字"搜索属性名称或编码"
3. 输入关键词后实时触发搜索（支持防抖，延迟300ms）
4. 支持按回车键触发搜索
5. 搜索结果处理：
   - 匹配的属性在列表中展示
   - 匹配的关键词高亮显示
   - 无匹配结果时显示"暂无搜索结果"提示
6. 清空搜索框时恢复属性列表的原始展示状态
7. 搜索结果支持按分类进一步筛选

#### 字段说明
| 字段名称 | 字段说明 | 匹配规则 |
|---------|---------|---------|
| 属性名称 | 按属性名称模糊匹配 | 包含关键词即匹配 |
| 属性编码 | 按属性编码精确匹配或前缀匹配 | 完全匹配或前缀匹配 |

#### 界面元素
- 搜索输入框
- 搜索图标
- 清空按钮（输入内容后显示）
- 搜索结果提示（无结果时显示）

### 3.3 新增属性

#### 功能说明
通过弹窗表单填写属性信息，创建新的商品属性

#### 交互逻辑
1. 点击"新增属性"按钮弹出新增属性弹窗
2. 表单包含以下字段：
   - **属性名称**：
     - 必填项，带*号标识
     - 输入框，最多20个字符
     - 实时校验：输入时显示剩余字符数（如"10/20"）
     - 失焦时校验唯一性（同一分类下名称唯一）
   - **所属分类**：
     - 下拉选择框，必填项，带*号标识
     - 下拉框以树形结构展示分类
     - 选择分类后自动生成属性编码前缀
   - **输入类型**：
     - 单选按钮组，必填项，带*号标识
     - 选项：文本 / 多行文本 / 单选 / 多选 / 下拉选择
     - 选择后根据类型显示或隐藏属性值设置区域
   - **是否必填**：
     - 单选按钮组，必填项，带*号标识
     - 选项：是 / 否
     - 默认选中"否"
   - **排序**：
     - 输入框，默认值为当前分类下属性的最大排序值+1
     - 只能输入正整数
     - 不填则使用默认值
   - **状态**：
     - 单选按钮组，默认选中"启用"
     - 选项：启用 / 禁用
     - 禁用状态时给出提示："禁用后创建商品时不可选"
   - **属性值设置**：
     - 当输入类型为"单选"、"多选"、"下拉选择"时显示
     - 支持批量添加属性值
     - 每个属性值包含：属性值名称、属性值编码、排序
     - 属性值编码自动生成（属性编码_序号）
     - 支持添加/删除属性值行
3. 点击"确定"按钮：
   - 前端校验表单完整性
   - 调用后端接口创建属性
   - 系统自动生成属性编码（根据分类和序号）
   - 创建成功后关闭弹窗，刷新属性列表
   - 弹出成功提示："属性创建成功"
4. 点击"取消"按钮或弹窗外部关闭弹窗，表单数据不保存

#### 字段说明
| 字段名称 | 字段说明 | 数据类型 | 约束 | 默认值 |
|---------|---------|---------|------|--------|
| 属性名称 | 属性名称，最多20字 | VARCHAR(20) | 必填，同一分类下唯一 | - |
| 所属分类 | 关联的分类ID | BIGINT | 必填 | - |
| 输入类型 | 属性输入类型 | TINYINT | 必填 | - |
| 是否必填 | 属性是否必填 | TINYINT | 必填 | 0（否） |
| 排序 | 排序值，正整数 | INT | 可选 | 当前分类最大排序值+1 |
| 状态 | 属性状态 | TINYINT | 必填 | 1（启用） |
| 属性值 | 属性值列表，仅特定输入类型需要 | ARRAY | 可选 | 空 |

#### 界面元素
- 弹窗标题：新增属性
- 表单容器
- 属性名称输入框（带字符计数）
- 所属分类下拉树选择器
- 输入类型单选按钮组
- 是否必填单选按钮组
- 排序输入框
- 状态单选按钮组
- 属性值设置区域（条件显示）
- 添加属性值按钮
- 删除属性值按钮
- 确定按钮
- 取消按钮
- 表单验证提示

### 3.4 编辑属性

#### 功能说明
编辑已存在的属性信息，操作与新增属性一致

#### 交互逻辑
1. 在属性列表的操作按钮中点击"编辑"按钮
2. 弹出编辑属性弹窗，表单字段与新增属性一致
3. 表单自动填充当前属性的原有数据
4. 字段规则：
   - 属性名称：可修改，规则同新增
   - 所属分类：可修改，但有以下限制：
     - 修改后重新生成属性编码
     - 新分类下属性名称不能重复
   - 输入类型：不可修改（创建后输入类型不可修改）
   - 是否必填：可修改，规则同新增
   - 排序：可修改，规则同新增
   - 状态：可修改，规则同新增
   - 属性值：可修改（仅当输入类型需要属性值时）：
     - 可以添加新的属性值
     - 可以编辑现有的属性值
     - 可以删除未被商品使用的属性值
     - 被商品使用的属性值不能删除，只能禁用
5. 点击"确定"按钮：
   - 前端校验表单完整性
   - 调用后端接口更新属性
   - 更新成功后关闭弹窗，刷新属性列表
   - 弹出成功提示："属性更新成功"
6. 点击"取消"按钮或弹窗外部关闭弹窗，修改不保存

#### 字段说明
同新增属性字段说明，输入类型字段不可修改

#### 界面元素
- 弹窗标题：编辑属性
- 其他界面元素同新增属性
- 输入类型选择框禁用状态

---

### 3.5 删除属性

#### 功能说明
删除指定属性，删除前需要校验是否有关联数据

#### 交互逻辑
1. 在属性列表的操作按钮中点击"删除"按钮
2. 弹出确认删除对话框
3. 系统执行删除前校验：
   - **校验1：检查是否有属性值**
     - 如果有属性值，对话框显示警告信息："该属性下有 {数量} 个属性值，删除属性将同时删除所有属性值，确定要删除吗？"
     - 显示"确认"和"取消"按钮
   - **校验2：检查是否被商品使用**
     - 如果被商品使用，对话框显示警告信息："该属性被 {数量} 个商品使用，请先取消商品关联"
     - "确认"按钮置灰，不可点击
     - 显示"取消"按钮
   - **校验3：无关联数据**
     - 对话框显示确认信息："确定要删除属性「{属性名称}」吗？删除后不可恢复。"
     - 显示"确认"和"取消"按钮
4. 点击"确认"按钮：
   - 调用后端接口删除属性
   - 删除成功后关闭对话框，刷新属性列表
   - 弹出成功提示："属性删除成功"
5. 点击"取消"按钮或弹窗外部关闭对话框，不执行删除

#### 字段说明
无额外字段，主要是校验逻辑

#### 界面元素
- 确认删除对话框
- 警告信息展示区域
- 确认按钮（校验通过时可用）
- 取消按钮

---

### 3.6 属性值管理

#### 功能说明
管理属性的属性值，包括新增、编辑、删除、启用/禁用属性值

#### 交互逻辑
1. 在属性列表中点击属性值数量标签，展开属性值列表
2. 属性值列表展示字段：
   - 属性值名称
   - 属性值编码（灰色小字）
   - 排序
   - 状态（启用/禁用标签）
   - 操作按钮（编辑/删除/启用/禁用）
3. **新增属性值**：
   - 点击属性值列表上方的"添加属性值"按钮
   - 弹出新增属性值弹窗
   - 表单包含：属性值名称、排序、状态
   - 属性值编码自动生成（属性编码_序号）
   - 点击"确定"按钮保存
4. **编辑属性值**：
   - 点击属性值操作列的"编辑"按钮
   - 弹出编辑属性值弹窗
   - 可修改：属性值名称、排序、状态
   - 属性值编码不可修改
   - 点击"确定"按钮保存
5. **删除属性值**：
   - 点击属性值操作列的"删除"按钮
   - 弹出确认删除对话框
   - 校验是否被商品使用
   - 被商品使用的属性值不能删除，只能禁用
   - 未被使用的属性值可以删除
   - 点击"确认"按钮删除
6. **启用/禁用属性值**：
   - 点击属性值操作列的"启用/禁用"按钮
   - 弹出确认对话框
   - 点击"确认"按钮更新状态
   - 禁用的属性值在创建商品时不可选
7. **属性值排序**：
   - 支持拖拽排序
   - 拖拽后自动重新计算排序值

#### 字段说明
| 字段名称 | 字段说明 | 数据类型 | 约束 |
|---------|---------|---------|------|
| 属性值名称 | 属性值名称，最多20字 | VARCHAR(20) | 必填，同一属性下唯一 |
| 属性值编码 | 属性值编码，自动生成 | VARCHAR | 必填，全局唯一 |
| 排序 | 排序值，正整数 | INT | 可选 |
| 状态 | 属性值状态 | TINYINT | 必填 |

#### 界面元素
- 属性值列表容器
- 添加属性值按钮
- 编辑属性值弹窗
- 删除属性值确认对话框
- 启用/禁用确认对话框

### 3.7 属性启用/禁用

#### 功能说明
启用或禁用属性，控制属性在创建商品时是否可选

#### 交互逻辑
1. 在属性列表的操作按钮中点击"启用"或"禁用"按钮
2. 弹出确认对话框
   - 启用时显示："确定要启用属性「{属性名称}」吗？启用后创建商品时可以选择。"
   - 禁用时显示："确定要禁用属性「{属性名称}」吗？禁用后创建商品时不可选，已使用该属性的商品不受影响。"
3. 点击"确认"按钮：
   - 调用后端接口更新属性状态
   - 更新成功后关闭对话框，刷新属性列表
   - 弹出成功提示："属性状态更新成功"
4. 点击"取消"按钮或弹窗外部关闭对话框，不执行操作

#### 字段说明
| 字段名称 | 字段说明 | 数据类型 |
|---------|---------|---------|
| 状态 | 属性状态（启用/禁用） | TINYINT |

### 3.8 属性拖拽排序

#### 功能说明
在属性列表中支持拖拽调整属性的显示顺序

#### 交互逻辑
1. 属性列表支持拖拽操作
2. 拖拽规则：
   - 只能在同一分类下的属性之间拖拽排序
   - 不能拖拽到其他分类下（如需修改分类请使用编辑功能）
3. 拖拽视觉效果：
   - 拖拽时显示拖拽占位符
   - 拖拽节点半透明显示
   - 可放置区域高亮显示
4. 拖拽完成后：
   - 系统自动重新计算所有兄弟节点的排序值
   - 调用后端接口批量更新排序值
   - 刷新属性列表
   - 弹出成功提示："排序更新成功"
5. 支持取消拖拽：按ESC键或拖拽到无效区域取消拖拽操作

#### 字段说明
| 字段名称 | 字段说明 | 数据类型 |
|---------|---------|---------|
| 排序 | 属性的排序值 | INT |

#### 界面元素
- 拖拽占位符
- 可放置区域高亮
- 拖拽节点半透明效果

## 四、数据校验规则

### 4.1 属性名称校验
- 必填校验：属性名称不能为空
- 长度校验：最多20个字符
- 唯一性校验：同一分类下的属性名称不能重复
- 特殊字符校验：不能包含特殊字符（支持中文、英文、数字、下划线、横线）

### 4.2 所属分类校验
- 必填校验：所属分类不能为空
- 存在性校验：选择的分类必须存在且启用

### 4.3 输入类型校验
- 必填校验：输入类型不能为空
- 不可修改：输入类型创建后不可修改

### 4.4 属性值校验
- 必填校验：输入类型为单选、多选、下拉选择时，至少需要一个属性值
- 唯一性校验：同一属性下的属性值名称不能重复
- 长度校验：属性值名称最多20个字符

### 4.5 删除校验
- 属性值校验：有属性值的属性删除时需要二次确认
- 商品关联校验：被商品使用的属性不能删除
- 属性值删除校验：被商品使用的属性值不能删除，只能禁用

### 5.1 表单验证异常
- 实时显示验证错误信息

## 八、接口设计

### 8.1 获取属性列表
**接口路径**：`GET /api/product/attribute/list`  
**接口说明**：获取属性列表  
**请求参数**：
- `categoryId`：分类ID（可选，不传则查询所有）
- `keyword`：搜索关键词（可选）
- `pageNum`：页码（可选）
- `pageSize`：每页数量（可选）
**响应数据**：属性列表数据

### 8.2 搜索属性
**接口路径**：`GET /api/product/attribute/search`  
**接口说明**：搜索属性  
**请求参数**：
- `keyword`：搜索关键词（必填）
- `categoryId`：分类ID（可选）
**响应数据**：匹配的属性列表

### 8.3 新增属性
**接口路径**：`POST /api/product/attribute`  
**接口说明**：新增属性  
**请求参数**：
- `attributeName`：属性名称（必填）
- `categoryId`：所属分类ID（必填）
- `inputType`：输入类型（必填）
- `isRequired`：是否必填（必填）
- `sort`：排序值（可选）
- `status`：状态（必填）
- `attributeValues`：属性值列表（可选，仅特定输入类型需要）
**响应数据**：新增的属性信息

### 8.4 编辑属性
**接口路径**：`PUT /api/product/attribute/{id}`  
**接口说明**：编辑属性  
**请求参数**：
- `attributeName`：属性名称（必填）
- `categoryId`：所属分类ID（必填）
- `isRequired`：是否必填（必填）
- `sort`：排序值（可选）
- `status`：状态（必填）
- `attributeValues`：属性值列表（可选）
**响应数据**：更新后的属性信息

### 8.5 删除属性
**接口路径**：`DELETE /api/product/attribute/{id}`  
**接口说明**：删除属性  
**请求参数**：无  
**响应数据**：删除结果

### 8.6 更新属性状态
**接口路径**：`PUT /api/product/attribute/{id}/status`  
**接口说明**：更新属性状态  
**请求参数**：
- `status`：状态（必填）
**响应数据**：更新结果

### 8.7 更新排序
**接口路径**：`PUT /api/product/attribute/sort`  
**接口说明**：批量更新属性排序  
**请求参数**：
- `sorts`：排序信息数组，包含属性ID和排序值（必填）
**响应数据**：更新结果

### 8.8 检查删除条件
**接口路径**：`GET /api/product/attribute/{id}/check-delete`  
**接口说明**：检查属性是否可以删除  
**请求参数**：无  
**响应数据**：
- `canDelete`：是否可以删除（true/false）
- `hasValues`：是否有属性值
- `hasProducts`：是否被商品使用
- `productCount`：使用该属性的商品数量

### 8.9 获取属性值列表
**接口路径**：`GET /api/product/attribute/{attributeId}/values`  
**接口说明**：获取属性值列表  
**请求参数**：无  
**响应数据**：属性值列表

### 8.10 新增属性值
**接口路径**：`POST /api/product/attribute/{attributeId}/value`  
**接口说明**：新增属性值  
**请求参数**：
- `valueName`：属性值名称（必填）
- `sort`：排序值（可选）
- `status`：状态（必填）
**响应数据**：新增的属性值信息

### 8.11 编辑属性值
**接口路径**：`PUT /api/product/attribute/value/{id}`  
**接口说明**：编辑属性值  
**请求参数**：
- `valueName`：属性值名称（必填）
- `sort`：排序值（可选）
- `status`：状态（必填）
**响应数据**：更新后的属性值信息

### 8.12 删除属性值
**接口路径**：`DELETE /api/product/attribute/value/{id}`  
**接口说明**：删除属性值  
**请求参数**：无  
**响应数据**：删除结果

## 九、总结

本商品属性功能详细设计完整描述了商品属性管理的所有功能点，包括：

1. **属性列表展示** - 以列表形式展示属性，支持按分类筛选
2. **搜索属性** - 支持按名称或编码搜索，结果高亮展示
3. **新增属性** - 通过弹窗表单创建新属性，自动生成属性编码
4. **编辑属性** - 修改属性信息，操作与新增一致
5. **删除属性** - 删除前校验是否有属性值或被商品使用
6. **属性值管理** - 管理属性的属性值，包括增删改查和启用/禁用
7. **属性启用/禁用** - 控制属性在创建商品时是否可选
8. **属性拖拽排序** - 支持在同一分类下拖拽调整顺序
9. **查看属性使用情况** - 跳转到商品列表查看使用该属性的商品

每个功能点都包含了详细的功能说明、交互逻辑、字段说明和界面元素，同时明确了业务规则、数据校验规则、异常处理和权限控制，为商品属性功能的开发提供了完整的设计指导。
