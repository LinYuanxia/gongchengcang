## 一、功能概述

### 1.1 功能定位

商品SPU（Standard Product Unit，标准化产品单元）管理是平台商品体系的核心功能，负责商品的全生命周期管理。SPU是一组具有共同属性和特征的商品的集合，是商品SKU的上层概念。SPU管理涵盖商品的创建、编辑、审核、上下架、库存管理等完整业务流程，为平台的商品交易提供基础支撑。

### 1.2 核心概念

| 概念 | 说明 | 示例 |
|-----|------|------|
| SPU | 标准化产品单元，是一组具有共同属性的商品集合 | 普通硅酸盐水泥 |
| SKU | 库存保有单位，是具体的商品实例 | 普通硅酸盐水泥 32.5级 袋装 |
| 商品属性 | 商品的特征参数，如材质、规格、型号等 | 材质:硅酸盐水泥 |
| 规格属性 | 用于区分不同SKU的属性，如等级、包装 | 等级:32.5级，包装:袋装 |
| 商品状态 | SPU的生命周期状态 | 已上架、已下架（平台端）；草稿、待审核、审核通过、已上架、已下架、审核拒绝（供应商端） |

### 1.3 目标用户

- **平台端**：平台管理员、平台运营人员
- **供应商端**：供应商（管理自己的商品）
- **工程仓端**：工程仓（管理自己的商品）

## 二、业务规则

### 2.1 SPU编码规则

- **编码规则**：使用分类编码+序号的方式
  - 例如：分类编码为001，SPU编码为SPU001001、SPU001002
- **编码唯一性**：SPU编码在系统中全局唯一
- **编码自动生成**：新增SPU时，系统自动根据分类和序号生成SPU编码
- **编码不可修改**：SPU编码一旦创建不可修改

### 2.2 SPU状态流转规则

#### 2.2.1 平台端SPU状态

- **平台端SPU状态**：2种状态
  1. **已上架**：商品正常销售中
  2. **已下架**：商品暂停销售
- **状态流转**：
  - 已上架 → 已下架：商品下架
  - 已下架 → 已上架：商品重新上架
- **状态约束**：
  - 已上架状态的商品才能被搜索和购买
  - 已下架状态的商品不能被搜索和购买

#### 2.2.2 供应商/工程仓端SPU状态

- **供应商/工程仓端SPU状态**：6种状态
  1. **草稿**：商品信息未完善，暂存状态
  2. **待审核**：商品信息已完善，提交平台审核
  3. **审核通过**：平台审核通过，可以上架销售
  4. **已上架**：商品正常销售中
  5. **已下架**：商品暂停销售
  6. **审核拒绝**：平台审核拒绝，需要修改
- **状态流转**：
  - 草稿 → 待审核：提交审核
  - 待审核 → 审核通过：平台审核通过
  - 待审核 → 审核拒绝：平台审核拒绝
  - 审核拒绝 → 待审核：修改后重新提交
  - 审核通过 → 已上架：商品上架
  - 已上架 → 已下架：商品下架
  - 已下架 → 已上架：商品重新上架
- **状态约束**：
  - 草稿状态的商品不能提交订单
  - 已上架状态的商品才能被搜索和购买
  - 已下架状态的商品不能被搜索和购买

### 2.3 商品分类规则

- **分类必填**：SPU必须关联到具体的商品分类
  - 一个SPU只能关联到一个分类
  - 分类必须是末级分类（三级分类）
- **分类变更**：SPU的分类可以修改，但修改后：
  - 原分类的属性和规格不再适用
  - 新分类的属性和规格需要重新设置
  - 已有的SKU不受影响，但新增SKU使用新分类的规格

### 2.4 商品品牌规则

- **品牌可选**：SPU可以关联品牌，也可以不关联
  - 一个SPU只能关联一个品牌
  - 品牌必须是已启用的品牌
- **品牌变更**：SPU的品牌可以修改

### 2.5 商品单位规则

- **单位必填**：SPU必须设置计量单位
  - 如：吨、千克、立方米、平方米等
- **单位换算**：支持单位换算
  - 如：1吨 = 1000千克
  - 销售时可以按不同单位销售

### 2.6 商品价格规则

- **价格必填**：SPU必须设置销售价格
  - 销售价格必须大于0
- **价格类型**：支持多种价格类型
  1. **市场价格**：市场价，仅供参考
  2. **销售价格**：实际销售价格
  3. **成本价格**：成本价格，用于利润计算
- **价格精度**：价格保留2位小数
- **价格继承**：SKU价格默认继承SPU价格，可以单独修改

### 2.7 商品库存规则

- **库存汇总**：SPU库存是所有SKU库存的总和
  - SPU库存 = 所有SKU库存的总和
  - SKU库存增加时，SPU库存自动增加
  - SKU库存减少时，SPU库存自动减少
- **库存预警**：支持库存预警设置
  - 当库存低于预警值时，系统自动提醒
- **库存锁定**：下单时锁定库存
  - 下单成功后锁定库存
  - 订单取消后释放库存
  - 订单完成后扣减库存

### 2.8 商品图片规则

- **主图必填**：SPU必须设置主图
  - 主图用于列表展示
  - 建议尺寸：800×800像素
- **轮播图**：支持多张轮播图
  - 最多支持5张轮播图
  - 建议尺寸：800×800像素
- **详情图**：支持详情图
  - 用于商品详情页展示
  - 支持富文本编辑
- **图片格式**：支持JPG、PNG格式
- **图片大小**：单张图片不超过2MB

### 2.9 商品描述规则

- **商品简介**：简短的商品描述，用于列表展示
  - 最多100个字符
- **商品详情**：详细的商品描述，用于详情页展示
  - 支持富文本编辑
  - 支持图片、文字、表格等

### 2.10 删除约束规则

- **删除校验**：删除SPU前必须检查是否有关联数据
  - 检查是否有SKU：有SKU的SPU不能直接删除，需先删除SKU
  - 检查是否有订单：有订单的SPU不能删除
  - 检查是否有库存：有库存的SPU不能删除
- **删除提示**：删除时根据校验结果给出明确的提示信息
  - 有SKU时提示："该商品下有SKU，请先删除SKU"
  - 有订单时提示："该商品有订单记录，不能删除"
  - 有库存时提示："该商品有库存，不能删除"
- **逻辑删除**：SPU删除采用逻辑删除，不物理删除数据

## 三、功能点详细设计

### 3.1 商品SPU列表展示（平台端）

#### 功能说明
以列表形式展示商品SPU，支持筛选、搜索和分页

#### 交互逻辑
1. 页面顶部显示筛选条件：
   - 商品分类筛选：下拉选择框，树形结构
   - 商品状态筛选：多选标签（已上架、已下架）
   - 供应商筛选：下拉选择框
   - 时间范围筛选：创建时间、更新时间
2. 筛选条件右侧显示搜索框：
   - 支持按商品名称、商品编码搜索
   - 支持按回车键搜索
3. 商品SPU列表展示字段：
   - 商品主图（小图）
   - 商品名称（最多显示20字，超出显示省略号）
   - 商品编码（灰色小字）
   - 商品分类（显示分类名称）
   - 品牌（显示品牌名称，无品牌显示"-"）
   - 销售价格（显示价格区间，如"¥100.00 - ¥200.00"）
   - 库存总量（显示所有SKU库存总和）
   - SKU数量（显示SKU数量，可点击查看）
   - 商品状态（标签展示：已上架/已下架）
   - 创建时间（格式：YYYY-MM-DD HH:mm）
   - 操作按钮（查看/编辑/删除/上架/下架）
4. 支持分页加载，默认每页20条
5. 支持列表批量操作：
   - 批量上架
   - 批量下架
   - 批量删除

#### 字段说明
| 字段名称 | 字段说明 | 数据类型 | 展示方式 |
|---------|---------|---------|---------|
| 商品主图 | 商品主图URL | VARCHAR | 图片展示 |
| 商品名称 | 商品名称，最多100字 | VARCHAR | 主标题展示 |
| 商品编码 | SPU编码，自动生成 | VARCHAR | 灰色小字展示 |
| 商品分类 | 关联的分类名称 | VARCHAR | 文本展示 |
| 品牌 | 关联的品牌名称 | VARCHAR | 文本展示 |
| 销售价格 | SKU价格区间 | VARCHAR | 文本展示 |
| 库存总量 | 所有SKU库存总和 | INT | 数字展示 |
| SKU数量 | SKU的数量 | INT | 数字展示，可点击 |
| 商品状态 | 商品状态 | TINYINT | 标签展示 |
| 创建时间 | SPU创建时间 | DATETIME | 文本展示 |

#### 界面元素
- 筛选条件区域
- 搜索框
- 商品列表表格
- 批量操作按钮
- 分页控件

### 3.3 新增商品SPU

#### 功能说明
创建新的商品SPU，支持多步骤表单填写

#### 交互逻辑
1. 点击"新增商品"按钮，打开新增商品页面
2. 新增商品采用多步骤表单：
   - 第一步：基本信息（分类、品牌、名称、单位等）
   - 第二步：商品属性（选择分类关联的属性）
   - 第三步：规格属性（选择分类关联的规格，用于生成SKU）
   - 第四步：商品图片（主图、轮播图、详情图）
   - 第五步：价格库存（设置价格、库存等）
3. 每一步填写完成后，点击"下一步"按钮
4. 最后一步点击"保存"按钮，创建商品SPU
5. **供应商/工程仓端**：保存后商品状态为"草稿"，需要点击"提交审核"按钮提交平台审核
6. **平台端**：保存后商品状态默认为"已下架"，可以直接上架

#### 字段说明
| 字段名称 | 字段说明 | 数据类型 | 必填 | 验证规则 |
|---------|---------|---------|------|---------|
| 商品分类 | 关联的商品分类 | VARCHAR | 是 | 必须选择末级分类 |
| 商品品牌 | 关联的商品品牌 | VARCHAR | 否 | 必须选择已启用的品牌 |
| 商品名称 | 商品名称，最多100字 | VARCHAR | 是 | 不能为空，最多100字 |
| 商品单位 | 计量单位，如吨、千克等 | VARCHAR | 是 | 不能为空 |
| 商品简介 | 商品简介，最多100字 | VARCHAR | 否 | 最多100字 |
| 商品详情 | 商品详情，富文本 | TEXT | 否 | - |
| 商品属性 | 关联的商品属性 | JSON | 否 | 根据分类选择 |
| 规格属性 | 关联的规格属性 | JSON | 否 | 根据分类选择 |
| 商品主图 | 商品主图URL | VARCHAR | 是 | 不能为空 |
| 轮播图 | 轮播图URL列表 | JSON | 否 | 最多5张 |
| 市场价格 | 市场价格，最多10位，2位小数 | DECIMAL | 否 | 必须大于0 |
| 销售价格 | 销售价格，最多10位，2位小数 | DECIMAL | 是 | 必须大于0 |
| 成本价格 | 成本价格，最多10位，2位小数 | DECIMAL | 否 | 必须大于0 |
| 库存预警值 | 库存预警值 | INT | 否 | 必须大于等于0 |

#### 界面元素
- 步骤导航条
- 表单字段
- 下一步按钮
- 上一步按钮
- 保存按钮
- 取消按钮
- 提交审核按钮（供应商/工程仓端）

### 3.4 编辑商品SPU

#### 功能说明

   - 审核通过状态：修改后需要重新提交审核
   - 已上架状态：修改后需要重新提交审核
   - 已下架状态：修改后需要重新提交审核
   - 审核拒绝状态：修改后可以重新提交审核
6. 点击"保存"按钮，保存修改

#### 字段说明
同新增商品SPU

#### 界面元素
- 步骤导航条
- 表单字段（已填充当前值）
- 下一步按钮
- 上一步按钮
- 保存按钮
- 取消按钮

### 3.6 商品SPU审核（平台端）

#### 功能说明
审核供应商/工程仓提交的商品SPU

#### 交互逻辑
1. 进入商品审核页面，显示待审核的商品列表
2. 点击"审核"按钮，打开审核弹窗
3. 审核弹窗展示：
   - 商品基本信息
   - 商品图片
   - 商品属性
   - 规格属性
   - 价格信息
   - 库存信息
   - 审核意见输入框
   - 审核通过按钮
   - 审核拒绝按钮
4. 点击"审核通过"按钮，商品状态变为"审核通过"
5. 点击"审核拒绝"按钮，商品状态变为"审核拒绝"，需要填写审核意见

#### 字段说明
| 字段名称 | 字段说明 | 数据类型 | 必填 |
|---------|---------|---------|------|
| 审核意见 | 审核意见，最多500字 | VARCHAR | 是（审核拒绝时） |

#### 界面元素
- 待审核商品列表
- 审核弹窗
- 审核意见输入框
- 审核通过按钮
- 审核拒绝按钮

### 3.7 商品SPU上下架

#### 功能说明
控制商品SPU的销售状态

#### 交互逻辑
1. 在商品列表或详情页，点击"上架"或"下架"按钮
2. 确认操作：
   - 上架：将商品状态从"已下架"变为"已上架"
   - 下架：将商品状态从"已上架"变为"已下架"
3. 操作成功后，刷新列表状态
4. 支持批量上下架操作

#### 业务规则
- **平台端**：可以直接上下架商品
- **供应商/工程仓端**：
  - 商品必须是"审核通过"状态才能上架
  - 已上架的商品可以下架
  - 下架的商品可以重新上架

#### 界面元素
- 上架按钮
- 下架按钮
- 确认弹窗

### 3.8 删除商品SPU

#### 功能说明
删除商品SPU

#### 交互逻辑
1. 点击操作列的"删除"按钮，弹出确认删除弹窗
2. 确认删除后，执行删除校验：
   - 检查是否有SKU：有SKU的SPU不能直接删除，需先删除SKU
   - 检查是否有订单：有订单的SPU不能删除
   - 检查是否有库存：有库存的SPU不能删除
3. 校验通过后，执行逻辑删除
4. 删除成功后，刷新列表

#### 业务规则
- **平台端**：可以删除任何状态的商品（需满足删除约束）
- **供应商/工程仓端**：
  - 只能删除草稿、审核拒绝状态的商品
  - 其他状态的商品不能删除

#### 界面元素
- 删除按钮
- 确认删除弹窗
- 提示信息

### 3.9 复制商品SPU

#### 功能说明
复制已存在的商品SPU，快速创建新商品

#### 交互逻辑
1. 点击操作列的"复制"按钮，弹出确认复制弹窗
2. 确认复制后，创建新的商品SPU：
   - 复制原商品的所有信息（分类、品牌、名称、属性、规格、图片、价格等）
   - 商品名称自动添加"（副本）"后缀
   - 商品编码重新生成
   - 商品状态设置为"草稿"（供应商/工程仓端）或"已下架"（平台端）
3. 复制成功后，跳转到编辑页面，可以修改信息

#### 界面元素
- 复制按钮
- 确认复制弹窗

### 3.10 导入导出商品SPU

批量导入导出商品SPU数据

   - 显示导入结果（成功数量、失败数量、失败原因）
