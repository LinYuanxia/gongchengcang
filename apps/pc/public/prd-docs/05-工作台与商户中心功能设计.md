## 一、功能概述

### 1.1 功能定位

工作台是工程仓登录后的**首页仪表盘**，提供关键运营指标的概览看板，让主管和采购员第一时间了解今日订单状态、采购金额、待收货情况、库存预警等核心数据，辅助日常决策。

商户中心负责工程仓的**主体信息管理和合同管理**，包含企业资质展示编辑以及供应商/施工方合同查阅。

### 1.2 核心概念

| 概念 | 说明 | 示例 |
|:----|------|------|
| 工作台 | 登录后的首页仪表盘，展示运营指标 | 今日订单数/采购金额/库存预警 |
| 主体信息 | 工程仓的企业注册信息 | 企业名称/统一信用代码/营业执照 |
| 合同 | 工程仓与供应商/施工方签订的合作协议 | "深圳湾项目水泥采购框架协议" |
| 指标卡片 | 展示单个运营指标的数字卡片组件 | 待收货数=5（橙色预警） |

### 1.3 目标用户

- **采购员**（核心用户）：关注今日订单数和采购金额
- **仓管员**：关注待收货数和库存预警
- **主管**：全局了解整体运营状况，编辑主体信息
- **财务**：查看合同（关联付款条款）

### 1.4 模块范围

| 功能分类 | 主要功能 | 涉及角色 |
|:--------|---------|---------|
| 工作台-数据看板 | 运营指标概览、时间范围切换、卡片跳转 | 采购员、仓管员、主管 |
| 商户中心-主体信息查看 | 企业资质展示（只读） | 所有角色 |
| 商户中心-主体信息编辑 | 联系人/地址/营业执照修改 | 主管 |
| 商户中心-合同列表查看 | 合同查询、合同详情预览 | 主管、财务 |

## 二、核心设计原则

> **工作台遵循"轻量概览"原则，不做深度分析；商户中心遵循"RBAC角色权限"原则。**

### 2.1 数据轻量化原则

- 工作台只展示关键运营指标，不做明细数据下钻
- 每个指标卡片独立加载，互不影响（容错隔离）
- 默认展示"今日"数据，切换时间范围时重新请求

### 2.2 只读与编辑分离原则

- 商户中心的主体信息默认只读展示
- 只有主管角色可见「编辑」按钮
- 编辑模式下实时校验字段，保存后自动恢复只读状态

### 2.3 线下生意适配

- 联系电话自动脱敏展示（中间4位显示\*\*\*\*）
- 营业执照图片支持在线放大查阅，无需额外工具

## 三、业务规则

### 3.1 工作台数据展示规则

- **默认时间范围**：登录后默认展示"今日"数据
- **时间切换**：支持今日/本周/本月三个时间范围，切换时重新调取统计接口
- **卡片独立加载**：每个指标卡片单独请求，一个卡片超时不阻塞其他卡片
- **自动刷新**：页面打开后每5分钟自动刷新一次（可配置刷新间隔）
- **卡片跳转**：点击指标卡片中的数字可配置跳转到对应模块列表页
- **空数据**：指标为0时正常显示"0"，不做隐藏

### 3.2 商户中心展示规则

- **数据脱敏**：联系电话中间4位自动脱敏（138\*\*\*\*1234）
- **缺失处理**：不存在的信息显示"-"代替空处理
- **编辑权限**：仅主管角色可见「编辑」按钮，其他角色看不到编辑入口
- **编辑时效**：编辑后立即生效，操作记录写日志（谁+什么时间+改了哪些字段）

### 3.3 商户中心编辑规则

- **实时校验**：输入即校验，错误时输入框边框变红+错误提示
- **图片限制**：营业执照支持替换上传，仅限 jpg/png 格式，单张≤5MB
- **防重复提交**：保存按钮提交时显示 loading，禁止重复点击

| 合同列表 | 响应时间 | ≤ 500ms | 含PDF元数据 |

### 5.2 埋点需求

| 页面 | 事件名 | 触发时机 | 上报字段 |

## 六、功能点详细设计

### 6.1 工作台-数据概览看板（P0）

#### 交互逻辑

1. 页面加载：默认选中"今日"时间范围 → 并行调用6个统计接口 → 渲染指标卡片网格
2. 切换时间：点击今日/本周/本月 → 时间参数变更 → 重新调用所有统计接口 → 卡片数据更新（骨架屏过渡）
3. 点击卡片数字：根据卡片类型跳转对应模块（如点击待收货数→跳转入库列表）
4. 自动刷新：页面激活状态下每5分钟自动刷新全部卡片（页面失焦时暂停轮询）

#### 原子字段定义

| 字段 | 类型 | 必填 | 来源 | 校验规则 | 展示规则 | 默认值 |
|:----|:----|:----:|:----|:--------|:--------|:-----:|
| 时间范围 | Enum(今日/本周/本月) | 否 | 前端选择 | 三选一 | Radio按钮组 | 今日 |
| 今日订单数 | Integer | 否 | 统计接口 | ≥0 | 数字卡片 | 加载中... |
| 采购总金额 | Decimal(12,2) | 否 | 统计接口 | ≥0 | 数字+单位"元" | 加载中... |
| 待收货数 | Integer | 否 | 统计接口 | ≥0 | >0时橙色背景预警 | 加载中... |
| 库存预警数 | Integer | 否 | 统计接口 | ≥0 | >0时红色背景预警 | 加载中... |
| 待处理销售单 | Integer | 否 | 统计接口 | ≥0 | 数字卡片 | 加载中... |
| 上次更新时间 | DateTime | 否 | 前端记录 | - | 底部"上次更新：HH:mm:ss" | - |

#### 边界情况覆盖

| 场景 | 处理逻辑 | 提示文案 |
|:----|:--------|---------|
| 单个统计接口超时 | 对应卡片显示"--"，其他卡片正常 | 不提示，卡片本身显示"--" |
| 全部接口失败 | 显示整页重试按钮 | "数据加载失败，请重试" |
| 时间切换后接口失败 | 保持上一次数据不变，新数据不更新 | Toast提示"数据更新失败" |
| 所有指标为0 | 正常显示0 | - |
| 大面积预警 | 预警卡片标红/标橙，其他卡片正常 | - |

### 6.2 商户中心-主体信息查看（P1）

#### 交互逻辑

1. 页面加载：调用主体信息接口 → 渲染信息卡片（只读）
2. 所有字段以两列布局展示（字段名+字段值）
3. 联系电话自动脱敏展示（中间4位\*\*\*\*）
4. 主管身份下显示「编辑」按钮
5. 营业执照缩略图可点击放大查看

#### 原子字段定义

| 字段 | 类型 | 必填 | 来源 | 校验规则 | 展示规则 | 默认值 |
|:----|:----|:----:|:----|:--------|:--------|:-----:|
| 主体名称 | String(100) | 是 | 商户接口 | 非空 | 左对齐主标题 | - |
| 统一信用代码 | String(18) | 是 | 商户接口 | 18位 | 等宽字体 | - |
| 法人代表 | String(30) | 是 | 商户接口 | 非空 | 文本 | - |
| 联系人 | String(20) | 否 | 商户接口 | - | 文本 | - |
| 联系电话 | String(11) | 否 | 商户接口 | 11位手机号 | 脱敏：138****1234 | - |
| 详细地址 | String(100) | 否 | 商户接口 | - | 文本 | - |
| 营业执照 | URL | 否 | 商户接口 | URL格式 | 缩略图，可放大 | - |

#### 边界情况覆盖

| 场景 | 处理逻辑 | 提示文案 |
|:----|:--------|---------|
| 信息不存在 | 对应字段显示"-" | - |
| 图片加载失败 | 显示默认占位图 | - |
| 接口超时 | 整体重试按钮 | "信息加载失败，请重试" |

### 6.3 商户中心-主体信息编辑（P1）

#### 交互逻辑

1. 前置条件：当前用户=主管
2. 点击编辑按钮 → 切换为编辑模式/弹窗
3. 联系人/联系电话/详细地址输入即校验
4. 营业执照支持替换上传（删除旧图→上传新图）
5. 保存 → 调用更新接口 → 记录修改日志 → 恢复只读状态
6. 取消 → 恢复原始数据，退出编辑模式

#### 原子字段定义

| 字段 | 类型 | 必填 | 来源 | 校验规则 | 展示规则 | 默认值 |
|:----|:----|:----:|:----|:--------|:--------|:-----:|
| 联系人 | String(20) | 否 | 前端输入 | 2-20个字符 | Input框 | 原始值 |
| 联系电话 | String(11) | 否 | 前端输入 | ^1[3-9]\d{9}$ | Input框+格式校验 | 原始值 |
| 详细地址 | String(100) | 否 | 前端输入 | 最大100字 | Input框 | 原始值 |
| 营业执照 | File | 否 | 前端文件 | jpg/png ≤5MB | ImageUpload+缩略图 | 原始图片 |

#### 边界情况覆盖

| 场景 | 处理逻辑 | 提示文案 |
|:----|:--------|---------|
| 联系电话格式错误 | 输入框边框变红，阻止提交 | "请输入正确的手机号" |
| 营业执照文件过大 | 前端拦截不上传 | "营业执照图片不能超过5M" |
| 保存失败 | Toast提示，编辑内容保留 | "保存失败，请重试" |
| 编辑时他人修改了数据 | 提交时版本号冲突提示 | "信息已被其他人修改，请刷新后重试" |

### 6.4 商户中心-合同列表查看（P2）

#### 交互逻辑

1. 页面加载：默认选中"全部"Tab → 调用合同列表接口 → 渲染表格
2. 状态Tab切换：生效/已过期 → 按状态筛选
3. 时间范围筛选：选择生效日期区间 → 与状态叠加查询
4. 点击合同编号/名称 → 查看合同详情（PDF预览/下载）

#### 原子字段定义

| 字段 | 类型 | 必填 | 来源 | 校验规则 | 展示规则 | 默认值 |
|:----|:----|:----:|:----|:--------|:--------|:-----:|
| 合同编号 | String(32) | 是 | 合同接口 | 非空 | 超链接，可点击 | - |
| 合同名称 | String(100) | 是 | 合同接口 | 非空 | 文本超链接 | - |
| 签约方 | String(50) | 是 | 合同接口 | 非空 | 文本 | - |
| 生效日期 | Date | 是 | 合同接口 | 合法日期 | YYYY-MM-DD | - |
| 截止日期 | Date | 否 | 合同接口 | - | YYYY-MM-DD | - |
| 状态 | Enum | 是 | 合同接口 | 生效/已过期 | 标签（绿色/灰色） | - |

#### 边界情况覆盖

| 场景 | 处理逻辑 | 提示文案 |
|:----|:--------|---------|
| 合同列表为空 | 显示空状态 | "暂无合同信息" |
| 筛选无结果 | 显示空状态 | "未找到符合条件的合同" |
| 合同详情加载失败 | Toast提示 | "合同加载失败，请重试" |
| PDF不支持预览 | 提供下载按钮 | - |

## 七、异常处理汇总表

| 异常场景 | 触发条件 | 前端处理 | 后端处理 | 提示文案 |
|:--------|:--------|:--------|:--------|---------|
| 统计接口超时 | 响应>3s | 该卡片显示"--" | 无特殊处理 | 无提示（卡片直接显示"数据异常"） |
| 全部统计失败 | 所有卡片都失败 | 整页重试按钮 | 无特殊处理 | "数据加载失败，请重试" |
| 主体信息加载失败 | 接口返回错误 | 重试按钮 | 无特殊处理 | "信息加载失败，请重试" |
| 编辑-保存失败 | 网络异常/后端错误 | Toast提示 | 不更新数据 | "保存失败，请重试" |
| 编辑-版本冲突 | 版本号不匹配 | Toast提示，刷新数据 | 返回版本冲突 | "信息已被其他人修改，请刷新后重试" |
| 合同加载失败 | PDF加载异常 | 提供下载按钮 | - | "合同加载失败" |
