# 全链路产品需求生成器

## 技能概述

标准化、可复制、全链路的产品需求生成脚手架，让任何项目都能快速产出专业级需求文档！

## 🎯 何时调用

**自动触发场景：**
- 用户说"帮我写需求"
- 用户说"生成PRD"
- 用户说"做产品设计"
- 用户说"需求清单"
- 任何项目开始前需要梳理需求

## 📋 标准16列输出格式

| 列名 | 说明 | 示例值 |
|------|------|--------|
| 功能编码 | 唯一ID，待分配Jira号 | `待分配` |
| Jira链接 | 任务链接 | 空列，后续填充 |
| 开发状态 | 生命周期 | `待开发` |
| 预估工时 | 开发人天 | 空列 |
| 开发负责人 | 责任人 | 空列 |
| 测试负责人 | 测试人 | 空列 |
| 所属端 | 平台端/工程仓端/供应商端/施工方端/C端/管理后台 | |
| 模块 | 所属大模块 | 商户/商品/订单/库存/财务/系统 |
| 一级菜单 | 左侧导航 | |
| 优先级 | P0/P1/P2/P3 | P0=核心MVP |
| 二级菜单 | 子菜单 | |
| 功能点 | 原子级功能名称 | 商户列表查询 |
| 功能说明 | 一句话描述 | |
| 适用角色 | 谁能用这个功能 | 平台管理员/运营/财务 |
| 前置条件 | 什么条件才能用 | 登录成功+有权限 |
| 输入字段 | 每个参数+校验规则 | 商户名称(必填,2-50字) |
| 输出字段 | 返回什么数据 | 分页列表+总条数 |
| 交互说明 | 具体UI/交互 | 点击弹窗打开,实时校验 |
| 后置条件 | 执行后发生什么 | 更新状态+发通知+记日志 |
| 异常场景 | 每种错误的提示 | 名称重复→提示"已存在" |
| 业务场景逻辑 | 为什么做这个 | 入驻商户审批流程 |
| 数据流转 | 前端→API→DB | 表单→校验→入库→通知 |
| 页面元素 | 按钮/弹窗/表单项 | 审核按钮/二次确认弹窗 |
| 依赖外部接口 | 后端接口名 | 获取商户列表接口 |

## 🚀 使用方法

### 第一步：告诉AI
```
帮我生成一个【XXX项目】的全链路需求，包含：
1. 端：平台端/供应商端/C端
2. 核心模块：用户/商品/订单
3. 侧重：哪些功能是P0核心
```

### 第二步：AI自动输出
```
✅ 已生成：全链路需求_你的项目.csv
```

### 第三步：直接用
1. Excel打开CSV
2. 评审调整优先级和细节
3. 开发前批量导入Jira

## 💡 标准优先级定义

| 级别 | 标准 |
|------|------|
| **P0** | 没有这个功能，整个业务跑不通 |
| **P1** | 一期必须完成，影响核心体验 |
| **P2** | 二期优化，不影响主流程 |
| **P3** | 后续迭代，锦上添花 |
