# PRD查看器功能手册

> 版本：v2.0 | 更新日期：2026-04-27
> 本文档描述PRD查看器的全部功能及使用方法。

---

## 一、查看器核心能力

PRD查看器是产品团队的一站式文档管理平台，具备以下三大核心能力：

| 能力 | 说明 |
|:-----|:-----|
| 📖 **沉浸式阅读** | 左侧目录导航 + 右侧大纲索引，快速定位 |
| ✏️ **在线编辑** | 浏览器内直接改文档，保存自动提交Git |
| 🐦 **一键同步语雀** | 单文档/全量文档一键同步到语雀知识库 |

---

## 二、沉浸式阅读

### 2.1 左侧树形目录

- ✅ **点击文字展开/收起**：不用精确点小箭头，点标题就能切换
- ✅ **层级缩进清晰**：分组/文档层级视觉区分明确
- ✅ **默认全展开**：打开就能看到完整文档结构

### 2.2 右侧大纲索引

- ✅ **零边框清爽设计**：无左侧边框、无标题下边框、无滚动条
- ✅ **圆角高亮激活**：只用背景色区分当前位置
- ✅ **点击跳转**：点目录项直接滚动到对应标题

### 2.3 内容排版

- ✅ **内容居中展示**：最大宽度960px，阅读舒适
- ✅ **Mermaid图表支持**：自动渲染流程图、架构图、甘特图
- ✅ **图表放大**：点击图表可弹窗放大查看

## 三、在线编辑功能

### 3.1 功能入口

每个文档右上角有 **「✏️ 编辑文档」** 按钮

### 3.2 使用流程

```
1. 打开任意PRD文档 → 点右上角「编辑文档」
2. 在编辑器中修改Markdown内容
3. 点「💾 保存」→ 自动：
   ✅ 写入本地文件
   ✅ Git commit
   ✅ Git push origin main
4. 页面自动刷新，显示修改后内容
```

### 3.3 技术实现

```
前端点保存 → /api/save-prd-doc（Vite中间件）
                      ↓
        1. 写本地 public/prd-docs/xxx.md
        2. 执行 git add / commit / push
        3. 返回成功 → 前端重新渲染
```

## 四、一键同步到语雀

### 4.1 功能入口

每个文档右上角有 **「🐦 同步到语雀」** 按钮

### 4.2 两种同步模式

| 模式 | 适用场景 |
|:-----|:---------|
| 📄 **仅同步当前文档** | 修改了单个文档，增量更新 |
| 🌲 **同步全部文档** | 首次初始化知识库 |

### 4.3 准备工作

1. **创建语雀知识库**
   - 新建一个空的知识库
   - 获得知识库ID：`https://www.yuque.com/用户名/知识库ID`

2. **申请个人Token**
   - 访问：https://www.yuque.com/settings/tokens
   - 勾选 `doc.write` 权限
   - 生成Token并保存

### 4.4 使用流程

```
1. 点右上角「同步到语雀」
2. 选择同步模式
3. 填入 Token 和 知识库ID
4. 点「开始同步」
5. 等待同步完成：
   ✅ 实时进度条
   ✅ 每个文档的成功/失败状态
   ✅ 每个文档的语雀直达链接
```

### 4.5 语雀后续整理

同步完成后，在语雀中手动创建目录结构：

```
📌 项目总览
   ├── 项目总览
   ├── 系统功能说明书
   ├── 版本迭代记录
   ├── 跨部门协同PRD
   └── 文档目录架构与口径说明

📋 PRD文档 - 26.4.1期
   ├── 版本概览
   ├── 跨部门协同
   ├── 更新日志
   ├── 工程仓端
   ├── 资源平台端
   ├── ...

📓 系统操作手册
   ├── ...
```

## 五、部署说明

### 5.1 本地开发

```bash
pnpm dev
```

启动后访问：http://localhost:8000/gongchengcang/#/warehouse/prd

### 5.2 功能开关

所有功能开箱即用，无需额外配置：

- ✅ 在线编辑：Vite中间件自动启用
- ✅ 语雀同步：Vite代理自动解决跨域
- ✅ 配置自动保存：Token存在localStorage
