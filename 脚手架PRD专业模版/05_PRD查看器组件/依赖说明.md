# PRD查看器组件 — 依赖说明

> 以下为运行 PRD 查看器所需的 npm 包和配置项。
> 适用于 Vue 3 + TypeScript + Vite 项目。

---

## 核心依赖

| 包名 | 版本参考 | 用途 |
|:----|:--------|:-----|
| `marked` | ^12.x | 将 Markdown 渲染为 HTML |
| `mermaid` | ^10.x | 渲染 Mermaid 流程图 |
| `@arco-design/web-vue` | ^2.x | UI 组件库（Drawer/Select/Modal/Steps等） |
| `vue-router` | ^4.x | 路由匹配（PrdPanel需要） |
| `highlight.js` | ^11.x | （可选）代码块语法高亮 |

## 安装命令

```bash
npm install marked mermaid @arco-design/web-vue
# 或
pnpm add marked mermaid @arco-design/web-vue
```

## Vite 配置

### 1. PRD 文档静态资源目录

在 `public/` 下创建 `prd-docs/` 目录，将生成的 PRD 文档（md文件）放入：

```
public/prd-docs/
├── 项目总览.md
├── 系统功能说明书.md
├── 版本迭代记录.md
├── 更新日志.md
├── {端名}/
│   ├── prd.md
│   ├── 01-系统概览与架构.md
│   └── ...
└── {端名}/...
```

### 2. base 路径配置

如果项目部署在子路径下（如 `/gongchengcang/`），需在 `vite.config.ts` 中设置：

```typescript
export default defineConfig({
  base: process.env.VITE_BASE_URL || '/{你的base路径}/',
})
```

同时在 `prdData.ts` 和 `PrdPageViewer.vue` 中将 `{basePath}` 替换为相同的路径。

### 3. Arco Design 按需加载（推荐）

```typescript
// vite.config.ts
import { vitePluginForArco } from '@arco-plugins/vite-vue'

export default defineConfig({
  plugins: [
    vue(),
    vitePluginForArco({ style: 'css' }),
  ],
})
```

## 集成步骤

| 步骤 | 操作 |
|:----|:-----|
| 1 | 安装依赖 |
| 2 | 复制组件文件到项目 `src/components/PrdPageViewer/` |
| 3 | 修改 `prdData.ts` 中的 `{basePath}` 和模块列表 |
| 4 | 在路由中注册全页查看器（如 `/prd`） |
| 5 | `public/prd-docs/` 下放入实际的 PRD 文档 |
| 6 | 验证：访问 `/prd` 查看模块列表是否正常加载 |
