## 一、项目背景与目标

### 1.1 业务背景

工程仓平台目前的材料采购存在以下痛点：
- 各工程仓独立找供应商，采购分散议价能力弱
- 同品不同价，价格透明度低
- 施工材料规格不标准，质量参差不齐
- 施工方下单时材料选择成本高，漏买错买频发

### 1.2 项目目标

| 目标类型 | 具体指标 | 衡量方式 |
|---|---|---|
| **业务目标** | 平台统一商品覆盖率达到80% | 后台数据统计 |
| **业务目标** | 材料采购均价下降10% | 对比历史采购数据 |
| **效率目标** | 施工方下单时间缩短50% | BOM套餐占比提升 |
| **运营目标** | SKU标准化率达到90% | 分类商品标准化率 |

### 1.3 需求范围

✅ **包含**：
- 平台端供应商品市场管理
- 工程仓端销售商品市场
- BOM基装包管理
- 商品分类展示管理

❌ **暂不包含**：
- 实时库存对接
- 物流配送系统
- 支付结算系统
- 供应商入驻审核

## 二、用户角色与使用场景

### 2.1 用户角色矩阵

| 用户角色 | 核心诉求 | 使用频率 |
|---|---|---|
| **平台运营专员** | 商品标准化管控、价格监控 | 每日 |
| **工程仓采购员** | 快速找到合适价格的材料 | 每日 |
| **施工方工长** | 按项目快速配齐材料 | 每周 |
| **供应商业务员** | 商品报价与库存更新 | 每周 |

### 2.2 典型使用场景

#### 场景1：平台运营每日巡检
> 运营小王上班后打开供应商品市场，查看：
> 1. 待上架的新品审核
> 2. 供应商停货预警商品
> 3. 价格异常波动的SKU
> 4. 集中批量上架新入库的品牌商品

#### 场景2：工程仓紧急补料
> 采购员小李收到工地紧急需求："还差2吨32.5水泥"
> 1. 打开商品市场搜索"水泥"
> 2. 筛选有货的供应商
> 3. 选择价格最低的下单
> 4. 加入购物车结算

#### 场景3：新项目开工备料
> 张工长接到新家装项目：
> 1. 打开BOM基装包
> 2. 选择"水电标准包" + "木工基础包"
> 3. 调整可选材料规格
> 4. 一键生成订单

## 三、功能详情

---

### 模块1：供应商品市场（平台端）

#### 1.1 商品状态看板

| 功能点 | 详细说明 | 交互逻辑 |
|---|---|---|
| **状态标签页** | 全部 / 待上架 / 已上架 / 已下架 | 切换时自动刷新数据，显示各状态数量 |
| **分类导航树** | 左侧显示商品分类树，每类后显示商品数量 | 点击分类联动筛选商品列表 |
| **分类搜索** | 搜索框输入快速定位分类 | 输入时实时过滤分类树 |

#### 1.2 商品筛选条件

| 筛选项 | 类型 | 说明 |
|---|---|---|
| 关键词 | 输入框 | 模糊匹配SKU名称/SKU编码 |
| 商品分类 | 级联选择器 | 支持一/二/三级分类筛选 |
| 供应商 | 下拉选择 | 选择特定供应商的商品 |
| 供货状态 | 下拉选择 | 供货中/暂停供货/已停货 |

#### 1.3 批量运营操作

| 操作按钮 | 触发条件 | 二次确认 |
|---|---|---|
| **导入商品** | 始终显示 | 否（跳转导入页） |
| **批量上架** | 选中行>0且行包含待上架商品 | 确认上架N件商品？ |
| **批量下架** | 选中行>0且行包含已上架商品 | 确认下架N件商品？ |
| **批量调价** | 选中行>0 | 弹出调价窗口输入比例 |
| **批量设置权限** | 选中行>0 | 弹出权限配置窗口 |

#### 1.4 商品表格字段

| 列名 | 展示规则 | 交互 |
|---|---|---|
| **商品信息** | 主图(60×60) + SKU名称 + SKU编码 + 规格标签 | 点击跳转SKU详情 |
| **分类/SPU** | 分类名称 / SPU名称+编码 | - |
| **采购库存** | 库存数 + 单位 | 数字变红（库存<安全值） |
| **供应商** | 单家：名称+价格<br>多家："N家供应商"+悬浮展示全部 | Popover悬浮 |
| **供货状态** | 标签：绿/橙/红 | - |
| **进入方式** | 标签：手动导入(橙) / 自动进入(绿) | - |
| **上架状态** | Switch开关 | 点击直接切换 |
| **可见范围** | "全部可见"或"M个工程仓" | 点击Popover展示工程仓列表 |
| **操作** | 查看 / 编辑 | - |

### 模块2：销售商品市场（工程仓端）

#### 2.1 商品瀑布流展示

| 功能点 | 详细说明 |
|---|---|
| **搜索框** | 顶部固定，搜索SKU名称/编码 |
| **购物车入口** | 右上角Badge显示商品总数，点击跳转购物车 |
| **排序方式** | 综合 / 销量 / 价格 |
| **卡片规格** | 一行3列卡片式布局 |

#### 2.2 商品卡片规范

```
┌───────────────────────────┐
│  ┌─────────────────────┐  │
│  │      主图 200x200   │  │
│  │  [热销] [推荐]      │  │  <- 营销标签
│  └─────────────────────┘  │
│  SKU标准名称（最多2行）   │
│  规格：HRB400 16mm        │
│  3家供应商 → ￥4000起     │  <- 多供应商提示
│  ─────────────────────    │
│  ￥4000 / 吨      [+]     │  <- 快速加购按钮
└───────────────────────────┘
```

#### 2.3 交互规则
- 点击卡片区域：跳转商品详情
- 点击[+]按钮：直接加入购物车，弹出成功提示
- 多供应商商品加入购物车时自动选择最低价供应商

---

### 模块3：购物车

#### 3.1 购物车列表

| 功能 | 说明 |
|---|---|
| **批量删除** | 选中商品后顶部显示删除按钮 |
| **数量调整** | 按钮式增减，最大值=库存 |
| **实时计算** | 数量变更时小计、合计自动刷新 |
| **供应商分组** | 按供应商分组展示商品（可选） |

#### 3.2 底部结算栏

```
☑️ 全选    已选3件        合计：￥28,640.00    [ 结算(3) ]
```

| 元素 | 逻辑 |
|---|---|
| 全选 | 切换全部商品选中状态 |
| 已选N件 | 统计选中行数 |
| 合计金额 | 选中商品(单价×数量)求和 |
| 结算按钮 | 选中行数=0时禁用 |

### 模块4：BOM基装包管理

#### 4.1 BOM包列表页

| 功能点 | 说明 |
|---|---|
| **创建BOM包** | 右上角创建按钮，跳转向导页 |
| **筛选条件** | BOM名称模糊搜索 / 上架状态 |
| **列表字段** | BOM编码 + 包信息(图+名称) + SKU数量 + SKU总价 + 标签 + 可见工程仓 + 上架开关 + 操作 |

#### 4.2 BOM包创建向导（2步）

**步骤1：基本信息**
| 字段 | 必填 | 规则 |
|---|---|---|
| BOM包名称 | ✅ | 50字以内 |
| BOM主图 | ❌ | 1张，推荐正方形 |
| 标签 | ❌ | 最多5个，用于筛选 |
| BOM说明 | ❌ | 500字以内 |

**步骤2：SKU明细配置**
| 功能 | 说明 |
|---|---|
| 添加SKU | 弹窗选择平台SKU |
| 批量导入 | Excel模板导入 |
| 必选标记 | 控制该SKU能否从包中删除 |
| 实时汇总 | SKU种数 + SKU单价总计 |

#### 4.3 BOM包权限配置

- 支持穿梭框方式选择可见的工程仓
- 支持全选/清空快速操作
- 可设置同步上架到商品市场

### 模块5：分类展示管理

#### 5.1 分类列表

| 功能 | 说明 |
|---|---|
| 分类名称 | 按层级缩进显示 |
| 商品统计 | 总数/已上架/待上架/热门/推荐 |
| 市场可见开关 | 控制分类在前端是否展示 |
| 排序 | 数字越小越靠前 |
| 操作 | 查看该分类下商品列表 |

#### 5.2 商品分类内查看

- 可查看该分类下所有商品
- 显示市场状态、热门/推荐标签
- 支持快速批量操作

## 四、非功能需求

### 4.1 性能指标

| 指标 | 要求 |
|---|---|
| 商品列表加载 | < 1s |
| 分类树渲染 | < 500ms |
| 购物车计算更新 | < 100ms |
| 页面并发支持 | 500人同时在线 |

### 4.2 数据埋点需求

| 埋点 | 说明 |
|---|---|
| 商品曝光 | 每一个SKU卡片曝光次数 |
| 商品点击 | 卡片点击次数 |
| 加购行为 | 直接加购 vs 详情页加购 |
| 分类转化率 | 从分类到下单的转化漏斗 |
| BOM使用率 | BOM包在订单中的占比 |

### 4.3 异常处理

| 场景 | 处理方式 |
|---|---|
| 加购时库存不足 | 提示："当前供应商库存仅剩下X件" |
| 商品已下架 | 购物车置灰，结算时自动排除 |
| 供应商停货 | 自动切换到下一个最低价供应商 |

## 五、上线与验收标准

### 5.1 上线计划

| 阶段 | 时间 | 范围 |
|---|---|---|
| 灰度测试 | 第1周 | 3家试点工程仓 |
| 全量上线 | 第2周 | 所有工程仓 |
| 数据运营 | 第3周 | 根据数据调优 |

### 5.2 验收标准

✅ **功能验收**：
1. 供应商品市场所有筛选、批量操作正常
2. 商品市场加购、购物车、结算流程完整
3. BOM包创建、上架、权限配置正常
4. 分类可见性开关前后端联动生效

✅ **性能验收**：
1. 首屏加载 < 1.5s
2. 500条数据表格渲染无卡顿
3. 无前端控制台报错
