# 项目PRD工作流Skill定义

> 本文定义AI助手在项目中生成PRD文档的完整工作流程。
> 版本：v1.0 | 创建日期：YYYY-MM-DD

---

## 一、工作流概述

### 1.1 适用场景

- **新项目起步**：从头创建一套完整的PRD文档体系
- **新增模块**：在已有项目中新增一个功能模块的PRD
- **模块修订**：对现有PRD文档做重大修订
- **终端扩展**：为多端系统新增一个终端的PRD文档

### 1.2 输入要求

| 输入项 | 格式 | 说明 |
|:------|:----|:-----|
| 项目基本信息 | 文本 | 项目名称、目标用户、核心定位 |
| 功能清单 | CSV/表格 | 功能点列表（模块/菜单/功能/优先级） |
| 现有PRD参考 | MD文件 | 同项目其他模块的已有PRD |
| 业务规则 | 文本 | 业务约束和核心逻辑 |

### 1.3 输出产物

| 产物 | 数量 | 说明 |
|:----|:----:|:-----|
| PRD总纲（prd.md） | 1 | 文档索引+设计原则+术语+权限+功能全景 |
| 系统概览 | 1 | 系统定位+架构+模块树 |
| 业务流程 | 1 | 核心业务流程图 |
| 数据模型 | 1 | 表结构设计 |
| 领域模型 | 1 | DDD领域模型 |
| 模块详细设计 | N | 每功能模块1篇 |
| 更新日志 | 1 | 全局变更记录 |

## 二、工作流步骤

### 步骤1：信息收集

```
┌────────────┐     ┌────────────┐     ┌────────────┐
│ 项目信息    │ ──→ │ 功能清单    │ ──→ │ 业务规则    │
└────────────┘     └────────────┘     └────────────┘
                          │
                          ▼
                   ┌──────────────┐
                   │ 明确文档边界   │
                   └──────────────┘
```

**AI动作**：
- 询问项目名称、定位、目标用户
- 询问系统端数（单端/多端）
- 询问已有功能清单或产品范围
- 确认是否已有历史PRD文档参考

**输出**：确认信息清单

---

### 步骤2：文档体系规划

```
┌──────────────────┐
│ 选择文档架构方案  │
│ （单端/多端）     │
└──────┬───────────┘
       │
       ▼
┌──────────────────┐
│ 确定文件编号方案  │
│ 端名+NN-名称.md   │
└──────┬───────────┘
       │
       ▼
┌──────────────────┐
│ 确定模板选择     │
│ 含技术/PM纯净版   │
└──────────────────┘
```

**AI动作**：
- 判断单端/多端，选择对应文档架构
- 生成文件清单（哪些文件需要创建）
- 输出文档规划表

---

### 步骤3：编写PRD总纲

```
┌──────────────────┐
│ prd.md 总纲      │
│ ①文档索引         │
│ ②设计原则         │
│ ③术语表           │
│ ④角色权限矩阵      │
│ ⑤功能全景         │
│ ⑥核心流程图       │
│ ⑦全局交互规范      │
└──────────────────┘
```

**AI动作**：
- 用 `prd.md` 模板生成总纲
- 填充所有变体 `{变量}`
- 完成后标记为完整

---

### 步骤4：编写系统级文档

```
┌────────────┐  ┌────────────┐  ┌────────────┐
│ 01-系统概览  │  │ 02-业务流程  │  │ 03-数据模型  │
└────────────┘  └────────────┘  └────────────┘
                                               ┌────────────┐
                                               │ 04-领域模型  │
                                               └────────────┘
```

**AI动作**：
- 顺序编写系统概览 → 业务流程 → 数据模型 → 领域模型
- 每完成一篇，使用 `docs/01_PRD/` 或对应路径存放
- 同步更新 `prd.md` 中的文档索引

---

### 步骤5：编写模块详细设计（核心）

```
功能点1  ─→ 功能概述 → 业务规则 → 功能点详情 → 状态矩阵 → 流程图
功能点2  ─→ 功能概述 → 业务规则 → 功能点详情 → 状态矩阵 → 流程图
功能点N  ─→ ...
```

**AI动作**：
- 每模块独立1篇 `NN-{模块名称}功能设计.md`
- 使用 `05-系统模块详细设计.md` 或 `05-系统模块详细设计-PM纯净版.md`
- 每篇文档结构：
  1. 功能概述（定位/概念/用户/范围）
  2. 核心设计原则
  3. 业务规则（校验/查询/命名）
  4. 功能点详情（逐点说明/入口/交互/字段/规则/UI/异常）
  5. 功能权限矩阵
  6. 状态治理矩阵
  7. 非功能性需求
  8. 接口依赖
  9. 版本历史

---

### 步骤6：补充更新日志

- 创建 `更新日志/` 目录下的全局 `更新日志.md`
- 记录本次 PRD 变更范围覆盖的文件列表
- 每篇PRD文件内的版本历史同步追加

---

### 步骤7：质量检查

| 检查项 | 标准 |
|:------|:-----|
| 格式统一 | 所有文档头部元数据格式一致 |
| 编号连续 | 文档索引无空号、无重复 |
| 术语一致 | 全系统术语统一，无歧义 |
| 变量填充 | 所有 `{变量}` 已被替换 |
| 引用可访问 | 文档间相互引用的路径正确 |
| 流程图完整 | Mermaid 语法正确，无孤立节点 |

## 三、多端PRD工作流

### 3.1 四端独立编写模式

```
平台端PRD/
├── prd.md               ← 平台端总纲
├── 01-系统概览与架构.md   ← 平台端视角
├── 02-业务流程设计.md     ← 跨端流程（标注端边界）
├── 03-{模块}功能设计.md   ← 平台端功能
└── ...

供应商端PRD/
├── prd.md               ← 供应商端总纲
├── 01-系统概览与架构.md   ← 供应商端视角
├── 02-{模块}功能设计.md   ← 供应商端功能
└── ...

工程仓端PRD/
├── prd.md
├── ...
└── ...

施工方端PRD/
├── prd.md
├── ...
└── ...
```

### 3.2 共享文档策略

| 文档类型 | 平台端 | 供应商端 | 工程仓端 | 施工方端 |
|:--------|:-----:|:--------:|:--------:|:--------:|
| 业务流程 | ✅ 全量 | 🔄 子集 | 🔄 子集 | 🔄 子集 |
| 数据模型 | ✅ 主 | 🔄 引用 | 🔄 引用 | 🔄 引用 |
| 领域模型 | ✅ 主 | 🔄 引用 | 🔄 引用 | 🔄 引用 |

### 3.3 编写顺序建议

```
建议顺序：平台端 → 供应商端 → 工程仓端 → 施工方端
                      ↓
          每完成一端，同步更新跨端差异点
```

## 四、Skills集成说明

### 4.1 Skill加载触发词

| 场景 | 触发词 |
|:----|:-------|
| 完整PRD生成 | `/generate-prd` |
| 新增模块 | `/new-module` |
| 修订PRD | `/update-prd` |
| 生成操作手册 | `/gen-manual` |

### 4.2 Skill配置文件路径

```
.trae/skills/
├── prd-generator-helper/
│   └── SKILL.md          ← 当前Skill
├── product-requirement-generator/
│   └── SKILL.md          ← 通用PRD生成器
└── ...
```

### 4.3 Skill输入示例

```markdown
用户输入：
> 使用 `/generate-prd` 为一个工程项目管理SaaS系统生成PRD
> - 系统名称：工程仓端SaaS
> - 终端：小程序（UniApp）
> - 已有功能清单：[链接]
> - 目标用户：仓库管理员、采购员
```
