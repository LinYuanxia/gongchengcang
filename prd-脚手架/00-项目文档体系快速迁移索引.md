# 项目文档体系 — 快速迁移索引

> 复制本索引到新项目，按路径逐一创建文件，即可复用完整的 PRD 文档体系。
> 版本：v1.1 | 适用：任何 SaaS / B 端项目（含单系统/多端两种场景）

---

## 场景适配概览

| 项目类型 | 举例 | 适用度 | 需要调整的地方 |
|:--------|:----|:-----:|:-------------|
| **多端协同系统**（本文档默认场景） | 平台+供应商+工程仓+施工方采供平台 | ✅ 原生支持 | 直接使用，替换端名即可 |
| **单系统 + PC+APP** | 门店资产管理系统（管理后台+APP） | ✅ 完全适用 | 简化PRD端数、调整PrdSource、合并文档 |
| **纯单系统PC端** | CRM / ERP / OA 管理后台 | ✅ 完全适用 | 去掉PrdSource切换，直接用一套模块 |
| **纯移动端应用** | 小程序/App | ✅ 完全适用 | 用模板B（快速参考卡）替代模板A |

### 4.2 命名规范

| 规则 | 示例 |
|:----|:------|
| 目录命名：`NN_中文名称/` | `01_PRD/`、`00_项目总览/` |
| 文件命名：`NN-中文名称.md` | `05-商品中心功能设计.md` |
| 总纲命名：`prd.md` | `prd.md` |
| 操作手册目录：`02_系统操作手册/{端名}/` | `02_系统操作手册/01_平台端/` |
| 通用资源目录：`03_通用资源/{NN_类别}/` | `03_通用资源/03_规范/` |
| 截图命名：`模块_页面_序号.png` | `order_list_01.png` |
| 截图目录：`images/` 放在各端PRD目录内 | `门店资产/images/` |

### 4.3 PRD 章节结构（六层模型）

```
Layer 1: 文档头部元数据（版本/状态/所属章节/负责人）
Layer 2: 功能概述（功能定位/核心概念/目标用户/模块范围）
Layer 3: 业务规则（状态规则/校验规则/查询规则/命名规范）
Layer 4: 功能点详细设计（逐个功能点的说明/入口/交互/字段/规则）
Layer 5: 状态治理矩阵（动作定义/状态×操作矩阵/角色权限/错误提示）
Layer 6: 附录（Mermaid流程图/版本历史）
```

### 4.4 操作手册章节结构（9章+2附录）

- **PC端操作手册** 和 **APP端操作手册** 各一本，独立编写
- 章节结构一致，内容根据端差异调整：

```
1.  操作前必读        ← ⚠️核心规则表格（5条以上）
2.  快速入门          ← 登录+第一个核心任务（含截图占位符）
3.  功能详解          ← 每模块：页面概览→操作步骤→字段说明→状态说明→流转图
4.  典型工作场景      ← 2个以上跨模块场景
5.  常见问题FAQ       ← 14个以上Q&A表格
6.  常见错误提示      ← 错误信息→含义→处理方法的对照表
7.  注意事项          ← 结构化表格
8.  联系支持          ← 联系方式表格
附录A 文档版本记录
附录B 截图采集清单    ← 按模块列出需截取的页面+标注要点
```


```bash

# 同步项目级文档
cp -f docs/00_项目总览/*.md apps/pc/public/prd-docs/
```

## 七、PM纯净版PRD — 内容边界速查

### 允许保留
- ✅ 业务规则与逻辑描述
- ✅ 字段定义及业务含义
- ✅ 领域模型（实体/值对象关系）
- ✅ 数据库表结构设计（表名+字段名+业务含义，去掉字段类型）
- ✅ 状态×操作矩阵
- ✅ 权限矩阵
- ✅ 角色定义
- ✅ 边界情况/异常场景
- ✅ 业务流程图（Mermaid）
- ✅ 需求版本历史

### 必须剔除
- ❌ 字段具体类型（String(100)/Integer/Decimal/Enum）
- ❌ API 接口路径（/api/xxx/create）
- ❌ HTTP 状态码（200/403/500）
- ❌ 前后端实现机制（"前端防抖+后端幂等"）
- ❌ 乐观锁/分布式锁等技术方案
- ❌ 数据库索引设计（普通索引/唯一索引）
- ❌ 缓存策略/队列设计
- ❌ 第三方SDK的具体调用方式
