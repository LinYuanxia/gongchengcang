# 目录结构说明

## 设计原则

1. **按文档类型分类** — 顶层目录按文档用途分（项目总览 / PRD / 操作手册 / 资源 / 归档）
2. **按端组织** — 同类型文档下按端（平台/供应商/工程仓/施工方）细分
3. **编号导向** — `NN_` 前缀保证目录排列有序

## 为什么这样组织？

| 场景 | 对应目录 |
|:-----|:---------|
| "项目整体长什么样？" | `00_项目总览/` |
| "平台端商品功能的需求在哪？" | `01_PRD/平台端/` |
| "供应商怎么用这个系统？" | `02_系统操作手册/02_供应商端/` |
| "写文档的模板在哪？" | `03_通用资源/02_模板/` |
| "之前那个旧版文档在哪？" | `04_历史归档/` |
