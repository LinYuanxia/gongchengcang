# 采购计划产品设计方案

## 一、产品概述

### 1.1 产品定位
采购计划是平台端和工程仓端协同管理采购需求的核心功能模块，支持两种流程模式：
- **平台发起模式**：平台创建采购计划并推送给工程仓，工程仓确认后生成采购订单
- **工程仓发起模式**：工程仓直接创建采购计划，确认后生成采购订单

### 1.2 产品目标
- 支持两种采购计划发起模式，满足不同业务场景
- 规范采购流程，确保采购需求的准确性和可追溯性
- 实现平台端与工程仓端的高效协同
- 为采购决策提供数据支持

### 1.3 适用范围
- 平台端：采购计划管理（创建、编辑、发布、推送、查看）
- 工程仓端：采购计划管理（创建、查看、确认）

### 2.2 采购计划状态流转

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│                       采购计划状态流转                                                       │
└─────────────────────────────────────────────────────────────────────────────────────────┘

平台端状态：
    草稿(draft)
        │
        ▼
    已发布(published)
        │
        ▼
    部分确认(partial)
        │
        ▼
    全部确认(confirmed)
        │
        ▼
    已完成(completed)

工程仓端状态：
    草稿(draft)
        │
        ▼
    待确认(pending)  ←─────────────────────────┐
        │                                      │
        ▼                                      │
    已确认(confirmed)                            │
        │                                      │
        ▼                                      │
    已下单(ordered)                              │
        │                                      │
        ▼                                      │
    已完成(completed)                            │
                                              │
    (接收平台推送的计划直接进入待确认状态)───────────┘

状态说明：
平台端：
- 草稿：计划已创建但未发布，可编辑、删除
- 已发布：计划已发布，已推送给工程仓，等待工程仓确认
- 部分确认：部分工程仓已确认，部分工程仓未确认
- 全部确认：所有工程仓已确认
- 已完成：所有工程仓已确认，所有订单已完成

工程仓端：
- 草稿：工程仓自己创建但未确认的计划，可编辑、删除
- 待确认：已接收平台推送或已创建但未确认的计划
- 已确认：工程仓已确认计划
- 已下单：已生成采购订单
- 已完成：订单已完成
```

## 四、核心功能模块

### 4.1 平台端 - 采购计划管理

#### 4.1.1 采购计划列表
- 展示所有采购计划
- 支持按计划编号、计划名称、状态筛选
- 支持分页加载
- 操作按钮：查看、编辑、发布、继续推送、删除

#### 4.1.2 创建采购计划
- 填写基本信息：计划名称、计划说明
- 添加采购商品清单：选择SKU、设置采购数量
- 保存为草稿或直接发布

#### 4.1.3 发布采购计划
- 选择推送的工程仓（支持多选）
- 设置截止时间
- 批量推送

#### 4.1.4 推送工程仓
- 查看已推送的工程仓列表
- 查看每个工程仓的确认状态
- 支持继续推送更多工程仓

#### 4.1.5 查看采购计划详情
- 查看计划基本信息
- 查看采购商品清单
- 查看推送工程仓列表
- 查看确认状态

### 4.2 工程仓端 - 采购计划管理

#### 4.2.1 采购计划列表
- 展示本工程仓的采购计划
- 支持标签页切换：全部、待处理、已确认
- 支持分页加载
- 操作按钮：查看、确认下单、查看订单

#### 4.2.2 新建采购计划
- 填写基本信息：计划名称、计划说明、截止时间
- 添加采购商品清单：选择SKU、设置采购数量
- 保存为草稿或直接确认

#### 4.2.3 确认采购计划
- 查看计划详情
- 确认采购商品清单
- 可调整实际采购数量
- 确认后生成采购订单

#### 4.2.4 查看采购订单
- 查看订单基本信息
- 查看订单商品清单
- 查看订单状态
- 查看物流信息

## 七、接口设计

### 7.1 平台端接口

#### 7.1.1 获取采购计划列表
- 接口路径：`GET /api/v1/procurement/plans`
- 请求参数：
  - planNo: 计划编号（可选）
  - planName: 计划名称（可选）
  - status: 计划状态（可选）
  - page: 页码
  - pageSize: 每页条数
- 返回数据：
  ```json
  {
    "code": 200,
    "message": "success",
    "data": {
      "list": [
        {
          "id": "1",
          "planNo": "PO202401001",
          "planName": "2024年1月建材采购计划",
          "skuCount": 15,
          "estimatedAmount": 125800,
          "status": "published",
          "statusText": "已发布",
          "pushCount": 2,
          "confirmCount": 1,
          "createTime": "2024-01-15 10:00:00",
          "pushList": [
            {
              "warehouseId": "1",
              "warehouseName": "北京工程仓",
              "status": "confirmed",
              "statusText": "已确认",
              "pushTime": "2024-01-15 14:00:00",
              "confirmTime": "2024-01-16 10:00:00"
            }
          ]
        }
      ],
      "total": 100,
      "page": 1,
      "pageSize": 10
    }
  }
  ```

#### 7.1.2 创建采购计划
- 接口路径：`POST /api/v1/procurement/plans`
- 请求参数：
  ```json
  {
    "planName": "2024年1月建材采购计划",
    "description": "本月常规建材补货计划",
    "skuList": [
      {
        "skuId": "1",
        "quantity": 100
      }
    ]
  }
  ```
- 返回数据：
  ```json
  {
    "code": 200,
    "message": "success",
    "data": {
      "planId": "1",
      "planNo": "PO202401001"
    }
  }
  ```

#### 7.1.3 发布采购计划
- 接口路径：`POST /api/v1/procurement/plans/:id/publish`
- 请求参数：
  ```json
  {
    "warehouseIds": ["1", "2", "3"],
    "deadline": "2024-01-20 18:00:00"
  }
  ```
- 返回数据：
  ```json
  {
    "code": 200,
    "message": "success",
    "data": {
      "success": true,
      "pushCount": 3
    }
  }
  ```

#### 7.1.4 继续推送工程仓
- 接口路径：`POST /api/v1/procurement/plans/:id/push-more`
- 请求参数：
  ```json
  {
    "warehouseIds": ["4", "5"]
  }
  ```
- 返回数据：
  ```json
  {
    "code": 200,
    "message": "success",
    "data": {
      "success": true,
      "pushCount": 2
    }
  }
  ```

#### 7.1.5 删除采购计划
- 接口路径：`DELETE /api/v1/procurement/plans/:id`
- 返回数据：
  ```json
  {
    "code": 200,
    "message": "success",
    "data": {
      "success": true
    }
  }
  ```

## 八、异常处理

### 8.1 平台端异常处理

| 异常场景 | 异常描述 | 处理方式 | 提示信息 |
|---------|---------|---------|---------|
| 计划不存在 | 计划ID不存在或已删除 | 返回错误信息 | "采购计划不存在" |
| 计划状态错误 | 计划已发布，不能编辑 | 返回错误信息 | "计划已发布，不能编辑" |
| 无推送权限 | 没有推送工程仓的权限 | 返回错误信息 | "没有推送工程仓的权限" |
| 工程仓不存在 | 工程仓ID不存在或已删除 | 返回错误信息 | "工程仓不存在" |
| 计划已确认 | 计划已确认，不能继续推送 | 返回错误信息 | "计划已确认，不能继续推送" |

### 8.2 工程仓端异常处理

| 异常场景 | 异常描述 | 处理方式 | 提示信息 |
|---------|---------|---------|---------|
| 计划不存在 | 计划ID不存在或已删除 | 返回错误信息 | "采购计划不存在" |
| 计划状态错误 | 计划已确认，不能确认 | 返回错误信息 | "计划已确认，不能重复确认" |
| 无确认权限 | 没有确认计划的权限 | 返回错误信息 | "没有确认采购计划的权限" |
| 商品已下架 | 商品已下架，不能确认 | 返回错误信息 | "部分商品已下架，请检查商品清单" |
| 库存不足 | 供应商库存不足 | 返回警告信息 | "部分商品供应商库存不足，建议调整数量" |
