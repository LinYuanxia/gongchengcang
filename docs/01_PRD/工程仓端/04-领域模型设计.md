## 一、功能概述

### 1.1 功能定位

本文档定义工程仓端的**领域模型**，包括核心领域实体、领域服务、领域事件。面向开发团队，指导后端代码的领域驱动设计（DDD）实现，确保业务逻辑的准确性和可维护性。

### 1.2 核心概念

| 概念 | 说明 |
|-----|------|
| 领域实体 | 有唯一标识的业务对象，如订单、库存 |
| 值对象 | 无唯一标识的概念性对象，如地址、金额 |
| 领域服务 | 跨实体的业务操作，如库存调拨 |
| 领域事件 | 业务操作触发的通知，如订单创建事件 |
| 聚合根 | 实体关系的根节点，如Order是OrderItem的聚合根 |

### 1.3 目标用户

- **后端开发工程师**：基于领域模型设计代码结构和数据库访问层
- **架构师**：评估领域划分和实体关系设计的合理性
- **技术负责人**：理解核心业务逻辑的代码映射

## 二、核心领域实体

### 2.1 工程仓主体（WarehouseMerchant）

**核心属性：**
- id, merchantName, creditCode, legalPerson, contactName, contactPhone, address, businessLicense, status

**关联关系：**
- 1:N 员工 (Employee)
- 1:N 子仓库 (SubWarehouse)
- 1:N 采购计划 (PurchasePlan)
- 1:N 订单 (Order) - 作为采购方
- 1:N 订单 (Order) - 作为销售方

**领域方法：**
- `viewProfile()`: 查看主体信息
- `updateProfile(profileData)`: 更新主体信息（仅主管可调用）

**业务规则：**
- 一个工程仓有且仅有一条主体信息
- 主体信息变更需记录操作日志

---

### 2.2 子仓库（SubWarehouse）

**核心属性：**
- id, merchantId, warehouseName, address, managerName, managerPhone, area, status

**关联关系：**
- N:1 工程仓主体
- 1:N 库存记录 (Inventory)
- 1:N 入库单 (InboundOrder)
- 1:N 出库单 (OutboundOrder)
- 1:N 盘点单 (StockCheck)

**领域方法：**
- `getInventoryList()`: 获取该仓库存列表
- `checkStockAvailability(skuId, quantity)`: 检查某SKU库存是否充足

**业务规则：**
- 停用的子仓库不可进行入库/出库操作
- 一个工程仓至少有一个子仓库

---

### 2.3 订单（Order） - 核心领域对象

**三状态分离属性：**
- `orderStatus`: pending → confirmed → shipped → completed / cancelled
- `paymentStatus`: unpaid → paid → refunded（**独立标记**，不影响其他状态）
- `shipStatus`: pending → partial → shipped（**独立运行**，不依赖支付）

**核心属性：**
- id, orderNo, orderType, buyerId, sellerId, totalAmount, remark

**关联关系：**
- 1:N 订单明细 (OrderItem)
- 1:1 采购扩展 (PurchaseOrder)
- 1:1 销售扩展 (SaleOrder)
- 1:N 售后单 (AfterSale)
- 1:N 资金流水 (FundFlow)

**领域方法：**
- `cancel(reason)`: 取消订单（仅待确认状态）
- `confirm()`: 确认订单（销售订单）
- `receive(warehouseId, items)`: 收货完成订单
- `canCancel()`: 校验是否可取消
- `canShip()`: 校验是否可发货

**三状态校验黄金准则：**
```typescript
// ✅ 正确：发货只依赖订单状态，不校验支付
const canShip = (order: Order): boolean => {
  return order.orderStatus === 'confirmed' 
      && order.shipStatus !== 'shipped'
}

// ✅ 正确：收货只依赖订单状态
const canReceive = (order: Order): boolean => {
  return order.orderStatus === 'shipped'
}
```

---

### 2.4 采购订单（PurchaseOrder extends Order）

**扩展属性：**
- supplierId, warehouseId, source

**领域方法：**
- `createFromCart(cartItems)`: 从购物车创建采购订单
- `createManually(params)`: 手动创建采购订单
- `receiveGoods()`: 收货入库（委托给InboundOrder）

**业务规则：**
- 一个采购订单只能关联一个供应商
- 订单来源标记：cart/manual/plan

---

### 2.5 销售订单（SaleOrder extends Order）

**扩展属性：**
- constructionId, shippingCompany, trackingNo

**领域方法：**
- `confirmByWarehouse()`: 工程仓确认
- `ship()`: 确认发货（委托给OutboundOrder）

**业务规则：**
- 确认后自动生成出库单
- 支持部分发货（shipStatus→partial）

---

### 2.6 库存（Inventory）

**核心属性：**
- id, warehouseId, skuId, quantity, lockQuantity, version

**领域方法：**
- `increase(quantity, operator)`: 增加库存（入库）
- `decrease(quantity, operator)`: 减少库存（出库，使用乐观锁）
- `isSufficient(quantity)`: 库存是否充足
- `checkAndDecrease(quantity)`: 检查并扣减（原子操作）

**乐观锁实现：**
```sql
-- 扣减时自动校验version
UPDATE inventory SET quantity = quantity - ?, version = version + 1
WHERE warehouse_id = ? AND sku_id = ? AND quantity >= ? AND version = ?
-- 影响行数=0 → 并发冲突或库存不足
```

---

### 2.7 入库批次（InboundBatch）

**核心属性：**
- id, inboundId, skuId, batchNo, orderedQty, receivedQty, damageQty, damageImages, damageDesc

**领域方法：**
- `hasDamage()`: 是否有货损
- `createAfterSale()`: 生成售后单（damageQty>0时调用）

---

### 2.8 购物车（CartItem）

**核心属性：**
- id, employeeId, merchantId, skuId, supplierId, quantity, selected

**领域方法：**
- `updateQuantity(qty)`: 修改数量
- `toggleSelect()`: 切换选中状态
- `isValid()`: 校验商品是否可售

---

### 2.9 采购计划（PurchasePlan）

**核心属性：**
- id, merchantId, planName, status, expectedDate, remark

**领域方法：**
- `convertToOrder()`: 转为采购订单
- `canConvert()`: 校验能否转订单（所有商品可售）

---

### 2.10 资金流水（FundFlow）

**核心属性：**
- id, merchantId, flowNo, orderId, amount, flowType, paymentMethod, voucherUrl

**领域方法：**
- `uploadVoucher(file)`: 上传支付凭证
- `linkOrder(orderId)`: 关联订单

## 三、领域服务

| 服务名 | 方法签名 | 描述 |
|-------|---------|------|
| **InventoryService** | `receive(skuId, warehouseId, quantity)` | 入库增加库存 |
|  | `ship(skuId, warehouseId, quantity)` | 出库减少库存（乐观锁） |
|  | `transfer(fromWarehouseId, toWarehouseId, skuId, qty)` | 库存调拨 |
|  | `check(skuId, warehouseId)` | 查询库存 |
| **OrderService** | `createFromCart(cartIds)` | 从购物车批量创建订单 |
|  | `createPurchaseOrder(params)` | 手动创建采购订单 |
|  | `cancelOrder(orderId, reason)` | 取消订单 |
|  | `confirmSaleOrder(orderId)` | 确认销售订单 |
| **AfterSaleService** | `create(orderId, damageInfo)` | 创建售后单 |
|  | `process(afterSaleId, result)` | 处理售后（补发/退款/拒绝） |
| **InvoiceService** | `upload(file, info)` | 上传发票凭证 |
|  | `certify(invoiceId)` | 认证发票 |
| **PlanService** | `convertToOrder(planId)` | 采购计划转订单 |
| **AuthService** | `login(username, password)` | 用户登录 |
|  | `checkPermission(userId, permission)` | 权限校验 |

## 四、领域事件

| 事件名称 | 触发条件 | 事件数据 | 监听处理 |
|---------|---------|---------|---------|
| **OrderCreated** | 订单创建成功 | orderId, orderType, buyerId, sellerId | 通知供应商端/施工方端 |
| **OrderConfirmed** | 销售订单确认 | orderId | 生成出库单 |
| **OrderReceived** | 收货完成 | orderId, warehouseId, items | 更新库存、生成售后单（如有货损） |
| **OrderCancelled** | 订单取消 | orderId, reason | 通知对方、释放库存（如有） |
| **OutboundConfirmed** | 出库确认 | outboundId, warehouseId, items | 更新库存、更新订单发货状态 |
| **AfterSaleCreated** | 售后单创建 | afterSaleId, orderId, type | 通知供应商端处理 |
| **PaymentProofUploaded** | 支付凭证上传 | fundFlowId, orderId, amount | 更新订单支付状态→paid |

## 五、领域模型分层架构

```
┌──────────────────────────────────────┐
│          展现层（Controller）           │
│  REST API / GraphQL 接口              │
└──────────────────┬───────────────────┘
                   │
┌──────────────────▼───────────────────┐
│           应用层（Application）         │
│  用例编排、事务管理、权限校验           │
└──────────────────┬───────────────────┘
                   │
┌──────────────────▼───────────────────┐
│           领域层（Domain） ⭐           │
│  实体 + 值对象 + 领域服务 + 领域事件    │
└──────────────────┬───────────────────┘
                   │
┌──────────────────▼───────────────────┐
│          基础设施层（Infrastructure）    │
│  Repository实现、数据库访问、外部服务调用 │
└──────────────────────────────────────┘
```
