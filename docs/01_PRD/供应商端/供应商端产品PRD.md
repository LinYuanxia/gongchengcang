## 1. 产品概述

### 1.1 产品定位

供应商端是**工程仓采供一体化平台**的核心端口之一，为建材供应商提供全链路数字化经营管理能力，涵盖订单履约、物流跟踪、资金结算、发票管理四大核心领域。

### 1.2 核心价值

| 价值维度 | 具体描述 |
|---------|---------|
| **订单协同** | 与工程仓实时订单同步，减少电话/微信沟通成本 |
| **物流透明** | 多物流单号跟踪，配送状态实时更新 |
| **资金安全** | 托管账户 + 分账机制，货款安全有保障 |
| **财务合规** | 发票-支付-对账三流合一，满足财税合规要求 |

### 1.3 目标用户

| 用户角色 | 典型特征 | 核心诉求 |
|---------|---------|---------|
| **老板/财务** | 中小型建材企业主 | 看报表、管资金、开发票 |
| **跟单员** | 订单处理专员 | 接单、发货、跟进物流 |
| **库管员** | 仓库管理人员 | 备货、出库、签收确认 |

### 5.2 售后管理模块（独立页面）

**设计理念：售后与订单完全解耦**

> ✅ **行业首创：三状态分离架构**
>
> | 传统系统问题 | 我们的解决方案 |
> |-------------|-------------|
> | 一退款订单就变"已取消" | 订单状态永远不变 |
> | 看不到原始订单数据 | 售后数据独立关联 |
> | 不能多次退款 | 一个订单N次售后 |

#### 5.2.1 售后工作台

**顶部统计看板（4项）**：
| 统计项 | 预警规则 |
|-------|---------|
| 待处理 | 数字标红 + 右上角红点 |
| 处理中 | 正常显示 |
| 已同意 | 正常显示 |
| 已拒绝 | 正常显示 |

#### 5.2.2 退款审批详情

**审批操作三要素**：
1. **查看凭证**：采购方上传的问题照片（最多9张）
2. **核对数量**：退款商品明细 + 申请退款数量
3. **输入意见**：同意/拒绝的具体说明

**⏰ 超时自动处理规则**：
```
退款申请提交后开始计时
        ↓
  72小时未处理 → 系统自动发送提醒
        ↓
  15天仍未处理 → 系统默认同意退款
        ↓
    触发自动分账退款
```

#### 5.2.3 售后数据溯源

| 数据维度 | 记录内容 |
|---------|---------|
| **操作日志** | 申请人 → 提交时间 → 审核人 → 审核时间 → 审核意见 |
| **资金流向** | 退款金额从哪笔支付款扣除 → 分账流水号 |
| **商品来源** | 对应原订单哪几行商品 → 发货批次号 |

#### 5.3.2 支付记录系统

**解决的痛点**：财务分不清哪笔钱对应哪笔订单

| 传统问题 | 我们的方案 |
|---------|---------|
| 只有账户流水，看不到订单 | 每笔支付关联订单号 |
| 不知道手续费扣了多少 | 支付金额/手续费/实际到账 三栏分列 |
| 找不到付款人是谁 | 完整显示付款方账户/名称 |

**表格核心字段（11列）**：

| 列名 | 说明 | 计算逻辑 |
|------|------|---------|
| 支付流水号 | PAY开头的唯一单号 | 系统自动生成 |
| 关联订单号 | 点击可跳转到订单详情 | - |
| 支付方式 | 对公/微信/支付宝/余额 | - |
| 支付状态 | 待支付/成功/失败/退款 | - |
| **支付金额** | 付款方实际支付 | - |
| **手续费** | 第三方收取的费用 | • 微信/支付宝：0.6%<br>• 对公/余额：0元 |
| **实际到账** | 扣除手续费后到账 | 支付金额 - 手续费 |
| 付款方 | 完整公司名称 + 脱敏账号 | - |
| 支付时间 | 支付成功时间戳 | - |
| 到账时间 | 资金到托管账户时间 | - |
| 操作 | 详情/下载回单 | - |

**💡 财务小贴士**：
> 实际到账 = 支付金额 - 手续费
>
> 例：微信支付10000元 → 手续费60元 → 实际到账9940元

#### 5.3.3 销项发票管理

**供应商场景简化**：只开销项给工程仓，不需要进项

**开票流程**：

```
选择待开票订单
        ↓
系统自动带入开票信息（购方名称/税号/金额）
        ↓
输入发票号码 + 发票代码
        ↓
选择税率（默认13%）
        ↓
上传发票PDF扫描件
        ↓
完成开票，发送通知给采购方
```

**开票三原则**：
| 规则 | 说明 |
|------|------|
| 1️⃣ **开票金额 ≤ 订单金额** | 不能多开，可分多次开 |
| 2️⃣ **发票内容 = 订单商品** | 不能开与订单不符的品名 |
| 3️⃣ **三流合一校验** | 订单金额 = 支付金额 = 开票金额（±0.01元容差） |

**发票红冲功能**：
- 仅"已开票"状态可红冲
- 红冲后生成负数发票记录
- 红冲原因需备注说明

**对账三步骤**：

#### 11.4.3 完整交互流程

**🔹 新增员工流程：**
```
点击【新增员工】→ 弹出表单
    │
    ├─ 必填字段：账号/初始密码/姓名/手机号/角色
    ├─ 选填字段：部门/职位/邮箱
    └─ 默认状态：启用
    │
    ↓
系统校验：账号唯一 / 手机号唯一
    │
    ↓
创建成功 → 短信通知初始密码 → 员工首次登录强制改密
```

**🔹 编辑员工流程：**
- 账号不可修改
- 可修改：姓名/手机号/部门/职位/角色/邮箱
- 修改手机号需验证原手机号 + 新手机号

**🔹 重置密码流程：**
```
管理员点击【重置密码】
    ↓
弹出确认框：确定重置该员工密码？
    ↓
系统生成随机密码 → 短信发送给员工
    ↓
员工使用临时密码登录 → 强制修改密码
```

**🔹 启用/禁用流程：**
```
点击【禁用】→ 二次确认
    ↓
账号禁用 → 强制踢下线 → 无法登录
    ↓
点击【启用】→ 恢复账号权限
```

> 💡 **安全规则：超级管理员账号不可禁用、不可删除**

**🔹 删除员工流程：**
- 仅非管理员账号可删除
- 删除前二次确认
- 操作日志永久保留删除记录
