# 采购计划功能详细设计

## 一、功能概述

采购计划是平台端和工程仓端协同管理采购需求的核心功能模块，支持两种流程模式：
- **平台发起模式**：平台创建采购计划并推送给工程仓，工程仓确认后采购计划完成，同时生成采购订单
- **工程仓发起模式**：工程仓直接创建采购计划，确认后采购计划完成，同时生成采购订单

**核心逻辑**：
- 采购计划和订单是两个独立的业务实体，有各自的状态流转
- 工程仓确认采购计划后，采购计划的生命周期就完成了
- 订单有自己的状态流转（待确认、已确认、发货中、已完成等）

## 二、平台端 - 采购计划管理

### 2.1 采购计划列表展示

#### 功能说明
以列表形式展示所有采购计划，支持搜索、筛选、分页加载

#### 交互逻辑
1. 页面加载时显示所有采购计划，默认按创建时间倒序排列
2. 支持按计划编号、计划名称、计划状态进行筛选
3. 支持分页加载，当计划数量较多时采用分页方式
4. 列表展示字段：
   - 计划编号：可点击查看详情
   - 计划名称：计划名称
   - SKU数量：包含的SKU种类数量
   - 预计金额：预计总金额，右对齐，红色字体
   - 推送工程仓：已推送的工程仓数量，鼠标悬停展示工程仓列表和状态
   - 计划状态：草稿、已发布、部分确认、全部确认、已完成、已取消
   - 创建时间：计划创建时间
   - 操作按钮：查看、编辑、发布、继续推送、删除

#### 字段说明
| 字段名称 | 字段说明 | 数据类型 | 展示方式 |
|---------|---------|---------|---------|
| 计划编号 | 计划唯一编号 | VARCHAR | 链接展示，可点击查看详情 |
| 计划名称 | 计划名称 | VARCHAR | 文本展示 |
| SKU数量 | 包含的SKU种类数量 | INT | 文本展示，居中显示，单位"个" |
| 预计金额 | 预计总金额 | DECIMAL | 文本展示，右对齐，红色字体，带千分位 |
| 推送工程仓 | 已推送的工程仓数量 | INT | 悬停展示，鼠标悬停展示工程仓列表和状态 |
| 计划状态 | 计划状态 | VARCHAR | 标签展示 |
| 创建时间 | 计划创建时间 | DATETIME | 文本展示，格式YYYY-MM-DD HH:mm:ss |

#### 界面元素
- 搜索区域：
  - 计划编号输入框：支持模糊搜索
  - 计划名称输入框：支持模糊搜索
  - 计划状态下拉框：草稿、已发布、部分确认、全部确认、已完成、已取消
  - 搜索按钮：点击执行搜索
  - 重置按钮：清空搜索条件
- 列表区域：
  - 表格展示所有采购计划
  - 支持分页
- 操作按钮：
  - 查看：查看计划详情
  - 编辑：编辑计划（仅草稿状态）
  - 发布：发布计划（仅草稿状态）
  - 继续推送：继续推送工程仓（已发布、部分确认状态）
  - 删除：删除计划（仅草稿状态）

#### 权限说明
- 平台管理员：可查看所有采购计划
- 平台运营：可查看所有采购计划

#### 异常处理
- 网络异常：提示"网络异常，请稍后重试"
- 无数据：提示"暂无数据"
- 搜索无结果：提示"未找到符合条件的采购计划"

### 2.2 创建采购计划

#### 功能说明
创建新的采购计划，可保存为草稿或直接发布

#### 交互逻辑
1. 点击"创建采购计划"按钮，弹出创建弹窗
2. 填写基本信息：
   - 计划名称：必填，限制200字
   - 计划说明：选填，限制500字
3. 添加采购商品清单：
   - 点击"添加商品"按钮，弹出商品选择器
   - 选择SKU，设置采购数量
   - 可删除已添加的商品
4. 保存计划：
   - 点击"创建"按钮，保存为草稿
   - 草稿状态可继续编辑、发布或删除

#### 字段说明
| 字段名称 | 字段说明 | 数据类型 | 必填 | 验证规则 |
|---------|---------|---------|------|---------|
| 计划名称 | 计划名称 | VARCHAR | 是 | 1-200字 |
| 计划说明 | 计划说明 | TEXT | 否 | 最多500字 |
| SKU列表 | 采购商品清单 | ARRAY | 是 | 至少选择1个SKU |
| 采购数量 | 每个SKU的采购数量 | INT | 是 | 1-99999 |

#### 界面元素
- 基本信息表单：
  - 计划名称输入框
  - 计划说明文本框
- 商品清单区域：
  - 添加商品按钮
  - 商品表格：SKU编码、商品名称、单位、建议采购量、供货价、预计金额、备注、操作
- 底部操作栏：
  - 商品统计：共X个SKU，预计总金额¥X
  - 取消按钮
  - 创建按钮

#### 权限说明
- 平台管理员：可创建采购计划
- 平台运营：可创建采购计划

#### 异常处理
- 表单验证失败：提示具体错误信息
- 商品选择器加载失败：提示"加载商品列表失败，请稍后重试"
- 保存失败：提示"创建采购计划失败，请稍后重试"

### 2.4 发布采购计划

- 计划基本信息：计划编号、计划名称、预计金额

## 四、采购计划与订单的关系

### 4.1 业务关系

采购计划和订单是两个独立的业务实体：

| 维度 | 采购计划 | 采购订单 |
|-----|---------|---------|
| 生命周期 | 平台创建 → 发布 → 工程仓确认 → 完成 | 待确认 → 已确认 → 发货中 → 已完成 |
| 状态流转 | 草稿 → 已发布 → 部分确认 → 全部确认 → 已完成 | 待确认 → 已确认 → 发货中 → 已完成 |
| 管理主体 | 平台创建，工程仓确认 | 工程仓确认，供应商处理 |

### 4.2 数据关联

- 采购计划确认后，生成采购订单
- 采购订单关联采购计划ID
- 采购计划详情页可查看关联的订单信息
- 采购订单详情页可查看关联的计划信息

### 4.3 业务流程

```
平台创建采购计划
    ↓
发布并推送给工程仓
    ↓
工程仓接收计划（待确认）
    ↓
工程仓确认计划
    ├─→ 采购计划状态变为"已确认"（计划生命周期完成）
    └─→ 生成采购订单（订单状态为"待确认"）
            ↓
        供应商确认订单
            ↓
        供应商发货
            ↓
        工程仓收货入库
            ↓
        订单状态变为"已完成"
```
