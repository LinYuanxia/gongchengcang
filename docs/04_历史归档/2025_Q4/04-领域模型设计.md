## 一、领域模型总览

### 1.1 核心领域概念

| 领域概念 | 描述 | 所属模块 |
|---------|------|---------|
| 商户主体 (Merchant) | 系统中的商业实体，包括供应商、工程仓、施工方 | 商户管理 |
| 员工 (Employee) | 各商户主体的工作人员 | 用户管理 |
| 项目 (Project) | 施工方负责的具体工程项目 | 项目管理 |
| 商品 (Product) | 系统中管理的材料商品，包括SPU和SKU | 商品管理 |
| 订单 (Order) | 采购或销售的交易订单 | 订单管理 |
| 库存 (Stock) | 工程仓的物料存储管理 | 库存管理 |
| 领料 (MaterialRequisition) | 施工方向工程仓申请领取材料 | 领料管理 |
| 财务 (Finance) | 系统中的资金和账务管理 | 财务管理 |

### 1.2 领域分层架构

```
┌─────────────────────────────────────────────────────────────────────────┐
│                          领域分层架构                                   │
└─────────────────────────────────────────────────────────────────────────┘

┌─────────────────────┐    ┌─────────────────────┐    ┌─────────────────────┐
│    应用层 (Application)    │    │    领域层 (Domain)    │    │    基础设施层 (Infrastructure)    │
├─────────────────────┤    ├─────────────────────┤    ├─────────────────────┤
│  - 应用服务        │    │  - 领域服务        │    │  - 数据访问        │
│  - 控制器          │    │  - 实体            │    │  - 外部集成        │
│  - 视图模型        │    │  - 值对象          │    │  - 工具类          │
│  - 命令/查询处理   │    │  - 领域事件        │    │  - 配置管理        │
│                    │    │  - 仓储接口        │    │                    │
└─────────────────────┘    └─────────────────────┘    └─────────────────────┘
```

## 二、核心领域实体

### 2.1 商户主体 (Merchant)

**核心属性：**
- `id`: 商户ID
- `merchantName`: 商户名称
- `merchantType`: 商户类型 (供应商/工程仓/施工方)
- `legalPerson`: 法人代表
- `contactName`: 联系人
- `contactPhone`: 联系电话
- `status`: 状态

**关联关系：**
- 1:N 员工 (Employee)
- 1:1 供应商信息 (Supplier)
- 1:1 工程仓信息 (Warehouse)
- 1:1 施工方信息 (Construction)
- 1:N 合同 (Contract)

**领域方法：**
- `createMerchant()`: 创建商户
- `updateMerchant()`: 更新商户信息
- `approveMerchant()`: 审批商户
- `suspendMerchant()`: 暂停商户

### 2.2 员工 (Employee)

**核心属性：**
- `id`: 员工ID
- `merchantId`: 所属商户ID
- `username`: 用户名
- `realName`: 真实姓名
- `phone`: 手机号
- `roleId`: 角色ID
- `status`: 状态

**关联关系：**
- N:1 商户 (Merchant)
- N:1 角色 (Role)
- N:N 项目 (Project) via ProjectMember

**领域方法：**
- `createEmployee()`: 创建员工
- `updateEmployee()`: 更新员工信息
- `assignRole()`: 分配角色
- `deactivateEmployee()`: 停用员工

### 2.3 项目 (Project)

**核心属性：**
- `id`: 项目ID
- `projectName`: 项目名称
- `projectNo`: 项目编号
- `warehouseId`: 工程仓ID
- `constructionId`: 施工方ID
- `address`: 项目地址
- `budget`: 项目预算
- `status`: 状态
- `progress`: 进度

**关联关系：**
- N:1 工程仓 (Warehouse)
- N:1 施工方 (Construction)
- N:N 员工 (Employee) via ProjectMember
- 1:N 项目预算 (ProjectBudget)
- 1:N 项目进度 (ProjectProgress)
- 1:N 订单 (Order)
- 1:N 领料单 (MaterialReq)

**领域方法：**
- `createProject()`: 创建项目
- `updateProject()`: 更新项目信息
- `updateProgress()`: 更新项目进度
- `completeProject()`: 完成项目
- `cancelProject()`: 取消项目

### 2.4 商品 (Product)

**SPU核心属性：**
- `id`: SPU ID
- `spuName`: 商品名称
- `spuCode`: 商品编码
- `categoryId`: 分类ID
- `brand`: 品牌
- `unit`: 单位
- `status`: 状态

**SKU核心属性：**
- `id`: SKU ID
- `spuId`: 所属SPU ID
- `skuCode`: SKU编码
- `specs`: 规格属性
- `stock`: 库存数量
- `price`: 销售价格
- `status`: 状态

**关联关系：**
- SPU 1:N SKU
- SPU 1:N 分类 (Category)
- SKU 1:N 供货价格 (SupplyPrice)
- SKU 1:N 库存 (Stock)
- SKU 1:N 订单明细 (OrderItem)

**领域方法：**
- `createProduct()`: 创建商品
- `updateProduct()`: 更新商品信息
- `approveProduct()`: 审核商品
- `shelveProduct()`: 上架商品
- `unshelveProduct()`: 下架商品

### 2.5 订单 (Order)

**核心属性：**
- `id`: 订单ID
- `orderNo`: 订单编号
- `buyerId`: 买方ID
- `sellerId`: 卖方ID
- `projectId`: 项目ID
- `orderType`: 订单类型 (采购/销售)
- `totalAmount`: 订单金额
- `status`: 订单状态
- `paymentStatus`: 支付状态

**关联关系：**
- 1:N 订单明细 (OrderItem)
- 1:1 采购订单 (PurchaseOrder)
- 1:1 销售订单 (SaleOrder)
- N:1 项目 (Project)
- N:1 入库单 (StockIn) / 出库单 (StockOut)

**领域方法：**
- `createOrder()`: 创建订单
- `confirmOrder()`: 确认订单
- `cancelOrder()`: 取消订单
- `payOrder()`: 支付订单
- `deliverOrder()`: 发货
- `receiveOrder()`: 收货
- `completeOrder()`: 完成订单

### 2.6 库存 (Stock)

**核心属性：**
- `id`: 库存ID

- `skuId`: SKU ID
- `quantity`: 库存数量
- `lockQuantity`: 锁定数量
- `costPrice`: 成本价格

**关联关系：**
- N:1 仓库 (Warehouse)
- N:1 SKU (SKU)
- N:1 入库明细 (StockInItem)
- N:1 出库明细 (StockOutItem)

**领域方法：**
- `increaseStock()`: 增加库存
- `decreaseStock()`: 减少库存
- `lockStock()`: 锁定库存
- `unlockStock()`: 解锁库存

**关联关系：**

- `deposit()`: 存款

## 三、领域服务

### 3.1 商户管理服务 (MerchantService)

**职责：**
- 商户生命周期管理
- 商户资质审核
- 商户状态管理

**核心方法：**
- `registerMerchant(merchantDTO)`: 注册商户
- `approveMerchant(merchantId, approvalDTO)`: 审批商户
- `updateMerchantInfo(merchantId, merchantDTO)`: 更新商户信息
- `getMerchantDetail(merchantId)`: 获取商户详情
- `getMerchantList(filterDTO)`: 获取商户列表

### 3.2 员工管理服务 (EmployeeService)

**职责：**
- 员工生命周期管理
- 员工权限分配
- 员工状态管理

**核心方法：**
- `createEmployee(employeeDTO)`: 创建员工
- `updateEmployee(employeeId, employeeDTO)`: 更新员工信息
- `assignRole(employeeId, roleId)`: 分配角色
- `deactivateEmployee(employeeId)`: 停用员工
- `getEmployeeList(merchantId, filterDTO)`: 获取员工列表

### 3.3 项目管理服务 (ProjectService)

**职责：**
- 项目生命周期管理
- 项目进度跟踪
- 项目资源管理

**核心方法：**
- `createProject(projectDTO)`: 创建项目
- `updateProject(projectId, projectDTO)`: 更新项目信息

- `completeProject(projectId)`: 完成项目

- `getProjectList(filterDTO)`: 获取项目列表

### 3.4 商品管理服务 (ProductService)

**职责：**
- 商品生命周期管理
- 商品审核流程
- 商品价格管理

**核心方法：**
- `createProduct(productDTO)`: 创建商品

- `unshelveProduct(productId)`: 下架商品
- `getProductList(filterDTO)`: 获取商品列表

### 3.5 订单管理服务 (OrderService)

**职责：**
- 订单生命周期管理
- 订单状态流转

**核心方法：**
- `createOrder(orderDTO)`: 创建订单

- `getOrderList(filterDTO)`: 获取订单列表

### 3.6 库存管理服务 (StockService)

**职责：**
- 库存变动管理
- 库存状态维护
- 库存盘点处理

**核心方法：**
- `stockIn(stockInDTO)`: 入库
- `stockOut(stockOutDTO)`: 出库

**职责：**
- 领料流程管理
- 领料审批处理

**核心方法：**
- `createRequisition(requisitionDTO)`: 创建领料申请
- `approveRequisition(requisitionId, approvalDTO)`: 审批领料申请

- `createScrap(scrapDTO)`: 创建报废申请
- `getRequisitionList(filterDTO)`: 获取领料列表

### 3.8 财务管理服务 (FinanceService)

**职责：**
- 账户管理
- 资金流转处理
- 账务核算

**核心方法：**
- `createAccount(merchantId)`: 创建账户
- `deposit(accountId, amount, remark)`: 存款
- `withdraw(accountId, amount, remark)`: 取款
- `transfer(fromAccountId, toAccountId, amount, remark)`: 转账
- `freezeAmount(accountId, amount, reason)`: 冻结金额
- `unfreezeAmount(accountId, amount, reason)`: 解冻金额
- `getAccountBalance(merchantId)`: 获取账户余额
- `getFundFlowList(accountId, filterDTO)`: 获取资金流水

## 六、领域规则与约束

### 6.1 商户规则

- **唯一性约束**: 商户名称、统一社会信用代码必须唯一
- **状态流转**: 商户状态只能从待审核 → 正常/拒绝 → 冻结 → 正常
- **资质要求**: 商户必须上传营业执照等资质文件
- **类型约束**: 商户类型一旦确定，不可修改

### 6.2 员工规则

- **唯一性约束**: 用户名、手机号、邮箱必须唯一
- **关联约束**: 员工必须属于某个商户
- **权限约束**: 员工权限不能超过其所属商户的权限范围
- **状态约束**: 员工状态只能在正常和禁用之间切换

### 6.3 项目规则

- **唯一性约束**: 项目编号必须唯一
- **关联约束**: 项目必须关联工程仓和施工方
- **状态流转**: 项目状态只能按待启动 → 进行中 → 已完工 → 已结算的顺序流转
- **进度约束**: 项目进度必须在0-100之间

### 6.4 商品规则

- **唯一性约束**: SPU编码、SKU编码必须唯一
- **状态流转**: 商品状态必须按草稿 → 审核中 → 已上架/审核拒绝 → 已下架的顺序流转
- **价格约束**: 商品价格必须大于0
- **库存约束**: 库存数量不能为负数

### 6.5 订单规则

- **唯一性约束**: 订单编号必须唯一
- **状态流转**: 订单状态必须按待确认 → 待发货 → 待收货 → 已完成的顺序流转
- **金额约束**: 订单金额必须大于0
- **支付约束**: 订单必须先支付才能发货
- **库存约束**: 下单时必须检查库存是否充足

### 6.6 库存规则

- **数量约束**: 库存数量不能为负数
- **锁定约束**: 锁定数量不能超过可用库存
- **批次约束**: 同一批次的商品必须同时入库和出库
- **盘点约束**: 盘点差异必须有合理的原因说明

### 6.7 领料规则

- **权限约束**: 领料申请必须由项目相关人员发起
- **审批约束**: 领料申请必须经过审批才能执行
- **库存约束**: 领料数量不能超过仓库可用库存
- **项目约束**: 领料必须关联到具体项目

### 6.8 财务规则

- **余额约束**: 账户余额不能为负数
- **冻结约束**: 冻结金额不能超过账户余额
- **流水约束**: 每笔资金变动必须生成流水记录
- **转账约束**: 转账金额必须小于转出账户余额
