> 版本：v1.0  
> 文档状态：评审中  
> 所属模块：供应商管理 - 供应商供货与工程仓配置

---

## 一、功能概述

### 1.1 功能定位

供应商供货与工程仓配置是平台供应链管理的核心功能，负责管理供应商与工程仓之间的供货关系，以及供应商向工程仓供货的完整业务流程。通过该功能，工程仓可以选择合适的供应商建立供货关系，供应商可以管理自己的供货商品和供货价格，为平台的采购业务提供基础支撑。

### 1.2 核心概念

| 概念 | 说明 | 示例 |
|-----|------|------|
| 供应商 | 提供商品的企业或个人 | 海螺水泥供应商 |
| 工程仓 | 采购商品的仓库或企业 | 北京工程仓、上海工程仓 |
| 供货关系 | 供应商与工程仓之间的供货合作关系 | 海螺水泥供应商 → 北京工程仓 |
| 供货商品 | 供应商向工程仓供货的商品 | 普通硅酸盐水泥 32.5级 |
| 供货价格 | 供应商向工程仓供货的价格 | ¥100.00/吨 |
| 供货协议 | 供应商与工程仓之间的供货协议 | 年度供货协议 |

### 1.3 目标用户

- 平台管理员
- 平台运营人员
- 供应商（管理自己的供货关系和供货商品）
- 工程仓（管理自己的供应商和供货商品）

## 二、业务规则

### 2.1 供货关系规则

- **供货关系建立**：
  - 工程仓可以主动邀请供应商建立供货关系
  - 供应商可以主动申请与工程仓建立供货关系
  - 供货关系需要双方确认后生效
- **供货关系类型**：
  1. **正式供货关系**：双方已确认，可正常供货
  2. **试用供货关系**：试用期，可小批量供货
  3. **终止供货关系**：已终止，不再供货
- **供货关系唯一性**：
  - 一个供应商与一个工程仓之间只能有一个有效的供货关系
  - 不能重复建立供货关系
- **供货关系期限**：
  - 支持设置供货关系有效期
  - 有效期到期后自动终止
  - 可以提前终止供货关系
- **供货关系审批**：
  - 供货关系建立需要平台审核
  - 审核通过后生效
  - 审核拒绝后可以重新申请

### 2.2 供货商品规则

- **供货商品关联**：
  - 供应商可以将自己的商品关联到供货关系中
  - 一个供货关系可以关联多个商品
  - 一个商品可以关联到多个供货关系
- **供货商品状态**：
  1. **正常供货**：商品正常供货中
  2. **暂停供货**：商品暂停供货
  3. **停止供货**：商品停止供货
- **供货商品价格**：
  - 每个供货商品可以设置独立的供货价格
  - 供货价格可以随时调整
  - 价格调整后，新订单使用新价格，已下单的订单不受影响
- **供货商品库存**：
  - 供应商管理自己的库存
  - 工程仓可以查看供应商的可售库存
  - 下单时扣减供应商库存

### 2.3 供货价格规则

- **价格类型**：
  1. **供货价**：供应商向工程仓供货的价格
  2. **市场价**：市场参考价格
  3. **最低供货价**：最低供货价格，用于防低价竞争
- **价格精度**：价格保留2位小数
- **价格调整**：
  - 支持批量调整供货价格
  - 支持按比例调整供货价格
  - 价格调整需要记录调整历史
- **价格有效期**：
  - 支持设置价格有效期
  - 有效期到期后自动恢复原价
  - 可以提前终止价格优惠

### 2.4 供货协议规则

- **协议类型**：
  1. **年度供货协议**：有效期一年
  2. **季度供货协议**：有效期三个月
  3. **临时供货协议**：有效期一个月
  4. **特殊供货协议**：按需定制
- **协议内容**：
  - 供货商品范围
  - 供货价格条款
  - 供货数量条款
  - 交货时间条款
  - 付款方式条款
  - 违约责任条款
- **协议签署**：
  - 支持线上签署电子协议
  - 支持线下签署纸质协议
  - 协议需要双方盖章后生效
- **协议变更**：
  - 协议变更需要双方协商一致
  - 变更后需要重新签署
  - 变更历史需要记录

### 2.5 供货配额规则

- **配额设置**：
  - 可以为每个供货商品设置供货配额
  - 配额可以按周期设置（月度、季度、年度）
  - 超过配额后不能继续下单
- **配额调整**：
  - 支持临时调整配额
  - 调整配额需要双方确认
  - 配额调整历史需要记录
- **配额预警**：
  - 当配额使用达到预警值时自动提醒
  - 预警值可以自定义设置

### 2.6 供货评价规则

- **评价类型**：
  1. **商品质量评价**：对商品质量的评价
  2. **交货速度评价**：对交货速度的评价
  3. **服务态度评价**：对服务态度的评价
  4. **价格合理性评价**：对价格合理性的评价
- **评价等级**：
  - 五星评级（1-5星）
  - 文字评价
- **评价规则**：
  - 订单完成后可以评价
  - 评价可以修改一次
  - 评价公开透明，可作为其他工程仓选择供应商的参考

### 2.7 删除约束规则

- **供货关系删除**：
  - 终止的供货关系可以删除
  - 有未完成订单的供货关系不能删除
  - 删除后历史数据保留
- **供货商品删除**：
  - 停止供货的商品可以删除
  - 有未完成订单的商品不能删除
  - 删除后历史数据保留

## 三、功能点详细设计

### 3.1 供应商管理（工程仓视角）

#### 3.1.1 供应商列表展示

##### 功能说明
工程仓查看与自己有供货关系的供应商列表

##### 交互逻辑
1. 页面顶部显示筛选条件：
   - 供应商状态筛选：多选标签（正式、试用、终止）
   - 供应商类型筛选：下拉选择框
   - 供货商品分类筛选：下拉选择框
2. 筛选条件右侧显示搜索框：
   - 支持按供应商名称、联系人搜索
   - 支持按回车键搜索
3. 供应商列表展示字段：
   - 供应商名称
   - 供应商类型
   - 联系人
   - 联系电话
   - 供货商品数量
   - 合作开始时间
   - 合作状态（标签展示：正式/试用/终止）
   - 操作按钮（查看/编辑/终止合作/查看供货商品）
4. 支持分页加载，默认每页20条

##### 字段说明
| 字段名称 | 字段说明 | 数据类型 | 展示方式 |
|---------|---------|---------|---------|
| 供应商名称 | 供应商名称 | VARCHAR | 主标题展示 |
| 供应商类型 | 供应商类型 | VARCHAR | 标签展示 |
| 联系人 | 联系人姓名 | VARCHAR | 文本展示 |
| 联系电话 | 联系电话 | VARCHAR | 文本展示 |
| 供货商品数量 | 供货商品的数量 | INT | 数字展示 |
| 合作开始时间 | 合作开始时间 | DATETIME | 文本展示 |
| 合作状态 | 合作状态 | TINYINT | 标签展示 |

##### 界面元素
- 筛选条件区域
- 搜索框
- 供应商列表容器
- 分页控件

---

#### 3.1.2 邀请供应商

##### 功能说明
工程仓主动邀请供应商建立供货关系

##### 交互逻辑
1. 点击"邀请供应商"按钮
2. 弹出邀请供应商弹窗
3. 表单包含以下字段：
   - **供应商***：
     - 下拉选择框，显示所有已入驻的供应商
     - 可以搜索供应商
   - **合作类型***：
     - 单选按钮组
     - 选项：正式合作 / 试用合作
     - 默认选中"正式合作"
   - **供货商品分类***：
     - 下拉选择框，树形结构
     - 可以选择多个分类
   - **合作说明**：
     - 文本框，最多500字
     - 描述合作意向和要求
   - **有效期**：
     - 日期选择框
     - 可以设置合作有效期
     - 不设置则长期有效
4. 点击"确定"按钮：
   - 前端校验表单完整性
   - 调用后端接口发送邀请
   - 供应商收到邀请通知
   - 弹出成功提示："邀请已发送"
5. 点击"取消"按钮或弹窗外部关闭弹窗

##### 字段说明
| 字段名称 | 字段说明 | 数据类型 | 约束 | 默认值 |
|---------|---------|---------|------|--------|
| 供应商ID | 供应商ID | BIGINT | 必填 | - |
| 合作类型 | 合作类型 | TINYINT | 必填 | 1（正式合作） |
| 供货商品分类 | 供货商品分类ID数组 | JSON | 必填 | - |
| 合作说明 | 合作说明 | VARCHAR(500) | 可选 | 空 |
| 有效期 | 合作有效期 | DATETIME | 可选 | 空 |

##### 界面元素
- 弹窗标题：邀请供应商
- 供应商选择下拉框
- 合作类型单选按钮组
- 供货商品分类选择器
- 合作说明文本框
- 有效期日期选择框
- 确定按钮
- 取消按钮

同供应商列表字段说明

- 基本信息展示区域

#### 3.1.4 终止供货关系

##### 功能说明
工程仓终止与供应商的供货关系

##### 交互逻辑
1. 在供应商列表的操作按钮中点击"终止合作"按钮
2. 弹出终止合作弹窗
3. 表单包含以下字段：
   - **终止原因***：
     - 单选按钮组
     - 选项：商品质量问题 / 交货不及时 / 价格不合理 / 服务不好 / 其他

     - 文本框，最多500字

   - **生效时间***：
     - 单选按钮组
     - 选项：立即生效 / 待订单完成后生效
4. 点击"确定"按钮：
   - 前端校验表单完整性
   - 调用后端接口终止供货关系
   - 供应商收到终止通知
   - 弹出成功提示："合作已终止"
5. 点击"取消"按钮或弹窗外部关闭弹窗

##### 字段说明
| 字段名称 | 字段说明 | 数据类型 | 约束 |
|---------|---------|---------|------|
| 终止原因 | 终止原因 | TINYINT | 必填 |
| 终止原因说明 | 终止原因说明 | VARCHAR(500) | 选择"其他"时必填 |
| 生效时间 | 生效时间 | TINYINT | 必填 |

##### 界面元素
- 弹窗标题：终止合作
- 终止原因单选按钮组
- 终止原因说明文本框
- 生效时间单选按钮组
- 确定按钮
- 取消按钮

## 四、数据校验规则

### 4.1 供货关系校验
- 唯一性校验：一个供应商与一个工程仓之间只能有一个有效的供货关系
- 状态校验：供货关系状态必须合法

### 4.2 供货商品校验
- 存在性校验：供货商品必须是供应商自己的商品
- 价格校验：供货价格必须大于0
- 库存校验：供货商品库存必须大于等于0

### 4.3 供货价格校验
- 数值校验：价格必须大于0
- 精度校验：价格保留2位小数
- 最低价格校验：供货价格不能低于最低供货价

### 4.4 供货协议校验
- 时间校验：生效时间必须早于到期时间
- 金额校验：协议金额必须大于等于0
- 状态校验：协议状态必须合法

### 4.5 供货评价校验
- 等级校验：评价等级必须在1-5之间
- 内容校验：评价内容最多500字
- 图片校验：最多上传5张图片

---

## 五、异常处理

### 5.1 表单验证异常
- 实时显示验证错误信息
- 提交时再次校验，阻止无效提交

### 5.2 供货关系重复异常
- 建立供货关系时检测到重复
- 给出明确的错误提示

### 5.3 供货价格调整异常
- 价格调整低于最低供货价
- 给出明确的错误提示

### 5.4 网络异常
- 操作失败时显示错误提示
- 提供重试按钮
- 失败时保持表单数据，避免重复输入

### 5.5 权限异常
- 无操作权限时隐藏操作按钮
- 尝试操作时显示权限不足提示

## 八、接口设计

**请求参数**：
- `status`：合作状态（可选）
- `supplierType`：供应商类型（可选）
- `categoryId`：供货商品分类ID（可选）
- `keyword`：搜索关键词（可选）
- `pageNum`：页码（可选）
- `pageSize`：每页数量（可选）
**响应数据**：供应商列表数据

### 8.2 邀请供应商
**接口路径**：`POST /api/supplier/warehouse/invite`  
**接口说明**：工程仓邀请供应商  
**请求参数**：
- `supplierId`：供应商ID（必填）
- `cooperationType`：合作类型（必填）

- `description`：合作说明（可选）
- `expireTime`：合作有效期（可选）
**响应数据**：邀请结果

### 8.3 获取工程仓列表（供应商视角）
**接口路径**：`GET /api/supplier/supplier/warehouses`  
**接口说明**：供应商获取工程仓列表  
**请求参数**：
- `status`：合作状态（可选）

- `pageSize`：每页数量（可选）
**响应数据**：工程仓列表数据

### 8.4 申请供货
**接口路径**：`POST /api/supplier/supplier/apply`  
**接口说明**：供应商申请供货  
**请求参数**：
- `warehouseId`：工程仓ID（必填）
- `cooperationType`：合作类型（必填）

- `keyword`：搜索关键词（可选）
- `pageNum`：页码（可选）
- `pageSize`：每页数量（可选）

**接口路径**：`POST /api/supplier/supply-products`  
**接口说明**：添加供货商品  
**请求参数**：
- `relationId`：供货关系ID（必填）

**接口路径**：`PUT /api/supplier/supply-products/price`  
**接口说明**：调整供货价格  
**请求参数**：
- `id`：供货商品ID（必填）
- `supplyPrice`：供货价格（必填）
- `marketPrice`：市场价格（可选）
- `minSupplyPrice`：最低供货价（可选）
- `priceDescription`：价格说明（可选）
- `effectTime`：生效时间（必填）
- `specifyEffectTime`：指定生效时间（指定时间生效时必填）
- `expireTime`：价格有效期（可选）
**响应数据**：调整结果

**接口说明**：批量调整供货价格  
**请求参数**：
- `ids`：供货商品ID数组（必填）
- `adjustType`：调整类型（1-统一设置，2-批量调整，3-按比例调整）（必填）
- `supplyPrice`：统一设置的供货价格（统一设置时必填）
- `marketPrice`：统一设置的市场价格（可选）
- `adjustAmount`：调整金额（批量调整时必填）
- `adjustRatio`：调整比例（按比例调整时必填）
**响应数据**：批量调整结果
