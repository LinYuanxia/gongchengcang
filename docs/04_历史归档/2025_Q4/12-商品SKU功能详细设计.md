## 一、功能概述

### 1.1 功能定位

商品SKU（Stock Keeping Unit，库存保有单位）管理是平台商品体系的核心功能之一，负责管理商品的具体实例。SKU是SPU下的具体商品，每个SKU有独立的价格、库存、规格等属性，是商品交易的基本单位。SKU管理涵盖SKU的创建、编辑、价格管理、库存管理、状态管理等完整业务流程，为平台的商品交易提供基础支撑。

### 1.2 核心概念

| 概念 | 说明 | 示例 |
|-----|------|------|
| SKU | 库存保有单位，是具体的商品实例 | 普通硅酸盐水泥 32.5级 袋装 |
| SPU | 标准化产品单元，是SKU的上层概念 | 普通硅酸盐水泥 |
| 规格组合 | SKU的具体规格，由多个规格值组成 | 等级:32.5级，包装:袋装 |
| SKU价格 | SKU的销售价格、成本价格 | 销售价格:¥100.00，成本价格:¥80.00 |
| SKU库存 | SKU的库存数量 | 库存:100吨 |
| SKU状态 | SKU的销售状态 | 正常、售罄、下架 |

### 1.3 目标用户

- 平台管理员
- 平台运营人员
- 供应商（管理自己的商品SKU）
- 工程仓（管理自己的商品SKU）

## 二、业务规则

### 2.1 SKU编码规则

- **编码规则**：使用SPU编码 + "-" + 规格值编码组合的方式
  - 例如：SPU编码为SPU001001，规格值编码为325和bag，则SKU编码为SPU001001-325-bag
- **编码唯一性**：SKU编码在系统中全局唯一
- **编码自动生成**：创建SKU时，系统自动根据SPU编码和规格值编码生成SKU编码
- **编码可修改**：SKU编码可以手动修改，但必须保证唯一性

### 2.2 SKU名称规则

- **名称规则**：使用SPU名称 + " " + 规格值名称组合的方式
  - 例如：SPU名称为"普通硅酸盐水泥"，规格值为"32.5级"和"袋装"，则SKU名称为"普通硅酸盐水泥 32.5级 袋装"
- **名称自动生成**：创建SKU时，系统自动根据SPU名称和规格值名称生成SKU名称
- **名称可修改**：SKU名称可以手动修改

### 2.3 规格组合规则

- **组合唯一性**：同一SPU下的SKU规格组合必须唯一
  - 不能有两个SKU具有完全相同的规格组合
  - 如"等级:32.5级 + 包装:袋装"只能有一个SKU
- **组合完整性**：SKU必须包含所有规格组的规格值
  - SPU设置了哪些规格组，SKU就必须包含这些规格组的规格值
  - 如SPU设置了"等级"和"包装"两个规格组，则SKU必须同时包含"等级"和"包装"的规格值
- **组合数量限制**：单个SPU下的SKU数量建议不超过100个
  - 超过100个时性能会受影响
  - 超过时给出警告提示

### 2.4 SKU价格规则

- **价格必填**：SKU必须设置销售价格
  - 销售价格必须大于0
- **价格类型**：支持多种价格类型
  1. **市场价格**：市场价，仅供参考
  2. **销售价格**：实际销售价格
  3. **成本价格**：成本价格，用于利润计算
  4. **最低价格**：最低销售价格，用于防低价竞争
- **价格精度**：价格保留2位小数
- **价格继承**：SKU价格默认继承SPU价格，可以单独修改
- **价格批量修改**：支持批量修改SKU价格
  - 统一设置：所有选中SKU设置为同一个价格
  - 批量调整：所有选中SKU价格增加或减少指定金额
  - 按比例调整：所有选中SKU价格按比例调整

### 2.5 SKU库存规则

- **库存必填**：SKU必须设置库存数量
  - 库存数量必须大于等于0
  - 库存数量必须是整数
- **库存独立**：每个SKU有独立的库存
  - 不同SKU的库存互不影响
  - 如"普通硅酸盐水泥 32.5级 袋装"库存100吨，"普通硅酸盐水泥 42.5级 袋装"库存50吨
- **库存汇总**：SPU库存是所有SKU库存的总和
  - SPU库存 = 所有SKU库存的总和
  - SKU库存增加时，SPU库存自动增加
  - SKU库存减少时，SPU库存自动减少
- **库存预警**：支持库存预警设置
  - 当库存低于预警值时，系统自动提醒
  - 预警值可以单独为每个SKU设置
- **库存锁定**：下单时锁定库存
  - 下单成功后锁定库存
  - 订单取消后释放库存
  - 订单完成后扣减库存
- **库存调整**：支持手动调整库存
  - 支持增加库存
  - 支持减少库存
  - 支持设置库存为指定数量

### 2.6 SKU图片规则

- **主图可选**：SKU可以设置主图，也可以不设置
  - 不设置时使用SPU的主图
  - 设置时使用SKU自己的主图
- **图片格式**：支持JPG、PNG格式
- **图片大小**：单张图片不超过2MB
- **图片尺寸**：建议尺寸：800×800像素

### 2.7 SKU状态规则

- **SKU状态**：SKU有3种状态
  1. **正常**：SKU正常销售中
  2. **售罄**：SKU库存为0，暂时无法购买
  3. **下架**：SKU暂停销售
- **状态自动变更**：
  - 库存为0时，状态自动变为"售罄"
  - 库存大于0时，状态自动变为"正常"
- **状态手动变更**：
  - 可以手动将SKU状态设置为"下架"
  - 下架状态的SKU库存不受影响
  - 下架状态的SKU不能被搜索和购买
- **状态约束**：
  - 正常状态的SKU可以被搜索和购买
  - 售罄状态的SKU可以被搜索，但不能被购买
  - 下架状态的SKU不能被搜索和购买

### 2.8 SKU条形码规则

- **条形码可选**：SKU可以设置条形码，也可以不设置
- **条形码唯一性**：条形码在系统中唯一（如果设置）
- **条形码格式**：支持EAN-13、UPC、Code 128等格式

### 2.9 删除约束规则

- **删除校验**：删除SKU前必须检查是否有关联数据
  - 检查是否有订单：有订单的SKU不能删除
  - 检查是否有库存：有库存的SKU不能删除（建议先清空库存）
- **删除提示**：删除时根据校验结果给出明确的提示信息
  - 有订单时提示："该SKU有订单记录，不能删除"
  - 有库存时提示："该SKU有库存，确定要删除吗？"
- **逻辑删除**：SKU删除采用逻辑删除，不物理删除数据

## 三、功能点详细设计

### 3.1 商品SKU列表展示

#### 功能说明
以列表形式展示商品SKU，支持筛选、搜索和分页

#### 交互逻辑
1. 页面顶部显示筛选条件：
   - 商品分类筛选：下拉选择框，树形结构
   - 商品SPU筛选：下拉选择框，根据分类联动
   - SKU状态筛选：多选标签（正常、售罄、下架）
   - 供应商筛选：下拉选择框（平台端显示，供应商端不显示）
   - 价格区间筛选：最小价格和最大价格输入框
   - 库存区间筛选：最小库存和最大库存输入框
2. 筛选条件右侧显示搜索框：
   - 支持按SKU名称、SKU编码、条形码搜索
   - 支持按回车键搜索
3. 商品SKU列表展示字段：
   - SKU主图（小图，不设置时显示SPU主图）
   - SKU名称（最多显示30字，超出显示省略号）
   - SKU编码（灰色小字）
   - 所属SPU（显示SPU名称）
   - 规格组合（显示规格组合，如"等级:32.5级 包装:袋装"）
   - 销售价格（显示销售价格，如"¥100.00"）
   - 库存数量（显示库存数量，库存为0时标红）
   - SKU状态（标签展示：正常/售罄/下架）
   - 条形码（显示条形码，不设置时显示"-"）
   - 创建时间（格式：YYYY-MM-DD HH:mm）
   - 操作按钮（查看/编辑/删除/上架/下架/调整库存）
4. 支持分页加载，默认每页20条
5. 支持列表批量操作：
   - 批量上架
   - 批量下架
   - 批量调整价格
   - 批量调整库存
   - 批量删除（仅下架状态）

#### 字段说明
| 字段名称 | 字段说明 | 数据类型 | 展示方式 |
|---------|---------|---------|---------|
| SKU主图 | SKU主图URL | VARCHAR | 图片展示 |
| SKU名称 | SKU名称，最多200字 | VARCHAR | 主标题展示 |
| SKU编码 | SKU编码，自动生成 | VARCHAR | 灰色小字展示 |
| 所属SPU | 所属SPU名称 | VARCHAR | 文本展示 |
| 规格组合 | 规格组合（JSON格式） | JSON | 文本展示 |
| 销售价格 | SKU销售价格 | DECIMAL(18,2) | 文本展示 |
| 库存数量 | SKU库存数量 | INT | 数字展示，库存为0时标红 |
| SKU状态 | SKU状态 | TINYINT | 标签展示 |
| 条形码 | SKU条形码 | VARCHAR | 文本展示 |
| 创建时间 | SKU创建时间 | DATETIME | 文本展示 |

#### 界面元素
- 筛选条件区域
- 搜索框
- SKU列表容器
- 批量操作按钮组
- 分页控件

---

### 3.2 搜索商品SKU

#### 功能说明
在搜索框输入关键词，支持模糊搜索SKU信息

#### 交互逻辑
1. 搜索框位于页面顶部，筛选条件右侧
2. 搜索框默认显示占位文字"搜索SKU名称、编码或条形码"
3. 输入关键词后实时触发搜索（支持防抖，延迟500ms）
4. 支持按回车键触发搜索
5. 搜索结果处理：
   - 匹配的SKU在列表中展示
   - 匹配的关键词高亮显示
   - 无匹配结果时显示"暂无搜索结果"提示
6. 清空搜索框时恢复SKU列表的原始展示状态

#### 字段说明
| 字段名称 | 字段说明 | 匹配规则 |
|---------|---------|---------|
| SKU名称 | 按SKU名称模糊匹配 | 包含关键词即匹配 |
| SKU编码 | 按SKU编码精确匹配或前缀匹配 | 完全匹配或前缀匹配 |
| 条形码 | 按条形码精确匹配 | 完全匹配 |

#### 界面元素
- 搜索输入框
- 搜索图标
- 清空按钮（输入内容后显示）
- 搜索结果提示（无结果时显示）

### 3.3 新增商品SKU

#### 功能说明
通过弹窗表单创建新的商品SKU

#### 交互逻辑
1. 在商品SPU编辑页面的"价格库存"步骤中，点击"新增SKU"按钮
2. 弹出新增SKU弹窗
3. 表单包含以下字段：
   - **规格组合**：
     - 显示SPU已设置的所有规格组
     - 每个规格组下方显示该规格组的所有启用规格值
     - 必须为每个规格组选择一个规格值
     - 选择后自动生成规格组合
   - **SKU名称**：
     - 自动生成：SPU名称 + " " + 规格值名称组合
     - 可手动修改
   - **SKU编码**：
     - 自动生成：SPU编码 + "-" + 规格值编码组合
     - 可手动修改，修改时校验唯一性
   - **销售价格***：
     - 必填项，带*号标识
     - 输入框，支持小数
     - 默认继承SPU的销售价格
     - 必须大于0
   - **成本价格**：
     - 输入框，支持小数
     - 可选
   - **库存数量***：
     - 必填项，带*号标识
     - 输入框，只能输入整数
     - 默认为0
     - 必须大于等于0
   - **SKU主图**：
     - 图片上传，可选
     - 不设置时使用SPU的主图
   - **条形码**：
     - 输入框，可选
     - 输入时校验唯一性
4. 点击"确定"按钮：
   - 前端校验表单完整性
   - 校验规格组合是否唯一
   - 调用后端接口创建SKU
   - 创建成功后关闭弹窗，刷新SKU列表
   - 弹出成功提示："SKU创建成功"
5. 点击"取消"按钮或弹窗外部关闭弹窗，表单数据不保存

#### 字段说明
| 字段名称 | 字段说明 | 数据类型 | 约束 | 默认值 |
|---------|---------|---------|------|--------|
| 规格组合 | 规格组合（JSON格式） | JSON | 必填，唯一 | - |
| SKU名称 | SKU名称，最多200字 | VARCHAR(200) | 必填 | 自动生成 |
| SKU编码 | SKU编码 | VARCHAR(50) | 必填，唯一 | 自动生成 |
| 销售价格 | 销售价格 | DECIMAL(18,2) | 必填，大于0 | 继承SPU价格 |
| 成本价格 | 成本价格 | DECIMAL(18,2) | 可选，大于0 | 空 |
| 库存数量 | 库存数量 | INT | 必填，大于等于0 | 0 |
| SKU主图 | SKU主图URL | VARCHAR(255) | 可选 | 空 |
| 条形码 | SKU条形码 | VARCHAR(50) | 可选，唯一 | 空 |

#### 界面元素
- 弹窗标题：新增SKU
- 规格组合选择区域
- SKU名称输入框
- SKU编码输入框
- 销售价格输入框
- 成本价格输入框
- 库存数量输入框
- SKU主图上传区域
- 条形码输入框
- 确定按钮
- 取消按钮
- 表单验证提示

### 3.4 编辑商品SKU

#### 功能说明
编辑已存在的商品SKU信息

#### 交互逻辑
1. 在SKU列表的操作按钮中点击"编辑"按钮
2. 弹出编辑SKU弹窗，表单字段与新增SKU一致
3. 表单自动填充当前SKU的原有数据
4. 字段规则：
   - 规格组合：不可修改（创建后规格组合不可修改）
   - SKU名称：可修改，规则同新增
   - SKU编码：可修改，规则同新增
   - 销售价格：可修改，规则同新增
   - 成本价格：可修改，规则同新增
   - 库存数量：可修改，规则同新增
   - SKU主图：可修改，规则同新增
   - 条形码：可修改，规则同新增
5. 点击"确定"按钮：
   - 前端校验表单完整性
   - 调用后端接口更新SKU
   - 更新成功后关闭弹窗，刷新SKU列表
   - 弹出成功提示："SKU更新成功"
6. 点击"取消"按钮或弹窗外部关闭弹窗，修改不保存

#### 字段说明
同新增SKU字段说明，规格组合字段不可修改

#### 界面元素
- 弹窗标题：编辑SKU
- 其他界面元素同新增SKU
- 规格组合选择区域禁用状态

### 3.6 SKU价格管理

#### 功能说明
管理SKU的价格，包括批量修改价格

#### 交互逻辑
1. **单个SKU价格修改**：
   - 在SKU列表中点击"编辑"按钮
   - 在编辑弹窗中修改价格
   - 点击"确定"按钮保存

2. **批量修改价格**：
   - 在SKU列表中勾选需要修改价格的SKU
   - 点击"批量调整价格"按钮
   - 弹出批量调整价格弹窗
   - 支持三种方式：
     - **统一设置**：
       - 输入统一的销售价格
       - 输入统一的成本价格（可选）
       - 点击"确定"按钮，所有选中SKU设置为该价格
     - **批量调整**：
       - 选择调整方式：增加 / 减少
       - 输入调整金额
       - 点击"确定"按钮，所有选中SKU价格按金额调整
     - **按比例调整**：
       - 选择调整方式：上调 / 下调
       - 输入调整比例（如10表示上调10%）
       - 点击"确定"按钮，所有选中SKU价格按比例调整
   - 点击"确定"按钮批量更新
   - 更新成功后刷新SKU列表
   - 弹出成功提示："价格更新成功"

#### 字段说明
| 字段名称 | 字段说明 | 数据类型 |
|---------|---------|---------|
| 销售价格 | SKU销售价格 | DECIMAL(18,2) |
| 成本价格 | SKU成本价格 | DECIMAL(18,2) |

### 3.9 SKU删除

#### 功能说明
删除商品SKU

#### 交互逻辑
1. 在SKU列表的操作按钮中点击"删除"按钮
2. 只有"下架"状态的SKU可以删除
3. 弹出确认删除对话框
4. 系统执行删除前校验：
   - **校验1：检查是否有订单**
     - 如果有订单，对话框显示警告信息："该SKU有订单记录，不能删除"
     - "确认"按钮置灰，不可点击
     - 显示"取消"按钮
   - **校验2：检查是否有库存**
     - 如果有库存，对话框显示警告信息："该SKU有库存，确定要删除吗？"
     - 显示"确认"和"取消"按钮
   - **校验3：无关联数据**
     - 对话框显示确认信息："确定要删除SKU「{SKU名称}」吗？删除后不可恢复。"
     - 显示"确认"和"取消"按钮
5. 点击"确认"按钮：
   - 调用后端接口删除SKU（逻辑删除）
   - 删除成功后关闭对话框，刷新SKU列表
   - 弹出成功提示："SKU删除成功"
6. 点击"取消"按钮或弹窗外部关闭对话框，不执行删除

#### 字段说明
无额外字段，主要是校验逻辑

#### 界面元素
- 确认删除对话框
- 警告信息展示区域
- 确认按钮（校验通过时可用）
- 取消按钮

## 四、数据校验规则

### 4.1 SKU名称校验
- 必填校验：SKU名称不能为空
- 长度校验：最多200个字符
- 特殊字符校验：不能包含特殊字符（支持中文、英文、数字、下划线、横线、空格）

### 4.2 SKU编码校验
- 必填校验：SKU编码不能为空
- 长度校验：最多50个字符
- 格式校验：支持英文、数字、横线、下划线
- 唯一性校验：SKU编码在系统中唯一

### 4.3 规格组合校验
- 必填校验：规格组合不能为空
- 唯一性校验：同一SPU下的SKU规格组合必须唯一
- 完整性校验：SKU必须包含所有规格组的规格值

### 4.4 价格校验
- 必填校验：销售价格不能为空
- 数值校验：价格必须大于0
- 精度校验：价格保留2位小数

### 4.5 库存校验
- 必填校验：库存数量不能为空
- 数值校验：库存数量必须大于等于0
- 整数校验：库存数量必须是整数

### 4.6 条形码校验
- 格式校验：如果设置，必须符合条形码格式规范
- 唯一性校验：条形码在系统中唯一（如果设置）

### 4.7 删除校验
- 订单校验：有订单的SKU不能删除
- 状态校验：只有下架状态的SKU可以删除

## 五、异常处理

### 5.1 表单验证异常
- 实时显示验证错误信息
- 提交时再次校验，阻止无效提交

### 5.2 规格组合重复异常
- 保存SKU时检测到重复的规格组合
- 给出明确的错误提示

### 5.3 库存不足异常
- 减少库存时库存不足
- 给出明确的错误提示

### 5.4 网络异常
- 操作失败时显示错误提示
- 提供重试按钮
- 失败时保持表单数据，避免重复输入

### 5.5 权限异常
- 无操作权限时隐藏操作按钮
- 尝试操作时显示权限不足提示

### 5.6 状态流转异常
- 状态流转不符合规则时给出明确提示
- 如：正常状态的SKU不能直接删除，必须先下架

## 八、接口设计

### 8.1 获取商品SKU列表
**接口路径**：`GET /api/product/sku/list`  
**接口说明**：获取商品SKU列表  
**请求参数**：
- `categoryId`：分类ID（可选）
- `spuId`：SPU ID（可选）
- `status`：SKU状态（可选）
- `supplierId`：供应商ID（可选，平台端使用）
- `minPrice`：最小价格（可选）
- `maxPrice`：最大价格（可选）
- `minStock`：最小库存（可选）
- `maxStock`：最大库存（可选）
- `keyword`：搜索关键词（可选）
- `pageNum`：页码（可选）
- `pageSize`：每页数量（可选）
**响应数据**：商品SKU列表数据

### 8.2 新增商品SKU
**接口路径**：`POST /api/product/sku`  
**接口说明**：新增商品SKU  
**请求参数**：
- `spuId`：SPU ID（必填）
- `specCombination`：规格组合（JSON格式）（必填）
- `skuName`：SKU名称（必填）
- `skuCode`：SKU编码（必填）
- `salePrice`：销售价格（必填）
- `costPrice`：成本价格（可选）
- `stock`：库存数量（必填）
- `skuImage`：SKU主图URL（可选）
- `barcode`：条形码（可选）
**响应数据**：新增的商品SKU信息

### 8.3 编辑商品SKU
**接口路径**：`PUT /api/product/sku/{id}`  
**接口说明**：编辑商品SKU  
**请求参数**：
- `skuName`：SKU名称（必填）
- `skuCode`：SKU编码（必填）
- `salePrice`：销售价格（必填）
- `costPrice`：成本价格（可选）
- `stock`：库存数量（必填）
- `skuImage`：SKU主图URL（可选）
- `barcode`：条形码（可选）
**响应数据**：更新后的商品SKU信息

**接口路径**：`GET /api/product/sku/{id}`  
**接口说明**：获取商品SKU详情
