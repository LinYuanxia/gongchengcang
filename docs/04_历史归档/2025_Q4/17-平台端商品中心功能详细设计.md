# 平台端商品中心功能详细设计

## 一、功能概述

### 1.1 功能定位

平台端商品中心是平台管理员和运营人员管理商品基础数据的核心模块，负责维护商品的基础信息体系，包括商品分类、商品属性、规格属性、SPU、SKU等。平台端商品中心是整个系统的商品数据基础，为供应商、工程仓、施工方等各业务端提供标准化的商品数据支持。

### 1.2 功能价值

- **数据标准化**：建立统一的商品数据标准，确保各业务端使用一致的商品数据
- **分类管理**：建立清晰的商品分类体系，便于商品组织和查找
- **属性管理**：灵活配置商品属性和规格，支持不同类型商品的特性描述
- **SPU/SKU管理**：建立标准化的商品基础信息库，支持多规格商品管理
- **数据支撑**：为供应商供货、工程仓采购、施工方下单等业务提供基础数据支撑

### 1.3 业务范围

- 商品分类管理：分类的创建、编辑、查询、删除、排序
- 商品属性管理：属性组、属性值的创建、编辑、查询、删除
- 规格属性管理：规格组、规格值的创建、编辑、查询、删除
- SPU管理：SPU的创建、编辑、查询、删除、导入导出
- SKU管理：SKU的创建、编辑、查询、删除、导入导出
- 数据统计：商品数据统计分析

## 二、商品分类功能详细设计

### 2.1 功能说明

商品分类功能负责管理商品的分类体系，支持三级分类结构，提供分类的增删改查、排序、拖拽等功能。分类是商品组织的基础，所有商品都必须关联到具体的分类上。

### 2.2 业务场景

**场景1：新建分类体系**
- 系统初始化时，需要建立完整的商品分类体系
- 从一级分类开始创建，逐步建立二级、三级分类
- 每个分类需要设置分类名称、编码、排序、状态等信息

**场景2：调整分类结构**
- 业务发展需要调整分类层级关系
- 通过拖拽功能调整分类的父级关系
- 调整后，该分类下的所有商品自动归属到新的分类路径下

**场景3：禁用分类**
- 某个分类不再使用，需要禁用该分类
- 禁用后，该分类下不能创建新商品
- 已有的商品仍保留该分类，但在新增商品时不可选择

**场景4：删除分类**
- 某个分类完全不需要了，需要删除
- 删除前需要检查该分类下是否有商品
- 如果有商品，则不允许删除，提示先处理商品关联

**场景5：查看分类关联商品**
- 需要了解某个分类下有多少SPU和SKU
- 点击分类卡片上的SPU/SKU数量，跳转到SPU/SKU列表页面
- 列表页面自动筛选该分类下的商品

#### 2.3.2 页面加载

1. 进入页面后，同时加载两部分数据：
   - 加载分类树数据，用于左侧树形展示
   - 加载一级分类列表，用于顶部卡片展示

2. 初始化搜索框为空
3. 初始化选中分类为空
4. 初始化右侧详情区域为空状态

#### 2.3.3 搜索分类

**功能说明**：通过分类名称快速搜索分类，支持模糊搜索。

**交互逻辑**：
1. 用户在搜索框中输入分类名称
2. 支持实时搜索（输入后自动搜索）或点击搜索按钮搜索
3. 搜索后：
   - 分类树自动展开匹配的节点
   - 分类卡片区域只显示匹配的分类（包括所有层级）
   - 搜索结果高亮显示匹配的关键词
4. 搜索无结果时：
   - 分类树显示空状态
   - 分类卡片区域显示空状态提示
   - 提示"未找到匹配的分类"

**字段说明**：
- **搜索关键词**：分类名称，支持模糊匹配

**操作按钮说明**：
- **搜索按钮**：触发搜索
- **清除按钮**：清除搜索条件，显示全部分类

**权限要求**：
- 平台管理员：可使用
- 平台运营：可使用
- 供应商：可使用（只读）
- 工程仓：可使用（只读）
- 施工方：可使用（只读）

**异常处理**：
- 网络错误：显示错误提示，提供重试按钮
- 搜索无结果：显示无搜索结果提示，建议调整搜索条件

#### 2.3.4 查看分类卡片

**功能说明**：以卡片形式展示一级分类，显示分类名称、SPU数量、SKU数量等信息。

**交互逻辑**：
1. 页面加载后，顶部显示所有一级分类的卡片
2. 每个卡片显示：
   - 分类名称（大字体）
   - SPU数量（该分类及子分类下的SPU总数）
   - SKU数量（该分类及子分类下的SKU总数）
   - 查看SPU按钮
   - 查看SKU按钮
3. 点击分类卡片：
   - 左侧分类树自动选中该分类
   - 右侧显示该分类的详情信息
   - 卡片高亮显示
4. 点击"查看SPU"按钮：
   - 跳转到SPU列表页面
   - SPU列表页面自动筛选该分类下的SPU
5. 点击"查看SKU"按钮：
   - 跳转到SKU列表页面
   - SKU列表页面自动筛选该分类下的SKU

**字段说明**：
- **分类名称**：分类的名称
- **SPU数量**：该分类及子分类下的SPU总数
- **SKU数量**：该分类及子分类下的SKU总数

**操作按钮说明**：
- **查看SPU**：跳转到SPU列表页面，筛选该分类下的SPU
- **查看SKU**：跳转到SKU列表页面，筛选该分类下的SKU

**权限要求**：
- 平台管理员：可使用
- 平台运营：可使用
- 供应商：可使用（只读）
- 工程仓：可使用（只读）
- 施工方：可使用（只读）

**异常处理**：
- 无数据：显示空状态提示，引导创建分类

#### 2.3.5 分类树操作

**功能说明**：以树形结构展示所有分类，支持展开/收起、点击选择、拖拽排序等操作。

**交互逻辑**：
1. **展开/收起**：
   - 点击分类前的箭头图标，展开/收起子分类
   - 展开时显示该分类下的所有子分类
   - 收起时隐藏子分类
   - 默认展开一级分类

2. **点击选择**：
   - 点击分类名称，选中该分类
   - 选中的分类高亮显示
   - 右侧显示该分类的详情信息
   - 顶部分类卡片中对应的分类也高亮显示

3. **拖拽排序**：
   - 支持拖拽分类调整顺序
   - 支持拖拽分类调整父级关系（改变分类层级）
   - 拖拽时显示拖拽提示线
   - 拖拽完成后，自动保存排序和层级关系
   - 拖拽限制：
     - 不能将分类拖拽到自己的子分类下
     - 不能超过三级分类（将二级分类拖拽到二级分类下会变成三级分类）
     - 不能将三级分类再拖拽子分类（已达到最大层级）

**字段说明**：
- **分类名称**：分类的名称
- **分类编码**：分类的唯一编码
- **分类层级**：分类的层级（一级、二级、三级）

**操作按钮说明**：
- **展开/收起箭头**：展开/收起子分类
- **拖拽手柄**：拖拽分类调整顺序和层级

**权限要求**：
- 平台管理员：可使用（包括拖拽）
- 平台运营：可使用（包括拖拽）
- 供应商：可使用（只读，不可拖拽）
- 工程仓：可使用（只读，不可拖拽）
- 施工方：可使用（只读，不可拖拽）

**异常处理**：
- 拖拽到无效位置：自动回弹到原位置，并提示"不能将分类拖拽到该位置"
- 拖拽超过三级分类：自动回弹到原位置，并提示"分类最多支持三级"
- 网络错误：拖拽后保存失败，提示保存失败，可重试

#### 2.3.6 分类详情

**功能说明**：显示选中分类的详细信息，包括基本信息、关联商品数量等。

**交互逻辑**：
1. 选中分类后，右侧显示该分类的详情
2. 详情信息包括：
   - 分类名称
   - 分类编码
   - 父级分类（显示父级分类名称，可点击跳转）
   - 分类层级（一级分类/二级分类/三级分类）
   - 排序号
   - 状态（启用/禁用）
   - 创建时间
   - 更新时间
   - 关联SPU数量（可点击查看）
   - 关联SKU数量（可点击查看）
3. 操作按钮：
   - 编辑按钮：打开编辑分类抽屉
   - 删除按钮：删除该分类（需二次确认）
   - 启用/禁用按钮：切换分类状态

**字段说明**：
- **分类名称**：分类的名称，必填，长度1-20字符
- **分类编码**：分类的唯一编码，必填，长度1-50字符，创建后不可修改
- **父级分类**：分类的父级分类，一级分类无父级
- **分类层级**：分类的层级（一级、二级、三级）
- **排序号**：分类的排序号，数字越小越靠前
- **状态**：分类的状态（启用/禁用）
- **创建时间**：分类的创建时间
- **更新时间**：分类的最后更新时间
- **关联SPU数量**：该分类下的SPU数量（包括子分类）
- **关联SKU数量**：该分类下的SKU数量（包括子分类）

**操作按钮说明**：
- **编辑**：打开编辑分类抽屉
- **删除**：删除该分类（需二次确认）
- **启用/禁用**：切换分类状态

**权限要求**：
- 平台管理员：可使用所有操作
- 平台运营：可使用所有操作
- 供应商：可查看详情，不可操作
- 工程仓：可查看详情，不可操作
- 施工方：可查看详情，不可操作

**异常处理**：
- 分类不存在：提示分类不存在，引导返回
- 网络错误：显示错误提示，提供重试按钮

#### 2.3.7 新增分类

**功能说明**：创建新的分类，支持创建一级、二级、三级分类。

**交互逻辑**：
1. 点击"新增分类"按钮，打开新增分类抽屉
2. 表单字段：
   - **分类名称**：输入框，必填，长度1-20字符
   - **分类编码**：输入框，必填，长度1-50字符，系统自动生成（格式：CAT001-1-1），可手动修改
   - **父级分类**：下拉选择框，选择父级分类（不选为一级分类）
   - **排序号**：输入框，数字，默认10
   - **状态**：单选框，启用/禁用，默认启用
3. 表单验证：
   - 分类名称必填，长度1-20字符
   - 分类编码必填，长度1-50字符，唯一
   - 排序号必填，数字
   - 父级分类选择时，检查是否超过三级分类
4. 点击"保存"按钮：
   - 验证通过后，创建分类
   - 关闭抽屉，刷新分类树和分类卡片
   - 显示成功提示
5. 点击"取消"按钮：
   - 关闭抽屉，不保存数据

**字段说明**：
- **分类名称**：分类的名称，必填，长度1-20字符
- **分类编码**：分类的唯一编码，必填，长度1-50字符，系统内唯一，创建后不可修改
- **父级分类**：分类的父级分类，不选为一级分类
- **排序号**：分类的排序号，数字越小越靠前
- **状态**：分类的状态（启用/禁用）

**操作按钮说明**：
- **取消**：关闭抽屉，不保存数据
- **保存**：提交表单，创建分类

**权限要求**：
- 平台管理员：可使用
- 平台运营：可使用
- 供应商：不可使用
- 工程仓：不可使用
- 施工方：不可使用

**异常处理**：
- 分类名称重复：提示分类名称已存在
- 分类编码重复：提示分类编码已存在
- 超过三级分类：提示"分类最多支持三级"
- 网络错误：提示保存失败，可重试
- 表单验证失败：显示具体错误信息，引导用户修改

5. 点击"取消"按钮：
   - 关闭抽屉，不保存数据

**字段说明**：
- **分类名称**：分类的名称，必填，长度1-20字符
- **排序号**：分类的排序号，数字越小越靠前
- **状态**：分类的状态（启用/禁用）

**操作按钮说明**：
- **取消**：关闭抽屉，不保存数据
- **保存**：提交表单，更新分类

**权限要求**：
- 平台管理员：可使用

#### 2.3.9 删除分类

**功能说明**：删除已有的分类，删除前需要检查是否有关联商品。

**交互逻辑**：
1. 点击分类详情中的"删除"按钮，弹出二次确认对话框
2. 确认对话框显示：
   - 提示信息："确定要删除该分类吗？"
   - 警告信息："删除后无法恢复，请谨慎操作！"
   - 检查该分类下是否有SPU或SKU：
     - 如果有：显示"该分类下有XX个SPU和XX个SKU，无法删除，请先处理商品关联。"
     - 如果没有：显示"该分类下没有关联商品，可以删除。"
3. 点击"确认"按钮：
   - 如果有关联商品：提示无法删除，关闭对话框
   - 如果没有关联商品：执行删除操作，刷新分类树和分类卡片，显示成功提示
4. 点击"取消"按钮：
   - 关闭对话框，不删除

**字段说明**：
- **分类ID**：分类的唯一标识

**操作按钮说明**：
- **取消**：关闭对话框，不删除
- **确认**：执行删除操作

**权限要求**：
- 平台管理员：可使用
- 平台运营：可使用
- 供应商：不可使用
- 工程仓：不可使用
- 施工方：不可使用

### 2.4 业务规则

#### 2.4.1 分类层级规则

- **层级限制**：最多支持三级分类
- **层级编码**：分类编码采用层级编码方式，如：
  - 一级分类：CAT001
  - 二级分类：CAT001-1
  - 三级分类：CAT001-1-1
- **层级路径**：分类的完整路径，如：钢材 > 螺纹钢 > HRB400E

#### 2.4.2 分类命名规则

- **名称必填**：分类名称必须填写
- **名称长度**：建议不超过20字符
- **名称唯一**：同一层级下分类名称不可重复
- **名称清晰**：分类名称应清晰描述商品类别

#### 2.4.3 分类编码规则

- **编码唯一性**：分类编码在系统中唯一
- **编码格式**：建议使用CAT+序号+层级编码，如CAT001-1-1
- **编码不可修改**：分类编码创建后不可修改
- **编码长度限制**：建议不超过50字符

#### 2.4.4 分类状态规则

- **启用状态**：分类正常使用，可以创建新商品
- **禁用状态**：分类不可创建新商品，已有的商品不受影响
- **状态切换**：支持随时切换分类状态

#### 2.4.5 分类删除规则

- **删除前检查**：删除分类前需要检查是否有商品关联
- **级联删除**：如果分类下有商品，不允许直接删除
- **软删除**：建议采用软删除机制，保留删除记录
- **删除确认**：删除分类需要二次确认

## 三、商品属性功能详细设计

### 3.1 功能说明

商品属性功能负责管理商品的属性体系，包括属性组和属性值。属性是描述商品特征的元数据，用于商品的搜索、筛选和详情展示。平台端可以全局配置商品属性，所有SPU都可以关联这些属性。

### 3.2 业务场景

**场景1：新建属性组**
- 需要为某类商品创建属性组，如"钢材属性"、"水泥属性"等
- 属性组用于归类相关的属性，便于管理和使用
- 每个属性组可以包含多个属性值

**场景2：添加属性值**
- 为属性组添加具体的属性值，如"材质"、"产地"、"品牌"等
- 属性值是描述商品特征的具体字段
- 每个属性值需要设置数据类型、是否必填、默认值等

**场景3：编辑属性值**
- 需要修改属性值的名称、数据类型、是否必填等配置
- 修改后，所有使用该属性的SPU都会受影响
- 需要考虑数据兼容性问题

**场景4：删除属性组/值**
- 某个属性组或属性值不再需要了，需要删除
- 删除前需要检查是否有SPU使用该属性
- 如果有SPU使用，则不允许删除，提示先处理关联

**场景5：属性排序**
- 需要调整属性组或属性值的显示顺序
- 通过拖拽功能调整顺序
- 调整后，SPU编辑页面的属性显示顺序也会同步更新

### 3.4 业务规则

#### 3.4.1 属性组规则

- **名称必填**：属性组名称必须填写
- **名称长度**：建议不超过50字符
- **名称唯一**：属性组名称不可重复
- **编码唯一**：属性组编码在系统中唯一
- **编码不可修改**：属性组编码创建后不可修改

#### 3.4.2 属性值规则

- **名称必填**：属性值名称必须填写
- **名称长度**：建议不超过50字符
- **名称唯一**：同一属性组下属性值名称不可重复
- **编码唯一**：属性值编码在系统中唯一
- **编码不可修改**：属性值编码创建后不可修改
- **数据类型**：支持字符串、数字、布尔、日期、枚举五种数据类型
- **枚举值**：数据类型为枚举时，必须设置枚举值
- **必填设置**：支持设置是否必填

#### 3.4.3 属性关联规则

- **属性组与属性值**：一个属性组可以包含多个属性值，一个属性值只能属于一个属性组
- **SPU与属性**：一个SPU可以关联多个属性（多个属性组的多个属性值）
- **属性值数据**：SPU关联属性时，需要填写属性值数据
- **属性值验证**：SPU保存时，验证必填属性值是否填写，数据类型是否正确

#### 3.4.4 属性删除规则

- **删除前检查**：删除属性组前需要检查是否有属性值
- **删除前检查**：删除属性值前需要检查是否有SPU使用
- **级联删除**：如果有关联数据，不允许直接删除
- **软删除**：建议采用软删除机制，保留删除记录
- **删除确认**：删除需要二次确认

## 四、规格属性功能详细设计

### 4.1 功能说明

规格属性功能负责管理商品的规格体系，包括规格组和规格值。规格是用于区分同一SPU下不同SKU的特征属性，如颜色、尺寸、材质等。平台端可以全局配置规格属性，所有SKU都需要关联这些规格。

规格属性与商品属性的区别：
- **商品属性**：用于描述商品的特征，不影响SKU的区分
- **规格属性**：用于区分同一SPU下的不同SKU，是SKU的关键属性

### 4.2 业务场景

**场景1：新建规格组**
- 需要为某类商品创建规格组，如"钢材规格"、"水泥规格"等
- 规格组用于归类相关的规格，便于管理和使用
- 每个规格组可以包含多个规格值

**场景2：添加规格值**
- 为规格组添加具体的规格值，如"直径"、"长度"、"强度等级"等
- 规格值是区分SKU的具体字段
- 每个规格值需要设置数据类型、单位等

**场景3：编辑规格值**
- 需要修改规格值的名称、数据类型、单位等配置
- 修改后，所有使用该规格的SKU都会受影响
- 需要考虑数据兼容性问题

**场景4：删除规格组/值**
- 某个规格组或规格值不再需要了，需要删除
- 删除前需要检查是否有SKU使用该规格
- 如果有SKU使用，则不允许删除，提示先处理关联

**场景5：规格排序**
- 需要调整规格组或规格值的显示顺序
- 通过拖拽功能调整顺序
- 调整后，SKU编辑页面的规格显示顺序也会同步更新
