## 一、功能概述

### 1.1 功能定位

业务流程设计是工程仓端所有功能模块的**业务操作指南**，描述从采购到销售、从入库到出库的完整业务流转。本文档面向产品、开发、测试团队，帮助理解工程仓在双交易链路中的位置和各流程的业务规则。

### 1.2 核心概念

| 概念 | 说明 | 涉及链路 |
|-----|------|:--------:|
| 链路一（采购） | 工程仓向供应商采购商品的全流程 | 采购 |
| 链路二（销售） | 施工方向工程仓采购商品的全流程 | 销售 |
| 入库 | 采购订单到货后验收入库 | 采购 |
| 出库 | 销售订单确认后发货出库 | 销售 |
| 货损售后 | 收货时发现货物损坏后的处理流程 | 采购/销售 |
| 发票对账 | 订单完成后的财务结算流程 | 采购/销售 |

### 1.3 目标用户

- **产品经理**：理解业务链路，指导功能设计
- **开发工程师**：了解业务上下文，指导数据库设计和接口开发
- **测试工程师**：基于流程设计测试用例
- **运营人员**：了解操作流程，指导日常使用

## 二、双交易链路概述

工程仓端在平台中处于**中枢位置**，同时参与两条核心交易链路：

```
链路一：工程仓采购（工程仓→供应商）
═════════════════════════════════════════════════════
    ┌──────────┐    采购订单    ┌──────────┐
    │  工程仓端  │ ───────────▶ │  供应商端  │
    │ (买方→采购)│◀───────────│  (卖方→发货)│
    └──────────┘      发货      └──────────┘

链路二：施工方采购（施工方→工程仓）
═════════════════════════════════════════════════════
    ┌──────────┐    销售订单    ┌──────────┐
    │  施工方端  │ ───────────▶ │  工程仓端  │
    │ (买方→下单)│◀───────────│  (卖方→发货)│
    └──────────┘      发货      └──────────┘
```

### 2.1 双链路业务规则

| 规则 | 链路一（采购） | 链路二（销售） |
|:----:|:-------------:|:-------------:|
| 买方 | 工程仓 | 施工方 |
| 卖方 | 供应商 | 工程仓 |
| 订单类型 | 采购订单 | 销售订单 |
| 支付方向 | 工程仓→供应商 | 施工方→工程仓 |
| 开票方向 | 供应商开给工程仓（进项） | 工程仓开给施工方（销项） |
| 核心操作 | 采购员加购下单→仓管员收货 | 销售确认订单→仓管员发货 |

## 三、链路一：工程仓采购流程（核心流程）

### 3.1 采购下单流程

```mermaid
sequenceDiagram
    participant 采购员 as 采购员
    participant 系统 as 工程仓系统
    participant 供应商 as 供应商端

    采购员->>系统: 浏览商品市场（分类/搜索）
    采购员->>系统: 加入购物车
    采购员->>系统: 去结算
    采购员->>系统: 确认订单（勾选协议）
    系统->>系统: 校验库存
    系统->>系统: 创建采购订单（状态：待确认）
    系统->>供应商: 推送订单通知
    系统->>采购员: 跳转订单详情
```

**采购下单业务规则：**
1. 采购员浏览商品市场 → 支持分类/搜索/价格过滤多种筛选方式
2. 加入购物车 → 跨供应商选购，购物车按供应商分组
3. 结算 → 按供应商拆分为多个独立采购订单
4. 提交前二次校验库存（防止库存变化）
5. 创建成功 → 通知供应商端

### 3.2 采购订单状态流转

```mermaid
stateDiagram-v2
    [*] --> 待确认: 采购员创建订单
    待确认 --> 已确认: 供应商确认
    待确认 --> 已取消: 采购员取消
    已确认 --> 待发货: 供应商备货中
    已确认 --> 已关闭: 供应商无法发货
    待发货 --> 待收货: 供应商发货
    待收货 --> 已完成: 仓管员收货入库
    已完成 --> 售后中: 货损售后
    售后中 --> 已完成: 售后处理完成
    已取消 --> [*]
    已关闭 --> [*]
    已完成 --> [*]
```

**状态流转规则：**
- 待确认（pending）→ 供应商48小时内确认，超时自动提醒
- 已确认（confirmed）→ 订单锁定，采购员不可取消
- 待收货（shipped）→ 仓管员联系供应商安排收货
- 已完成（completed）→ 三状态各自独立：订单完成/支付可能未付/发货已完结

### 3.3 采购入库流程

```mermaid
graph TB
    A[供应商发货] --> B[入库单生成]
    B --> C[仓管员打开入库单]
    C --> D[选择子仓库]
    D --> E[填写实收数量]
    E --> F{有货损?}
    F -->|是| G[填写货损数量+上传图片]
    F -->|否| H[直接确认]
    G --> H
    H --> I[提交收货]
    I --> J{校验通过?}
    J -->|否| K[提示错误]
    J -->|是| L[订单→已完成]
    L --> M[库存增加]
    M --> N[有货损→生成售后单]
    N --> O[通知供应商]
```

**入库校验规则：**
- 实收数量 ≤ 下单数量（不允许多收）
- 货损数量 ≤ 实收数量
- 货损 > 0 → 自动生成售后单

## 四、链路二：施工方采购流程（销售链路）

### 4.1 销售订单流转

```mermaid
sequenceDiagram
    participant 施工方 as 施工方
    participant 系统 as 工程仓系统
    participant 仓管员 as 仓管员/销售

    施工方->>系统: 提交采购订单
    系统->>仓管员: 收到销售订单通知
    仓管员->>系统: 查看订单详情
    仓管员->>系统: 确认订单（或取消）
    系统->>系统: 生成出库单（待出库）
    仓管员->>系统: 确认出库（选仓库+填数量）
    系统->>系统: 库存扣减
    系统->>施工方: 通知已发货
```

**销售流转规则：**
1. 施工方提交 → 工程仓销售/主管收到通知
2. 确认订单 → 自动生成出库单（待出库）
3. 出库发货 → 库存校验 → 扣减库存
4. 物流可选（自提场景）

### 4.2 销售订单状态流转

```mermaid
stateDiagram-v2
    [*] --> 待确认: 施工方创建订单
    待确认 --> 已确认: 工程仓确认
    待确认 --> 已取消: 工程仓拒绝
    已确认 --> 待出库: 生成出库单
    待出库 --> 待收货: 仓管员确认出库
    待收货 --> 已完成: 施工方确认收货
    已完成 --> 售后中: 售后处理
    售后中 --> 已完成: 售后完成
    已取消 --> [*]
    已完成 --> [*]
```

## 五、仓储作业流程

### 5.1 入库全流程

```mermaid
graph TB
    subgraph 入库全流程
        A[采购订单→待收货] --> B[仓管员点击「开始收货」]
        B --> C[选择子仓库]
        C --> D[逐商品填写实收数量]
        D --> E{实收<下单?}
        E -->|是| F[填写货损数量]
        F --> G[上传货损图片（最多5张）]
        G --> H[填写货损说明]
        E -->|否| H
        H --> I[提交]
        I --> J{校验: 实收≤下单 / 货损≤实收}
        J -->|失败| K[提示具体错误]
        J -->|成功| L[订单→已完成]
        L --> M[库存+=实收数量]
        M --> N[生成入库批次记录]
        N --> O{货损>0?}
        O -->|是| P[生成售后单]
        P --> Q[通知供应商端]
        O -->|否| R[完成]
    end
```

**入库关键业务规则：**
- 入库单由采购订单自动生成（供应商发货后）
- 实收数量可小于下单（允许部分到货/少发）
- 有货损必须拍照留证
- 入库后库存实时增加

### 5.2 出库全流程

```mermaid
graph TB
    subgraph 出库全流程
        A[销售订单→施工方创建] --> B[工程仓确认订单]
        B --> C[生成出库单（待出库）]
        C --> D[仓管员点击「确认出库」]
        D --> E[选择子仓库]
        E --> F[填写实发数量]
        F --> G{库存充足?}
        G -->|否| H[提示: 库存不足 剩余N件]
        G -->|是| I[填写物流信息（非必填）]
        I --> J[提交]
        J --> K[库存扣减（乐观锁）]
        K --> L[出库单→已完成]
        L --> M[销售订单→待收货]
    end
```

**出库关键业务规则：**
- 出库单由销售订单确认后自动生成
- 库存扣减使用乐观锁防止超卖
- 物流信息非必填（适配自提场景）
- 允许部分发货（实发≤应发）

## 六、货损售后流程

```mermaid
graph TB
    A[收货时发现货损] --> B[填写货损数量]
    B --> C[上传货损图片]
    C --> D[提交收货]
    D --> E[系统自动生成售后单]
    E --> F[售后单状态:processing]
    F --> G[供应商端收到售后提醒]
    G --> H{供应商处理}
    H -->|同意补发| I[供应商安排补发]
    H -->|同意退款| J[走退款流程]
    H -->|拒绝| K[需要平台介入]
    I --> L[售后单→已完成]
    J --> L
    K --> L
```

**售后规则：**
- 售后单在入库时自动生成（货损>0）
- 供应商必须处理售后（补发/退款/拒绝）
- 拒绝时需平台介入仲裁
